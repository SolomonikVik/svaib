# Онбординг

## Кратко

Последовательность, в которой клиент осваивает систему Second AI Brain: от первого артефакта до работающей системы. Каждый этап заканчивается работающим артефактом — клиент получает ценность сразу, не "когда всё будет готово".

## Связанные файлы

- methodology.md — ритуалы и протоколы (начинают работать после онбординга)
- ../ontology/ontology.md — какие сущности заполняются на каждом этапе
- ../scaffold/ — каркас, который клиент разворачивает и заполняет
- ../../clients/playbook/delivery/01_delivery_plan.md — тактика онбординга (ДЗ, шаблоны сообщений, инструменты)

---

## Scope системы

Second AI Brain — AI-инфраструктура для управления бизнесом. Система знает руководителя как личность (ценности, стиль, компетенции), чтобы помогать в бизнесе. Личные домены вне бизнеса (здоровье, хобби, семья) — вне scope. Scaffold разворачивается только для бизнес-проектов. В `01_ceo/01_my_profile.md` могут упоминаться значимые не-бизнес-проекты, если они влияют на бизнес-решения (обучение, переезд) — но как контекст, не как отдельная инфраструктура.

При онбординге объяснить клиенту эту границу — чтобы в рабочем пространстве не появлялись медкарты и хобби-проекты.

Детали решения — [04_decisions.md](../04_decisions.md) (пункт 2 — Границы продукта).

## Принципы

1. **Каждый этап = работающий артефакт.** Не "подготовка к будущему", а ценность прямо сейчас. `01_ceo/01_my_profile.md` уже персонализирует AI. Vision уже даёт фокус.
2. **От личного к бизнесу.** Сначала человек (`01_ceo/01_my_profile.md`), потом курс компании (`02_strategy/`), потом операционка (`03_team/`, `04_management/`, `processes/`). "Личное" здесь = ценности и стиль мышления в контексте бизнеса, не вся жизнь.
3. **Минимальный жизнеспособный набор.** Система "запускается" после этапа 2 (`01_ceo/01_my_profile.md` + `02_strategy/01_vision.md` + `02_strategy/02_goal.md` + `02_strategy/03_plan.md`). Остальное — расширение.
4. **Верификация артефактов.** Заполненный файл нужно проверить — работает ли он для AI как контекст (см. секцию "Верификация").

## Последовательность

### Этап 0. Pre-work

**Что:** Клиент до первой встречи собирает исходные данные.

- Список проектов/направлений (2-3 ключевых)
- Существующие документы (стратегия, планы, если есть)
- Формулировка болей ("что не работает", "чего не хватает")

**Артефакт:** Входные данные для первой встречи.

### Этап 1. Персональный профиль CEO

**Что:** Заполнение `01_ceo/01_my_profile.md` — кто клиент, ценности, стиль мышления, фокус.

**Как:** Через AI-интервью (AI задаёт вопросы по scaffold-шаблону) или самостоятельно.

**Верификация:** Инкогнито-тест (см. секцию ниже).

**Артефакт:** Заполненный `01_ceo/01_my_profile.md`. AI уже может персонализировать рекомендации.

### Этап 2. Vision + Goals + Plan — управленческое ядро

**Что:** Заполнение `02_strategy/01_vision.md` → `02_strategy/02_goal.md` → `02_strategy/03_plan.md`.

**Как:** Последовательно, каждый следующий файл опирается на предыдущий. Vision → "куда идём". Goal → "что хотим достичь в этом периоде". Plan → "что делаем на этой неделе".

**Артефакт:** Работающий управленческий контур. AI знает: кто клиент, куда идёт, что делает сейчас. Можно начинать еженедельный ритуал.

**Система запущена.** После этого этапа клиент может работать в еженедельном ритме (methodology.md → "Еженедельный ритуал").

### Этап 3. Настройка AI-интерфейса

**Что:** Конфигурация AI-инструмента клиента (Claude Projects, Cowork, VS Code) — User Preferences, Project Instructions, подключение файлов.

**Как:** На основе заполненных `01_ceo/01_my_profile.md` + `02_strategy/01_vision.md` + `02_strategy/02_goal.md`. Можно генерировать через промпт.

**Артефакт:** Настроенный AI-интерфейс, который знает контекст клиента.

### Этап 4+. Расширение — по мере необходимости

**Что:** Постепенное заполнение сущностей по мере работы. Файлы уже существуют с первого дня — вопрос в том, когда наполнять:

| Сущность | Когда заполнять |
|----------|----------------|
| 02_strategy/06_decisions.md | При первом стратегическом решении компании |
| 01_ceo/03_thoughts.md | При накоплении идей, гипотез, рефлексий |
| 01_ceo/04_decisions.md | При первом личном решении CEO |
| 03_team/{person}.md | При обсуждении людей (файлы существуют, заполняются из встреч) |
| 04_management/{org_node}/ | При первой обработке встречи по подразделению/направлению |
| product/, processes/, clients/, knowledge/ | По живой работе в домене (опциональные папки) |

**Принцип:** корневое ядро (`01_ceo/`, `02_strategy/`, `03_team/`, `04_management/`) существует с первого дня. Узлы в `04_management/{org_node}/` создаются только когда появляется живая работа по подразделению/направлению. Опциональные папки (`product/`, `processes/`, `clients/`, `knowledge/`) добавляются по необходимости.

## Верификация артефактов

### Инкогнито-тест

**Суть:** Заполненный файл бросается в "чистый" AI-чат (без контекста сессии, без истории) с промптом-верификатором. AI оценивает файл как независимый читатель.

**Зачем:** Тестирует сам артефакт, а не способность клиента объяснить себя в диалоге. Если AI в чистом чате понимает клиента — файл работает.

**Промпт-верификатор (для MyProfile):**

> Ты — AI-ассистент, который будет работать с этим человеком как партнёр: помогать думать, планировать, принимать решения, готовить документы. Этот профиль — твой ключевой источник контекста о нём. Ты обращаешься к нему, чтобы персонализировать рекомендации, фильтровать решения через его ценности и понимать, в каком контексте он работает.
>
> Прочитай профиль и ответь:
> 1. Что тебе стало понятно про этого человека? Кратко, своими словами.
> 2. Достаточно ли тебе этого, чтобы начать эффективно работать с ним?
> 3. Где есть противоречия или несостыковки?
> 4. Какие вопросы остались — что бы ты хотел уточнить?
> 5. Есть ли что-то лишнее, что не помогает тебе как AI-партнёру?

**Статус:** Проверено на `01_ceo/01_my_profile.md` (март 2026). Применимость к другим сущностям (`02_strategy/01_vision.md`, `02_strategy/02_goal.md`) — открытый вопрос.

## Фазы программы (стратегический горизонт)

Этапы выше — минимальная последовательность. В рамках полной программы работы с клиентом выделяются фазы:

| Фаза | Фокус | Результат |
|------|-------|-----------|
| 0. Pre-work | Сбор данных | `01_ceo/01_my_profile.md` |
| 1. Personal OS | Личная система | Настроенный AI, `01_ceo/01_my_profile.md`, `02_strategy/` и первые ритуалы |
| 2. Business context | Контекст компании | Scaffold компании, ключевые процессы |
| 3. Processes + автоматизация | Оживление системы | Автоматизированный процесс, метрики |
| 4. Масштабирование | Развитие | 90-дневный план развития |

Принцип прогрессии: **личное → команда → компания → измерение**. Каждая фаза производит артефакт, который "работает на следующий день".

Детали по фазам уточняются через реальные кейсы.

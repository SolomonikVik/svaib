🟢 Режим работы с информацией

**Формулировка:** Каждый файл работает с информацией в одном режиме: permanent, current, log, Immutable
**Зачем:** чтобы человек и агент понимали, как записывать информацию: аккуратно обновлять устойчивую правду, поддерживать актуальное состояние, дописывать, или ни в коем случае не трогать. Смешивание режимов превращает файл в свалку.

| Режим         | Дисциплина записи                  | Что важно при работе                                                          |
| ------------- | ---------------------------------- | ----------------------------------------------------------------------------- |
| **Permanent** | Подумай 10 раз перед изменением    | Не сломать то, что есть. Обоснование, проверка, аккуратная правка одного поля |
| **Current**   | Перезаписывай регулярно            | Держать актуальным. Отсутствие записи — это устаревание                       |
| **Log**       | Дописывай не думая, передавай суть | Зафиксировать факт, не потерять. Старое не трогаешь                           |
| **Immutable** | Не редактируй никогда              | Создано однажды, остаётся как есть.                                           |

**Как применять:** 
* определить режим файла до того, как в него что-то писать. 
* если один файл требует двух режимов одновременно — разделить файл.


### 4. G/Y/R (светофор) — универсальный сигнальный примитив scaffold

**G/Y/R (светофор)** — компактный сигнал состояния. Один символ + одна строка комментария.

| Символ | Значение                           |
| ------ | ---------------------------------- |
| 🟢     | Идёт по плану                      |
| 🟡     | Риск / задержка / требует внимания |
| 🔴     | Блокер / срыв / эскалация          |
|        |                                    |

Аббревиатура `G/Y/R` понятна нейросетям; для человека всегда подписывается «(светофор)», чтобы не путать.

**Где может появляться (по контексту, опционально):**

- `01_overview.md` — статус единицы целиком
- `02_active.md` — у задач / вопросов / рисков
- `02_backlog.md` — приоритет горящих позиций
- и т.д.

**Принцип:** примитив описан один раз на уровне общих конвенций scaffold. В шаблонах конкретных файлов не дублируется — используется по канону, без переопределений. Где уместно — добавляется опционально.

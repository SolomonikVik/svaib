# Scaffold README spec — канон карты папки

Канон `README.md` как карты папки в scaffold для AI-агента: структура, обязательные блоки, миссия папки + миссии файлов внутри неё.

## 🔵 Миссия README

README — карта папки. Главная функция — маршрутизация AI-агента: что лежит, куда идти, что читать дальше. Всё, что не помогает агенту принять решение, — режется.

**README клиентского scaffold — воплощение канона, не первоисточник миссий.** Канон миссий — в [03_node-files.md](03_node-files.md) (универсальные файлы узла) и [03_contours.md](03_contours.md) (контурно-специфичные файлы). README адаптирует канонические миссии под свой контур/узел в колонке «Миссия».

## 🔵 Где обязателен README и его отличия

**Где обязателен:** значимые папки — контур и узел.

**Где опционален:** служебные (`_inbox/`, `meetings/`) и маленькие папки (1-2 файла). Появляется, когда нужна навигация агенту.

**Иерархия:** локальный README перекрывает корневой. В корне держим только универсальное и не дублируем информацию.

**README vs 01_overview.md:**

| Артефакт | Описывает | Главная роль |
|---|---|---|
| `README.md` | Папку как **контейнер** | Маршрутизация AI-агента |
| `01_overview.md` | Сущность папки **как объект** | Состояние сущности и направление |

Карта связей папки живёт в README, не в `01_overview.md`.

## 🔵 Структура README

Все блоки обслуживают маршрутизацию — каждый отвечает на конкретный вопрос агента.

```markdown
---
YAML
---

# {имя-папки} — {короткий ярлык с указанием роли}

{Аннотация под H1}

## Содержимое папки
| Файл / папка | Миссия | Когда читать |
|---|---|---|

## Маршруты чтения (опц.)
| Триггер задачи | Что читать |
|---|---|

## Связанные контексты
— другие папки, домены или стабильные файлы, которые нужны для правильного чтения этой папки и перехода к смежной информации. Связь определяется смыслом и задачей, а не расположением в файловом дереве.
- {связанная папка, домен или стабильный файл} — что там и зачем переходить
```

### YAML
По канону [02_file-spec § YAML](02_file-spec.md).

### Аннотация под H1
**Рабочая миссия папки живёт здесь, не в H1.** H1 несёт короткий ярлык с указанием роли, аннотация — полную миссию. До 25–30 слов, без заголовка секции. Содержательно — [02_folder-spec § Миссия папки](02_folder-spec.md). Формат — [02_file-spec § Аннотация](02_file-spec.md).

```markdown
# 04_company — управление структурой компании

Контур управления структурой компании: подразделения, направления, встречи, задачи, риски, решения и прогресс управленческих узлов. Место, где CEO руководит организацией, как структурой
```

### `## Содержимое папки`
Таблица содержимого папки. Колонка «Миссия» = одна строка зачем существует каждый файл. Миссии универсальных файлов узла — в [03_node-files.md](03_node-files.md), миссии контурно-специфичных файлов — в [03_contours.md](03_contours.md).

### `## Маршруты чтения` (опц.)
Таблица «триггер задачи → что читать». Появляется, когда у папки есть устойчивые типовые запросы агента.

### `## Связанные контексты`
Другие папки, домены или стабильные файлы, которые нужны для правильного чтения этой папки и перехода к смежной информации. В обычных файлах связи живут inline; в README это обязательный отдельный блок.

## 🔵 Доставка README агенту

**Проблема.** README — warm on-demand: подтягивается агенту при заходе в папку, но не гарантированно в момент работы. У каждого агента свой нативный файл инструкций (`CLAUDE.md`, `AGENTS.md`, `GEMINI.md`). Нужна схема, работающая со всеми без дублирования.

**Решение — триплет в значимой папке:**

| Файл | Содержание | Зачем |
|---|---|---|
| `README.md` | Единственный источник: миссия папки, карта, связи | Source of truth |
| `AGENTS.md` | `@README.md` (pointer-only) | Стандарт индустрии: Codex, Gemini CLI, Cursor читают нативно |
| `CLAUDE.md` | `@README.md` (pointer-only) | Временный костыль: Claude Code пока не поддерживает AGENTS.md нативно |

**Принципы:**
- `AGENTS.md` и `CLAUDE.md` — **строго pointer-only**. Локальные правила запрещены — иначе появится вторая правда.
- Триплет ставится **только в значимых папках** (контуры, узлы, методологии, домены), т.к. подтягивается агенту в контекст.

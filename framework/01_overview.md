## Цели
Источник - [02_goal](../meta/management/02_goal.md)

### на экватор
*неделя 50 ~сентябрь 2026*
- Онтология, методология, scaffold стабильны — можно дать клиенту без переделки под каждого. 
- Контекстная архитектура работает как хорошая память агента — AI клиента навигирует по scaffold
- Работают помощники, боты и автоматизации (ритуалы) - которые являют  настоящего помощника для CEO - анализируют встречи, помогают со стратегией, помогают с планом дня и т.д.

### на текущую стадию (декаду)
*Первая декада (недели 18→30, до ~22 мая 2026)* 
- Рабочий scaffold: ядро (обязательные файлы), расширения (опциональные), структура каждого файла, закрывает основные запросы клиентов. Markdown / Google Docs — под клиента
- Контекстная архитектура клиента: протокол чтения, frontmatter, связи между файлами — реализация слоя «Память» из product_vision (на основе нашей — см. лаб)
- Помощники: обработка встреч, подготовка к встречам, трекинг задач и др. Запросы клиентов — источник приоритетов. 3-5 помощников
- Телеграм-бот для клиента — первый прототип
- Клиентские решения — добавляются в продукт
- Оформление в единый плагин — к концу декады если набирается масса, или сдвигается во вторую

## Roadmap декады (18→30, до ~22 мая 2026)

Три фокуса для первой декады (детали — в [../meta/management/02_goal.md](../meta/management/02_goal.md), секция «Первая декада / Продукт»):

1. **Онбординг за 1-2 сессии** — нет скилла
2. **Один помощник end-to-end** (встреча → протокол → файлы) — пайплайн в beta (entity-only), behavioral не подключён
3. **Плагин как единица** (показать, установить, работает) — скелет

**Текущий фокус по фреймворку:** meeting analysis — стабилизация пайплайна через подключение behavioral extractor и pre-processing транскрибации. Остальные домены скиллов заморожены до стабилизации meeting-analysis.

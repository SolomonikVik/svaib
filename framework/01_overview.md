# Framework — состояние реализации

## Кратко

Где мы сейчас в стройке продукта: в какой стадии каждая из шести частей, что зафиксировано в roadmap текущей декады, как устроена модель поставки. Тактический контур — обновляется каждую декаду (~10 недель) и при значимом сдвиге в реализации одной из частей.

**Работа с файлом:** reference — обновлять при смене декадных целей или значимом сдвиге в реализации части продукта. Не обновлять при ежедневной работе. Что сюда: стадия частей, roadmap декады, модель поставки. Что НЕ сюда: что за продукт и зачем (→ [product.md](product.md)), как устроен внутри (→ [architecture.md](architecture.md)), текущие задачи и Session Handoff (→ [02_active.md](02_active.md)), задачи на будущее (→ [02_backlog.md](02_backlog.md)), история (→ [03_progress.md](03_progress.md)).

## Стадия

Первая декада (недели 18→30, до ~22 мая 2026). Проекту ~2 месяца. Один человек, параллельно: клиенты + лаб + сайт + контент.

**Рамка готового продукта:** онтология десятков доменов + методология + scaffold + ~20 skills + 3-4 hooks + 1+ agent + onboarding skill + pipeline ритуалов.

**Бенчмарк:** Ars Contexta (26 skills), COG (17 skills).

## Roadmap декады (18→30, до ~22 мая 2026)

Три фокуса для первой декады (детали — в [../meta/management/02_goal.md](../meta/management/02_goal.md), секция «Первая декада / Продукт»):

1. **Онбординг за 1-2 сессии** — нет скилла
2. **Один помощник end-to-end** (встреча → протокол → файлы) — пайплайн в beta (entity-only), behavioral не подключён
3. **Плагин как единица** (показать, установить, работает) — скелет

**Текущий фокус по фреймворку:** meeting analysis — стабилизация пайплайна через подключение behavioral extractor и pre-processing транскрибации. Остальные домены скиллов заморожены до стабилизации meeting-analysis.

## Модель поставки

**Данные у клиента** (развёрнутый scaffold). **Методология у нас** (обновляемый plugin).

Плагин не хранит клиентские данные — это интеллектуальный слой поверх. Обновляется через Git. Работает поверх внешнего runtime / host (Claude Code, Cowork, nanobot и т.п.), который умеет доставить этот слой клиенту.

Граница plugin vs runtime зафиксирована в [04_decisions.md](04_decisions.md) (пункт 1 — Выбор runtime).

# Framework — обзор направления

## Кратко

Framework — ядро продукта Second AI Brain. Шесть частей: онтология, память, методология, scaffold, skills, plugin. Направление производит канон, по которому разворачивается персональная AI-инфраструктура руководителя: структуру данных, протоколы работы агента, шаблоны для клиента, пакет автоматизации. Стадия — зачаточная (проекту ~2 месяца, первая декада недель 18→30).

**Работа с файлом:** reference — обновлять при смене стратегии направления, целей декады или стадии. Не обновлять при ежедневной работе (текущие задачи → `02_active.md`, сделанное → `03_progress.md`).

## Миссия

Framework производит **канон продукта**: как устроены данные, как агент работает с памятью, какими методами получаются результаты, из каких шаблонов разворачивается пространство клиента, какие скиллы автоматизируют работу, как это собрано в устанавливаемый плагин.

**Входит в scope:** онтология, memory, методология, scaffold, skills, plugin, architecture. Ритуалы работы с фреймворком.
**НЕ входит:** код сайта и инфраструктура (→ `dev/`), клиентские пространства (→ `clients/`), стратегия проекта (→ `meta/`), мастерская AI-помощников для внутренней работы (→ `lab/`).

---

## Рамка

Готовый Second AI Brain ≈ онтология десятков доменов + методология + scaffold + ~20 skills + 3-4 hooks + 1+ agent + onboarding skill + pipeline ритуалов.

**Бенчмарк:** Ars Contexta (26 skills), COG (17 skills).

**Стадия:** зачаточная. Проекту ~2 месяца. Один человек, параллельно клиенты + лаб + сайт + контент.

---

## Цель декады (недели 18→30, до ~22 мая)

1. Онбординг за 1-2 сессии — нет скилла
2. Один помощник end-to-end (встреча → протокол → файлы) — пайплайн в beta (entity-only), behavioral не подключён
3. Плагин как единица (показать, установить, работает) — скелет

Детали → [../meta/management/02_goal.md](../meta/management/02_goal.md).

## Шесть частей фреймворка

Полный обзор со ссылками — в [README.md](README.md). Кратко: что каждая часть производит и её текущая стадия.

| Часть | Что производит | Стадия |
|-------|----------------|--------|
| **ontology/** | Каталог сущностей + правила размещения | Зафиксирована вокруг `01_ceo/`, `02_strategy/`, `03_team/`, `04_management/` |
| **memory/** | Протокол работы агента с информацией (фильтры, хуки, Rule Injection, температура файла) | Новый слой (апрель 2026). Канон: `01_context_memory.md`. Конвенции в работу |
| **methodology/** | Протоколы, decision frames, meeting pipeline, онбординг | Meeting analysis — beta, entity-only. Behavioral не подключён |
| **scaffold/** | Готовый каркас папок и документов для клиента | Канон: `MODEL.md` + `principles.md`. 4 обязательные + 5 опциональных папок. Обкатан на первом клиенте |
| **skills/** | Мастерская промптов и навыков по доменам | meeting-analysis — 9 файлов, beta. Остальные домены — 0 |
| **plugin/** | Собранный пакет для клиента (skills + agents + hooks) | Скелет: 4 README, ноль реализации |

**Ритуалы:** 6 описаны, pipeline не оформлен.

## Архитектура

v8, зафиксирована. Слои: мета-рамка (методология + онтология) → данные → память → помощники → универсальность → skills/plugin/runtime.

Детали: [architecture.md](architecture.md).

## Модель поставки

Данные у клиента (развёрнутый scaffold). Методология у нас (обновляемый plugin). Плагин не хранит данные — это интеллектуальный слой поверх.

# Framework

## Кратко

Ядро продукта Second AI Brain: онтология, методология и готовый плагин. Всё, чтобы развернуть персональную AI-инфраструктуру для руководителя.

**Четыре части:**
- **Ontology** — какие сущности существуют и как связаны (справочник)
- **Methodology** — принципы и модели мышления (почему так устроено)
- **Scaffold** — готовый каркас папок и документов (открыл — скопировал)
- **Plugin** — деплоимый пакет: skills + agents + hooks (то, что клиент устанавливает)

## Связанные файлы

- architecture.md — архитектура продукта (слои, компоненты, связи)
- ../meta/product/product_vision.md — видение продукта Second AI Brain
- ../meta/product/model.md — модель продукта для презентации клиентам
- ../clients/_playbook/onboarding.md — процесс онбординга клиента

---

## Почему "framework"

**Framework** задаёт пространство: какие сущности бывают, как связаны, по каким правилам живут. Внутри фреймворка работают конкретные playbook'и и skills.

Аналогия из программирования: framework — "класс" (архитектура + правила), клиентский проект — "экземпляр" (конкретные данные внутри той же структуры). Скилл `init-brain` разворачивает scaffold так же, как `rails new` создаёт проект на базе фреймворка.

## Как части связаны

```
ontology/          → описывает сущности
                        ↓
methodology/       → описывает как с ними работать
                        ↓
scaffold/          → воплощает в готовую структуру
                        ↓
plugin/skills/     → автоматизирует работу со структурой
plugin/agents/     → работает с данными внутри структуры
```

Онтология отвечает "что существует". Методология — "как с этим работать". Scaffold — "как это выглядит". Plugin — "как это работает само".

---

## Масштабирование

**Соло / малый бизнес:** ядро (8 файлов) + мета (README, SKILL.md). Опциональные файлы — по мере роста. Один человек, LLM помогает.

**CEO с командой (до 100-200 чел):** та же структура, но для личного пространства CEO. Подпроекты для направлений, team разворачивается в папку, metrics расширяется.

Принцип: CEO не ведёт больше файлов, чем соло-предприниматель. Разница — в глубине наполнения, не в количестве.

## Связь с моделью подписки

```
КЛИЕНТ                              SVAIB (плагин)
──────                              ──────────────
• Папка с документами               • skills/ (методология)
  (развёрнутый scaffold)            • agents/ (автоматизация)
• Встречи, контекст                 • hooks/ (триггеры)
• Любой AI-интерфейс                • Обновления через Git
  (Claude, Cowork, VS Code...)
```

Плагин не хранит данные клиента. Данные — у клиента (в scaffold-структуре). Плагин — интеллектуальный слой поверх.

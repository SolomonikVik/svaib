# Оркестратор: полная обработка встречи

Процедура для координатора (основной чат). Руководитель говорит "была встреча, обработай" — координатор проводит полный цикл: выжимка → обновление scaffold → Telegram-сводка.

## Scope

Встречи подразделений/направлений и кросс-функциональные встречи любого уровня внутри 04_management/ (включая встречи верхушки с target = `04_management/meetings/`). Целевая папка выбирается по правилу наименьшей общей родительской папки. Группировка обрабатывается как обычная единица — у неё свои файлы 01–04 на её уровне.

Вне scope этой версии: стратегические встречи, меняющие 02_strategy/ (vision, goal, plan, weekly_progress) — требуют отдельного чеклиста. Поведенческий экстрактор (behavioral) — выжимка строится только из entity extractor, каноническая сборка (entity + behavioral → конкатенация) будет добавлена позже.

## Когда использовать

Руководитель просит обработать встречу / говорит что транскрипт готов / "сделай всё что нужно".

## Критические правила

1. **НЕ читай транскрипт сам.** Передай путь субагенту. Транскрипт большой и забьёт контекст.
2. **Выжимку — ЧИТАЙ.** После L1 прочитай выжимку — она ~100 строк, нужна для всех последующих шагов.
3. **Субагенты — foreground.** НЕ используй `run_in_background: true`.

---

## Шаг 0: Собери контекст

Спроси руководителя:
- Что за встреча? По какому направлению?
- Кто участвовал?
- Какая цель встречи? (если не сказали — записать "неизвестно")
- Где транскрипт? (путь к файлу)

По ответу определи:
- Путь к транскрипту
- Тип встречи (1-on-1, планёрка, стратсессия, клиентская и т.д.)
- Целевую папку `{target_dir}` — по правилу наименьшей общей родительской папки, охватывающей всех участников:
  - Встреча внутри одного подразделения/направления → `04_management/{org_node}/` (или `04_management/{org_node}/{sub_node}/`)
  - Встреча дочерних узлов одной группировки → папка группировки. Пример: маркетинг и продажи внутри `commercial/` → `04_management/commercial/`
  - Встреча верхушки разных направлений → корень `04_management/`. Пример: стратсессия всех руководителей направлений → `04_management/`

### Контекст транскрипта (обязательно)

Собери три секции для L1:

1. **Люди** — прочитай `03_team/org_structure.md` + секции "Люди" и "Люди за пределами оргструктуры" из `04_management/glossary.md` (варианты имён, ошибки транскрибатора)
2. **Глоссарий** — прочитай секцию "Термины и сокращения" из `04_management/glossary.md`
3. **Тип/цель встречи** — из ответа руководителя. Если цель не озвучена — "неизвестно"

### Контекст проекта

Минимум определяется типом встречи:

| Тип встречи | Подгрузить как минимум |
|-------------|----------------------|
| По подразделению/направлению (планёрка, статус) | `04_management/{org_node}/01_overview.md` |
| 1-on-1 с сотрудником | `03_team/{person}.md` (⚠️ сначала `ls 03_team/` — не угадывай имя файла) |
| Клиентская встреча | `clients/{client}/project.md` |
| Стратсессия | `02_strategy/01_vision.md` + `02_strategy/02_goal.md` |

Если из контекста диалога видно, что полезно добавить ещё что-то — добавить. Если нет — минимума достаточно.

## Шаг 1: Выжимка (L1)

Запусти субагент через Agent tool:

```
Прочитай промпт из файла framework/skills/meeting-analysis/L1-prompt-entity-extractor.md
(секция "Промпт", содержимое внутри блока ```).

Подай контекстный пакет (собранный координатором на шаге 0):

## Контекст транскрипта
### Люди
{содержимое org_structure + glossary/люди}

### Глоссарий
{содержимое glossary/термины}

### Тип/цель встречи
{тип, цель, направление — или "неизвестно"}

## Контекст проекта
{overview, досье, карточка клиента — что собрано на шаге 0}

Прочитай транскрипт: {path_to_transcript}

Примени промпт к транскрипту.

Результат ЗАПИШИ в файл: {target_dir}/meetings/{date}_{topic}_summary.md

Use only simple, single bash commands - no &&, no ||, no pipes, no ;, no redirects.
```

После завершения субагента — прочитай выжимку.

Покажи руководителю:
- Где сохранена: ссылка на файл `{target_dir}/meetings/{date}_{topic}_summary.md`
- Содержание встречи: из секции «Содержание встречи» выжимки
- Спроси: "Выжимка готова — посмотри, всё ли на месте. Двигаемся дальше?"

**Не переходи к шагу 2 без подтверждения руководителя.**

**Если руководитель поправил имена, термины или ошибки транскрибатора** при вычитке выжимки — запомни эти правки. Они пойдут в L2 как кандидаты для обновления `04_management/glossary.md`.

## Шаг 2: Обновление файлов проекта (L2)

Выполни процедуру из L2-procedure-scaffold-update.md, начиная с Шага 2 (контекст от руководителя уже есть, выжимка готова). На этом шаге L2-procedure дочитает недостающие файлы (02_active, 03_progress, 04_decisions целевой папки + профайлы всех участников из 03_team/) — не пропускай их как «уже прочитано», минимум Шага 0 покрывал только L1.

1. Прочитай файлы затронутых подразделений/направлений (04_management/{org_node}/) + 03_team/ участников
2. Файл-центричный проход по чеклисту
3. Покажи обновления руководителю
4. Примени после ОК

## Шаг 3: Telegram-сводка (L2)

После завершения обновлений, предложи: "Сделать Telegram-сводку для отправки участникам?"

Если да — запусти субагент:

```
Прочитай промпт из файла framework/skills/meeting-analysis/L2-prompt-protocol-telegram.md
(секция "Промпт", содержимое внутри блока ```).

Прочитай выжимку: {target_dir}/meetings/{date}_{topic}_summary.md

Примени промпт к выжимке.

Результат выведи текстом (НЕ записывай в файл — координатор покажет руководителю для отправки).

Use only simple, single bash commands - no &&, no ||, no pipes, no ;, no redirects.
```

Когда сводка готова, спроси руководителя: **"Показать в чате или отправить в Telegram?"**

### Вариант A: Показать в чате (по умолчанию)

Выведи сводку в code block (```), чтобы руководитель мог скопировать одним действием.

### Вариант B: Отправить в Telegram

Найди `send_telegram.sh` в skills/ рабочего пространства и выполни: `./path/to/send_telegram.sh "ТЕКСТ СВОДКИ"`. Проверь `"ok":true` в ответе. Если скрипт или `.env` не найдены — переключись на вариант A.

> EXPERIMENTAL — требует настроенного Telegram-бота и `.env` в корне рабочего пространства. См. [telegram-delivery.md](telegram-delivery.md).

---

## Шаг 4: Финализация

1. Убедись что выжимка сохранена: `{target_dir}/meetings/{date}_{topic}_summary.md`
2. Перемести транскрипт в archive: `{target_dir}/meetings/archive/{date}_{topic}_transcript.md`

Конвенция именования: выжимка → `YYYY-MM-DD_{topic}_summary.md`, архивный транскрипт → `YYYY-MM-DD_{topic}_transcript.md`. Topic в snake_case, английский.

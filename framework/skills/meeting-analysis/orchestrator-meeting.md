# Оркестратор: полная обработка встречи

Процедура для координатора (основной чат). Руководитель говорит "была встреча, обработай" — координатор проводит полный цикл: выжимка → обновление scaffold → Telegram-сводка.

## Scope

Встречи по одному направлению или кросс-функциональные (2+ направления). Предполагает развёрнутый team/.

Вне scope этой версии: company-wide встречи (management-слой), контейнерная логика, поведенческий экстрактор (behavioral). Выжимка строится только из entity extractor — каноническая сборка (entity + behavioral → конкатенация) будет добавлена позже.

## Когда использовать

Руководитель просит обработать встречу / говорит что транскрипт готов / "сделай всё что нужно".

## Критические правила

1. **НЕ читай транскрипт сам.** Передай путь субагенту. Транскрипт большой и забьёт контекст.
2. **Выжимку — ЧИТАЙ.** После L1 прочитай выжимку — она ~100 строк, нужна для всех последующих шагов.
3. **Субагенты — foreground.** НЕ используй `run_in_background: true`.

---

## Шаг 0: Собери контекст

Спроси руководителя:
- Что за встреча? По какому направлению?
- Кто участвовал?
- Какая цель встречи? (если не сказали — записать "неизвестно")
- Где транскрипт? (путь к файлу)

По ответу определи:
- Путь к транскрипту
- Целевую папку направления: `{direction}/` (или `{direction}/{sub_direction}/`)
- Тип встречи (1-on-1, планёрка, стратсессия, клиентская и т.д.)

### Контекст транскрипта (обязательно)

Собери три секции для L1:

1. **Люди** — прочитай `team/org_structure.md` + секции "Люди" и "Люди за пределами оргструктуры" из `management/glossary.md` (варианты имён, ошибки транскрибатора)
2. **Глоссарий** — прочитай секцию "Термины и сокращения" из `management/glossary.md`
3. **Тип/цель встречи** — из ответа руководителя. Если цель не озвучена — "неизвестно"

### Контекст проекта

Минимум определяется типом встречи:

| Тип встречи | Подгрузить как минимум |
|-------------|----------------------|
| По направлению (планёрка, статус) | `{direction}/01_overview.md` |
| 1-on-1 с сотрудником | `team/{person}.md` (⚠️ сначала `ls team/` — не угадывай имя файла) |
| Клиентская встреча | `clients/{client}/project.md` |
| Стратсессия | `management/01_vision.md` + `02_goal.md` |

Если из контекста диалога видно, что полезно добавить ещё что-то — добавить. Если нет — минимума достаточно.

## Шаг 1: Выжимка (L1)

Запусти субагент через Agent tool:

```
Прочитай промпт из файла framework/skills/meeting-analysis/L1-prompt-entity-extractor.md
(секция "Промпт", содержимое внутри блока ```).

Подай контекстный пакет (собранный координатором на шаге 0):

## Контекст транскрипта
### Люди
{содержимое org_structure + glossary/люди}

### Глоссарий
{содержимое glossary/термины}

### Тип/цель встречи
{тип, цель, направление — или "неизвестно"}

## Контекст проекта
{overview, досье, карточка клиента — что собрано на шаге 0}

Прочитай транскрипт: {path_to_transcript}

Примени промпт к транскрипту.

Результат ЗАПИШИ в файл: {target_dir}/meetings/{date}_{topic}_summary.md

Use only simple, single bash commands - no &&, no ||, no pipes, no ;, no redirects.
```

После завершения субагента — прочитай выжимку.

Покажи руководителю:
- Где сохранена: ссылка на файл `{target_dir}/meetings/{date}_{topic}_summary.md`
- Содержание встречи: из секции «Содержание встречи» выжимки
- Спроси: "Выжимка готова — посмотри, всё ли на месте. Двигаемся дальше?"

**Не переходи к шагу 2 без подтверждения руководителя.**

## Шаг 2: Обновление файлов проекта (L2)

Выполни процедуру из L2-procedure-scaffold-update.md, начиная с Шага 2 (контекст от руководителя уже есть, выжимка готова):

1. Прочитай scaffold-файлы затронутых направлений + team/ участников
2. Файл-центричный проход по чеклисту
3. Покажи обновления руководителю
4. Примени после ОК

---

## Шаг 3: Telegram-сводка (L2)

После завершения обновлений, предложи: "Сделать Telegram-сводку для отправки участникам?"

Если да — запусти субагент:

```
Прочитай промпт из файла framework/skills/meeting-analysis/L2-prompt-protocol-telegram.md
(секция "Промпт", содержимое внутри блока ```).

Прочитай выжимку: {target_dir}/meetings/{date}_{topic}_summary.md

Примени промпт к выжимке.

Результат выведи текстом (НЕ записывай в файл — координатор покажет руководителю для отправки).

Use only simple, single bash commands - no &&, no ||, no pipes, no ;, no redirects.
```

Покажи результат руководителю. Он скопирует и отправит сам.

---

## Шаг 4: Финализация

1. Убедись что выжимка сохранена: `{target_dir}/meetings/{date}_{topic}_summary.md`
2. Перемести транскрипт в archive: `{target_dir}/meetings/archive/{date}_{topic}_transcript.md`

Конвенция именования: `YYYY-MM-DD_{topic}_{type}.md` — topic в snake_case, английский. Подробнее → file_spec.md.

# Meeting Analysis

Пайплайн превращает транскрипт встречи в структурированные артефакты. Два слоя: L1 (экстракция → выжимка), L2 (выжимка → клиентские форматы).

Методология (зачем так устроено): [meeting_analysis.md](../../methodology/meeting_analysis.md), [meeting_overlays.md](../../methodology/meeting_overlays.md).

---

## Пайплайн

```
                    транскрипт + контекст
                         │
              ┌──────────┴──────────┐
              ▼                     ▼
     Entity Extractor      Behavior Extractor
     (чанкинг если >60KB)  (всегда монолит)
              │                     │
              └──────────┬──────────┘
                         ▼
                    Synthesizer
                    (выжимка)
                         │
              ┌──────────┴──────────┐
              ▼                     ▼
         Протокол              Telegram
         (L2)                  (L2, TODO)
```

## Как запустить

### Что нужно перед стартом

- Транскрипт встречи (текст)
- Контекст: участники (имя → роль), цель встречи, глоссарий (опционально)
- Досье проекта (опционально, только для синтезатора и протокола)

### Шаги

**Шаг 1. Проверь размер транскрипта.**

- ≤60KB → один субагент на entity extractor (монолит)
- &gt;60KB → чанкинг по [L1-procedure-chunking.md](L1-procedure-chunking.md): координатор сам нарезает файл, запускает по субагенту на каждый чанк, мёржит результаты

**Шаг 2. Запусти два экстрактора параллельно.**

| Экстрактор | Вход | Чанкинг |
|------------|------|---------|
| Entity | промпт + контекст + транскрипт | да, если >60KB |
| Behavior | промпт + контекст + транскрипт | нет, всегда монолит |

Каждому субагенту: прочитай промпт (секция "Промпт" внутри ```блока```), подай контекст + транскрипт, верни результат в файл.

**Шаг 3. Запусти синтезатор.**

Вход: выходы обоих экстракторов + досье (если есть). Транскрипт НЕ подавать.

**Шаг 4. Запусти промпт Слоя 2.**

Вход: выжимка (выход синтезатора) + досье (опционально). Транскрипт НЕ подавать.

## Ответственности координатора

Координатор — это ты (главный агент). Субагенты — исполнители одного промпта.

- **Координатор** проверяет размер транскрипта
- **Координатор** решает, нужен ли чанкинг
- **Координатор** сам нарезает транскрипт на чанки
- **Координатор** запускает по субагенту на каждый чанк и мёржит результаты
- **Координатор** передаёт досье только синтезатору и протоколу (не экстракторам)
- **Субагент** получает один промпт + данные, возвращает результат. Не оркестрирует, не запускает других субагентов

## Частые ошибки

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Субагенту поручить чанкинг | Субагент не может надёжно запускать своих субагентов | Координатор чанкит сам |
| Дать досье экстракторам | Загрязняет экстракцию внешним контекстом | Досье → только синтезатору и L2 |
| Дать транскрипт синтезатору | Синтезатор работает по выходам экстракторов, не по сырому тексту | Только выходы экстракторов + досье |
| Дать транскрипт протоколу | Протокол работает по выжимке, не по сырому тексту | Только выжимка + досье |
| Behavior extractor с чанкингом | Precision важнее recall для паттернов | Всегда монолит |
| Пропустить контекст транскрипта | Модель не знает кто говорит, путает имена | Всегда подавать участников + роли |

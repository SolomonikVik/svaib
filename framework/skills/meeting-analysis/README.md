# Meeting Analysis

Пайплайн превращает транскрипт встречи в структурированные артефакты. Два слоя: L1 (экстракция → выжимка), L2 (выжимка → клиентские форматы + обновление файлов проекта).

Методология (зачем так устроено): [meeting_analysis.md](../../methodology/meeting_analysis.md), [meeting_overlays.md](../../methodology/meeting_overlays.md).

---

## Пайплайн

**Рабочая версия** (entity-only, используется оркестратором):

```
транскрипт + контекст
         │
         ▼
  Entity Extractor (extraction + classification в один проход)
         │
         ▼
      Выжимка
         │
    ┌────┼────┐
    ▼    ▼    ▼
 Протокол Telegram Обновление файлов
   (L2)   (L2)    (L2)
```

Behavioral extractor существует в двух вариантах (v1 аналитический, v2 клиентский), но не подключён к пайплайну — открытый вопрос. Целевая архитектура: entity + behavior → конкатенация → полная выжимка. Подробнее → [meeting_analysis.md](../../methodology/meeting_analysis.md).

## Как запустить

**Быстрый старт:** используй [orchestrator-meeting.md](orchestrator-meeting.md) — он проведёт через весь цикл: выжимка → обновление файлов → Telegram.

### Что нужно перед стартом (ручной запуск)

- Транскрипт встречи (текст)
- Контекст транскрипта (обязательно): люди (org_structure + glossary), глоссарий (ASR-ошибки, жаргон), тип/цель встречи
- Досье проекта (опционально, подаётся экстрактору сущностей как фон и протоколу)

### Шаги (рабочая версия, entity-only)

**Шаг 1. Запусти entity extractor.**

| Экстрактор | Промпт | Вход |
|------------|--------|------|
| Entity | L1-prompt-entity-extractor.md | контекст транскрипта + транскрипт + досье (как фон, опционально) |

Субагенту: прочитай промпт (секция "Промпт" внутри ```блока```), подай вход, верни результат в файл.

> Когда behavioral extractor будет подключён к пайплайну, здесь добавится параллельный шаг + конкатенация выходов.

**Шаг 2. Запусти промпт Слоя 2.**

- **Протоколы** (full, Telegram): вход — выжимка + досье (опционально). Транскрипт НЕ подавать
- **Обновление файлов проекта** (scaffold-update): вход — выжимка + контекст от руководителя (направление, участники) + файлы проекта (direction/, team/)

## Ответственности координатора

Координатор — это ты (главный агент). Субагенты — исполнители одного промпта.

- **Координатор** запускает entity extractor (субагент)
- **Координатор** передаёт досье экстрактору сущностей (как фон) и L2-промптам
- **Субагент** получает один промпт + данные, возвращает результат. Не оркестрирует, не запускает других субагентов

## Частые ошибки

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Дать транскрипт протоколу | Протокол работает по выжимке, не по сырому тексту | Только выжимка + досье |
| Пропустить контекст транскрипта | Модель не знает кто говорит, путает имена | Всегда подавать контекст транскрипта (люди, глоссарий, тип/цель) |

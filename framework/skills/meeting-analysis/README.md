# Meeting Analysis

Пайплайн превращает транскрипт встречи в структурированные артефакты. Два слоя: L1 (экстракция → выжимка), L2 (выжимка → клиентские форматы).

Методология (зачем так устроено): [meeting_analysis.md](../../methodology/meeting_analysis.md), [meeting_overlays.md](../../methodology/meeting_overlays.md).

## Как запустить

### Что нужно перед стартом

- Транскрипт встречи (текст)
- Контекст: участники (имя → роль), цель встречи, глоссарий (опционально)
- Досье проекта (опционально, подаётся экстрактору сущностей как фон и протоколу)

### Шаги

**Шаг 1. Запусти два экстрактора параллельно.**

| Экстрактор | Промпт | Вход |
|------------|--------|------|
| Entity | L1-prompt-entity-extractor.md | контекст транскрипта + транскрипт + досье (как фон, опционально) |
| Behavior | L1-prompt-behavior-extractor.md | контекст транскрипта + транскрипт |

Каждому субагенту: прочитай промпт (секция "Промпт" внутри ```блока```), подай вход, верни результат в файл.

**Шаг 2. Собери выжимку.**

Конкатенация: выход entity extractor + выход behavior extractor = выжимка. Отдельного промпта не нужно.

**Шаг 3. Запусти промпт Слоя 2.**

Вход: выжимка + досье (опционально). Транскрипт НЕ подавать.

---

## Ответственности координатора

Координатор — это ты (главный агент). Субагенты — исполнители одного промпта.

- **Координатор** запускает два экстрактора параллельно
- **Координатор** соединяет выходы в выжимку (конкатенация)
- **Координатор** передаёт досье экстрактору сущностей (как фон) и протоколу
- **Субагент** получает один промпт + данные, возвращает результат. Не оркестрирует, не запускает других субагентов

## Частые ошибки

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Дать досье behavior extractor | Загрязняет поведенческий анализ бизнес-контекстом | Досье → только entity extractor (как фон) и L2 |
| Дать транскрипт протоколу | Протокол работает по выжимке, не по сырому тексту | Только выжимка + досье |
| Behavior extractor с чанкингом | Precision важнее recall для паттернов | Всегда монолит |
| Пропустить контекст транскрипта | Модель не знает кто говорит, путает имена | Всегда подавать участников + роли |

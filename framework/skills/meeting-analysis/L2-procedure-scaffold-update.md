# Обновление файлов проекта по выжимке встречи

L2-надстройка над выжимкой встречи. Получает готовую выжимку, делает файл-центричный проход по файлам проекта, предлагает обновления руководителю.

**Слой:** L2 (надстройка поверх выжимки встречи, категория «Обновление файлов проекта»).

## Вход

1. **Выжимка** — готовый файл `_summary.md` (результат L1-пайплайна)
2. **Контекст от руководителя** — ответ на вопрос: "Что за встреча? По какому направлению? Кто участвовал?"

## Выход

Список обновлений по файлам. Каждое обновление: какой файл → что добавить/изменить → почему (ссылка на выжимку). Применяется ТОЛЬКО после ОК руководителя.

## Scope

Встречи подразделений/направлений и кросс-функциональные встречи любого уровня внутри 04_management/ (включая встречи верхушки с target = `04_management/`). Целевая папка выбирается по правилу наименьшей общей родительской папки. Группировка обрабатывается как обычная единица — у неё свои файлы 01–04 на её уровне.

Вне scope этой версии: стратегические встречи, меняющие 02_strategy/ (vision, goal, plan, weekly_progress) — требуют отдельного чеклиста по strategy-файлам.

---

## Шаг 1: Определи scope обновления

По ответу руководителя определи затронутые подразделения/направления и выбери целевую папку по правилу наименьшей общей родительской папки (папка, охватывающая всех участников).

**Встреча внутри одного подразделения/направления** (1-on-1, планёрка отдела):
- Целевая папка: `04_management/{org_node}/` (или `04_management/{org_node}/{sub_node}/`)
- Читать: файлы целевой папки + 03_team/ участников

**Кросс-функциональная встреча дочерних узлов одной группировки** (например, маркетинг + продажи внутри `commercial/`):
- Целевая папка: папка группировки, `04_management/{parent}/` (`04_management/commercial/`)
- Читать: файлы папки группировки + файлы КАЖДОГО затронутого дочернего узла + 03_team/ участников
- Обновлять: файлы дочерних узлов И файлы родительского уровня (01_overview, 02_active, 03_progress, 04_decisions группировки), если кросс-функциональный контекст изменил общую картину

**Встреча верхушки разных направлений** (стратсессия руководителей):
- Целевая папка: корень `04_management/` (target = `04_management/meetings/`)
- Читать: файлы КАЖДОГО затронутого подразделения/направления + 03_team/ участников + `02_strategy/06_decisions.md`
- Обновлять: файлы затронутых единиц + при необходимости `02_strategy/06_decisions.md` (решения уровня компании)

---

## Шаг 2: Собери контекст

1. Прочитай выжимку
2. Прочитай файлы целевой папки (папок): 01_overview.md, 02_active.md, 03_progress.md, 04_decisions.md
3. Прочитай `03_team/{person}.md` для каждого участника встречи
4. Прочитай `04_management/glossary.md`
5. При кросс-функциональной встрече — прочитай `02_strategy/06_decisions.md`

---

## Шаг 3: Файл-центричный проход

### Iron Law обновления

После обновления — состояние проекта в файлах актуально. Выжимка остаётся как полная запись встречи. Не всё из выжимки должно попасть в файлы — это нормально.

### Принцип: выжимка щедрая, файлы проекта строгие

Выжимка сохраняет всё (Iron Law L1). Файлы проекта — только то, что имеет устойчивую ценность: стратегические решения, факты о бизнесе, задачи с owner, изменения в целях и структуре. Операционная мелочь, разовые реплики, бытовые просьбы — остаются в выжимке, не попадают в файлы.

---

Пройди по каждому прочитанному файлу и спроси по его миссии: "нужно ли обновить этот файл с учётом новой информации из выжимки?"

**НЕ раскидывай сущности из выжимки по файлам.** Проверяй файлы — это принципиально другой алгоритм.

### Чеклист

| Файл | Миссия | Тест-вопрос |
|------|--------|-------------|
| 01_overview.md | ЧТО это направление и КУДА идёт | Новое о целях, KPI, scope, команде, ключевых проектах? Тест: будет актуально через квартал? |
| 02_active.md | ЧТО АКТУАЛЬНО сейчас | Сначала сопоставь новое с existing — связанные объединяй, не дублируй. Новые задачи (по owner)? Проблемы? Открытые вопросы? Риски? Идеи? Повестка следующей встречи? Выполненные задачи → отметить `[x]`, перенести в "Подтвердить" |
| 03_progress.md | КАК ДВИГАЛИСЬ | Дописать запись о встрече (формат ниже) |
| 04_management/{org_node}/04_decisions.md | ПОЧЕМУ так решили (уровень подразделения/направления) | Стратегические решения внутри единицы? Тест: через 3 мес спросят "почему так?" |
| 04_management/{parent}/04_decisions.md | ПОЧЕМУ так решили (уровень группировки) | Только для кросс-функциональных встреч. Решения, охватывающие несколько дочерних единиц группировки, но не меняющие курс компании? Тест: затрагивает 2+ дочерних узла группировки, но не vision/goal/plan компании |
| 02_strategy/06_decisions.md | ПОЧЕМУ так решили (курс компании) | Решения уровня всей компании (pivot, стратегия, структура, принципы найма)? Только при стратсессиях или кросс-функциональных встречах |
| 01_ceo/04_decisions.md | ПОЧЕМУ так решил CEO (личный уровень) | Личные решения руководителя как человека в роли (принципы работы, границы, как принимает решения)? Тест: про CEO как личность, не про бизнес-решение |
| 03_team/{person}.md | КТО этот человек | Наблюдения о стиле, сильных сторонах, зонах развития? Коучинг? Тест: про ЧЕЛОВЕКА или про ПОДРАЗДЕЛЕНИЕ/НАПРАВЛЕНИЕ? |
| 04_management/glossary.md | КАК ЧИТАТЬ транскрипты | Новые имена, варианты произношения, ошибки транскрибатора, незнакомые термины/жаргон? Если руководитель поправил имя/термин при вычитке — кандидат сюда |

При кросс-функциональных встречах — проход по чеклисту для КАЖДОГО затронутого подразделения/направления. Одна сущность из выжимки может породить обновления в файлах разных единиц.

---

## Шаг 4: Покажи обновления руководителю

Формат предложения:

```
### {имя файла} ({путь})

**Обновление:**
- {конкретная правка}

**Основание:** {ссылка на пункт выжимки}
```

Применяй ТОЛЬКО после ОК руководителя. Если руководитель отклоняет обновление — не спорь, оставь в выжимке (catch-all).

---

## Правила маршрутизации

**Один аспект — одно место.** Одно событие может обновить несколько файлов, если затрагивает разные аспекты. "Решили сменить лидера направления" → 01_overview (структура) + `03_team/{person}.md` (роль) + 02_active (задачи по передаче). Разные файлы отвечают на разные вопросы — это не дублирование.

**Catch-all.** Не можешь однозначно маршрутизировать — оставь в выжимке. Лучше не разложить, чем разложить неправильно.

**Антипаттерны:**
- Проблемы/задачи в overview → active
- Хроника в overview → progress
- Решения без контекста → decisions (нужны why + альтернативы)
- Коучинг в `04_management/{org_node}/` → `03_team/{person}.md`
- Операционные выборы в decisions → остаются в выжимке
- Company-wide решения в `04_management/{org_node}/04_decisions.md` → `02_strategy/06_decisions.md`
- Личные решения CEO (принципы, стиль) в `02_strategy/06_decisions.md` → `01_ceo/04_decisions.md`

**Даты-источники.** Volatile-данные (цифры, статусы, решения) — inline-дата `(YYYY-MM-DD)`. Стабильные факты — без дат.

---

## Форматы записей в log-файлах

**03_progress.md:**
```
## YYYY-MM-DD: {тема встречи}

**Участники:** {кто был}
**Ключевое:** {2-4 строки — что обсудили, что решили, что сдвинулось}
**Следующий шаг:** {главное действие}
```

**04_decisions.md / 02_strategy/06_decisions.md / 01_ceo/04_decisions.md:**
```
## YYYY-MM-DD: Название решения
**Статус:** Актуально
**Решение:** Что решили
**Почему:** Контекст
**Альтернатива:** Что рассматривали и отвергли
```

**03_team/{person}.md → Коучинг-лог:**
```
### YYYY-MM-DD
{Наблюдение: что заметили, как проявилось, контекст}
```

## Ограничения DRAFT v6

1. **Встречи уровня 02_strategy/ вне scope.** Стратегические встречи, меняющие vision/goal/plan/weekly_progress, требуют отдельного чеклиста по strategy-файлам. Кросс-функциональные встречи любого уровня внутри 04_management/ (включая верхушку) — в scope через правило общей родительской папки.
2. **Нет автоопределения подразделения/направления.** Руководитель указывает единицу и участников.

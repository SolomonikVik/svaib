# Оркестратор: обработка встречи с контрагентом

Процедура для координатора (основной чат). Исполнитель говорит "была встреча с клиентом, обработай" -- координатор проводит полный цикл: выжимка -> обновление досье -> Telegram-сводка.

## Scope

Встречи с одним контрагентом (клиент, партнёр, подрядчик). Предполагает досье по стандартной структуре клиентского проекта (profile.md, company.md, project.md и т.д.).

Вне scope этой версии: внутренние встречи (-> orchestrator-meeting.md), встречи с несколькими контрагентами, поведенческий экстрактор (behavioral).

## Когда использовать

Исполнитель просит обработать встречу с контрагентом / говорит что транскрипт готов / "обработай встречу с [имя]".

## Критические правила

1. **НЕ читай транскрипт сам.** Передай путь субагенту. Транскрипт большой и забьёт контекст.
2. **Выжимку -- ЧИТАЙ.** После L1 прочитай выжимку -- она ~100 строк, нужна для всех последующих шагов.
3. **Субагенты -- foreground.** НЕ используй `run_in_background: true`.

---

## Шаг 0: Собери контекст

Спроси исполнителя:
- Что за встреча? С кем? (имя контрагента)
- Кто участвовал?
- Какая цель встречи? (если не сказали -- записать "неизвестно")
- Где транскрипт? (путь к файлу)

По ответу определи:
- `{project_dir}` -- папка проекта контрагента
- Путь к транскрипту
- Тип встречи (кикофф, регулярная, стратсессия и т.д.)

### Контекст транскрипта (обязательно)

Собери три секции для L1:

1. **Люди** -- прочитай `{project_dir}/profile.md` (имя, роль, предпочтения) + `{project_dir}/company.md` (ключевые люди, оргструктура). Если company.md ещё нет -- только profile
2. **Глоссарий** -- термины и сокращения из profile.md и company.md. Если нет выделенного глоссария -- не угадывать, неоднозначное пойдёт в "Требует уточнения"
3. **Тип/цель встречи** -- из ответа исполнителя. Если цель не озвучена -- "неизвестно"

### Контекст проекта

Прочитай `{project_dir}/project.md` (если существует) -- что мы делаем, модель работы, программа. Подаётся L1 как фон.

---

## Шаг 1: Выжимка (L1)

Запусти субагент через Agent tool:

```
Прочитай промпт из файла framework/skills/meeting-analysis/L1-prompt-entity-extractor.md
(секция "Промпт", содержимое внутри блока ```).

Подай контекстный пакет (собранный координатором на шаге 0):

## Контекст транскрипта
### Люди
{содержимое profile + company/люди}

### Глоссарий
{термины, сокращения -- или "глоссарий не собран, неоднозначное в Требует уточнения"}

### Тип/цель встречи
{тип, цель -- или "неизвестно"}

## Контекст проекта
{project.md -- что делаем, модель работы}

Прочитай транскрипт: {path_to_transcript}

Примени промпт к транскрипту.

Результат ЗАПИШИ в файл: {project_dir}/meetings/{date}_{topic}_summary.md

Use only simple, single bash commands - no &&, no ||, no pipes, no ;, no redirects.
```

После завершения субагента -- прочитай выжимку.

Покажи исполнителю:
- Где сохранена: ссылка на файл `{project_dir}/meetings/{date}_{topic}_summary.md`
- Содержание встречи: из секции "Содержание встречи" выжимки
- Спроси: "Выжимка готова -- посмотри, всё ли на месте. Двигаемся дальше?"

**Не переходи к шагу 2 без подтверждения исполнителя.**

**Если исполнитель поправил имена, термины или ошибки транскрибатора** при вычитке выжимки -- запомни эти правки для будущих встреч.

## Шаг 3: Telegram-сводка (L2)

После завершения обновлений, предложи: "Сделать Telegram-сводку для отправки участникам?"

Если да -- запусти субагент:

```
Прочитай промпт из файла framework/skills/meeting-analysis/L2-prompt-protocol-telegram.md
(секция "Промпт", содержимое внутри блока ```).

Прочитай выжимку: {project_dir}/meetings/{date}_{topic}_summary.md

Примени промпт к выжимке.

Результат выведи текстом (НЕ записывай в файл -- координатор покажет исполнителю для отправки).

Use only simple, single bash commands - no &&, no ||, no pipes, no ;, no redirects.
```

Когда сводка готова, спроси исполнителя: **"Показать в чате или отправить в Telegram?"**

### Вариант A: Показать в чате (по умолчанию)

Выведи сводку в code block (```), чтобы исполнитель мог скопировать одним действием.

### Вариант B: Отправить в Telegram

Найди `send_telegram.sh` в skills/ рабочего пространства и выполни: `./path/to/send_telegram.sh "ТЕКСТ СВОДКИ"`. Проверь `"ok":true` в ответе. Если скрипт или `.env` не найдены -- переключись на вариант A.

> EXPERIMENTAL -- требует настроенного Telegram-бота и `.env` в корне рабочего пространства. См. [telegram-delivery.md](telegram-delivery.md).

---

## Шаг 4: Финализация

1. Убедись что выжимка сохранена: `{project_dir}/meetings/{date}_{topic}_summary.md`
2. Перемести транскрипт в archive: `{project_dir}/meetings/archive/{date}_{topic}_transcript.md`

Конвенция именования: `YYYY-MM-DD_{topic}_{type}.md` -- topic в snake_case, английский.

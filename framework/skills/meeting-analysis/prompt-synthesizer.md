# Синтезатор выжимки

Промпт финального шага пайплайна: собирает результаты обоих экстракторов в единую выжимку.

Каталог сущностей для классификации — в [entities.md](../../ontology/entities.md).

## Шаблон выжимки

```markdown
## Метаданные
Дата: ...
Участники: ...
Длительность: ...
Тип встречи: ...

## Краткое содержание
[2-3 предложения: о чём встреча, к чему пришли]

## Извлечённые сущности

### Решения
- [Решение] — опора: "цитата"

### Задачи
[Owner]:
- [ ] [Задача] — срок (если озвучен)
  опора: "цитата"

### Запросы / Потребности
- [Кто]: [что хочет] — опора: "цитата"

### Идеи
- [Идея] — опора: "цитата"

### Инсайты
- [Инсайт] — опора: "цитата"

### Факты
- [Факт] — опора: "цитата"

### Открытые вопросы
- [Вопрос] — опора: "цитата"

### Риски
- [Риск] — опора: "цитата"

## Поведенческие паттерны
- [Наблюдение] — паттерн: [ось +/-], участник: [кто], опора: "цитата"
  Возможная причина (гипотеза): [из behavioral_patterns.md]

## Ключевые цитаты
[Дословные цитаты, значимые целиком — тон, формулировка. Не дублируют цитаты-опоры]

## Требует уточнения
[Буфер: важное, но неоднозначное. Лучше сохранить, чем потерять]
- [Пункт] — [что неясно]
```

## Валидация (черновик чеклиста)

**Автоматическая (встроена в синтезатор):**

- Каждый пункт имеет цитату-опору
- Участники в выводе = участники во входе
- Задачи имеют owner (или "не назначен")
- Сроки — только если озвучены
- Один факт = одно место
- Решение ≠ задача
- Не выдавать обсуждение за консенсус — если нет явной фиксации ("решили", "договорились"), это Идея или Открытый вопрос, не Решение

**Содержательная (при наличии контекста):**

- Contamination check — не подмешано ли из контекста
- Полнота — не потеряно ли из кандидатов экстракторов
- Правильность классификации — верно ли разнесено между типами сущностей
- Over-generalization — не представлен ли спорный вопрос как консенсус

# L2 — Telegram-сводка

Один промпт: выжимка из L1 → готовое Telegram-сообщение для отправки участникам. Action-only: задачи, решения и договорённости, к следующей встрече. НЕ восстанавливает потерянное. Лимит ~4000 символов.

**Канон:** маппинг сущностей → секции, общие правила — в [protocol_format.md](../../ontology/protocol_format.md). Этот промпт — реализация Telegram-формата (action-only).

## Как использовать

Подай на вход:

```
## Выжимка
[полный выход L1]

## Досье (опционально)
[профайл участника, карточка проекта, история встреч — если есть]
```

Транскрипт НЕ подавать.

---

## Промпт

```
Твоя задача — превратить выжимку встречи в Telegram-сообщение: короткое, структурированное, готовое к отправке. Action-only: итог, задачи, решения и договорённости, к следующей встрече. Остальное — в выжимке и полном протоколе.

## Главное правило

**Telegram-сводка = только факты этой встречи.** Всё берётся из выжимки. Досье — только для пометок ("подтвердили: ...", "уточнили: ..."), НЕ для удаления и НЕ как источник содержания. Додумывать то, чего нет в выжимке — ЗАПРЕЩЕНО.

---

## Что делать с выжимкой

Выжимка содержит: метаданные, содержание встречи (нарратив), сущности по типам с цитатами-опорами.

**Убрать** (не для Telegram):
- Цитаты-опоры (служебная информация)
- Поведенческие паттерны (отдельный артефакт, не для чата)

**Упаковать** в четыре секции:

| Тип в выжимке | Секция Telegram |
|---------------|-----------------|
| Задачи | ✅ ЗАДАЧИ |
| Решения | 📌 РЕШЕНИЯ И ДОГОВОРЁННОСТИ (только то, что не дублирует задачи) |
| Цели | 📌 РЕШЕНИЯ И ДОГОВОРЁННОСТИ (если зафиксировано) |
| Задачи/вопросы к след. встрече | 📋 К СЛЕДУЮЩЕЙ ВСТРЕЧЕ |

Всё остальное (идеи, инсайты, факты, метрики, риски, открытые вопросы, справочные, требует уточнения) — НЕ включать в Telegram. Остаётся в выжимке и полном протоколе.

---

## Фильтрация

Telegram = action-only. Включаем только: итог, задачи, решения и договорённости, к следующей встрече. Всё остальное (идеи, инсайты, запросы, факты, метрики, риски, открытые вопросы, справочные, требует уточнения) — в выжимке и полном протоколе.

---

## Досье (если подано)

Используй ТОЛЬКО для пометок:
- Решение/задача совпадает с досье → пометь "подтвердили: [что]"
- Задача изменилась (срок/owner/формулировка) → пометь "уточнили: [что изменилось]"

**НЕ удаляй** пункты на основании досье.

---

## Формат Telegram-сообщения

🛑 **[тип встречи] — [участники] — [дата]**

❗️ **ИТОГ**

[2-3 предложения: что произошло, что прояснилось, что изменилось]

---

✅ **ЗАДАЧИ**

**[Исполнитель]:**
- [ ] [Задача] — [срок]. Если срок не оговорён — "срок не оговорён"

**[Не назначен]:**
- [ ] [Задача] — срок не оговорён

---

📌 **РЕШЕНИЯ И ДОГОВОРЁННОСТИ**

[Только то, что не дублирует задачи]
- [Решение / договорённость 1]
- [Решение / договорённость 2]

---

📋 **К СЛЕДУЮЩЕЙ ВСТРЕЧЕ**

- [Пункт] — [кто отвечает]

Пустые секции не пиши.

---

## Лимит Telegram

- Лимит сообщения: 4000 символов
- **Перед выдачей — посчитай длину готового текста**
- Если ≤4000 — выдай одним сообщением. НЕ разбивай на части "на всякий случай"
- Если >4000 — разбей и помечай: `(1/2)`, `(2/2)` в конце каждого сообщения

**Правила разбиения (только если >4000):**
1. Разбивай по границам секций (не рви секцию пополам)
2. Если одна секция >4000 символов — разрезай внутри по подгруппам исполнителей или по пунктам, сохраняя заголовок секции в каждой части

---

## Стиль

"Решили", "договорились", "упёрлись", "застряли", "разошлись во мнениях", "оставили открытым", "сняли с повестки", "решили не делать", "отложили".

**Запрещено:** "в рамках обсуждения", "было принято решение", "участники пришли к выводу", "осуществить", "данный", "в части".

Коротко и ёмко. Без воды. Без рекомендаций от себя.

---

## Чего не делать

- Не добавляй информацию из досье в сводку
- Не удаляй пункты на основании досье
- Не восстанавливай то, чего нет в выжимке — если L1 потерял, Telegram не спасёт
- Не выдумывай сроки, owner, метрики
- Не лей воду — каждый пункт должен нести конкретный смысл
- Не давай рекомендаций от себя
- Не дублируй между секциями (кроме пары решение+задача)
- Не оставляй пустые секции
- Не включай поведенческие паттерны, идеи, инсайты, запросы, справочные, проблемные зоны — они не для Telegram

---

Выводи ТОЛЬКО готовое Telegram-сообщение (или несколько, если разбито). Без вступлений, комментариев, объяснений.
```

## Примечания

**Источники:**
- Логика маппинга 11 типов → секции: `L2-prompt-protocol-full.md`
- Telegram-форматирование (эмодзи, разделители, лимит): из раннего черновика (удалён при cleanup 2026-03-30)
- Секция "К следующей встрече": `L2-prompt-protocol-full.md`

**Отличия от L2-prompt-protocol-full.md:**
- Вход тот же (выжимка + досье), но выход — Telegram-формат с эмодзи
- Action-only фильтрация: только итог, задачи, решения и договорённости, к следующей встрече
- Поведенческие паттерны исключены (не для мессенджера)
- Лимит ~3800 символов на сообщение, разбивка на части

**Архитектурное решение:**
Telegram-сводка — отдельный L2-промпт, не параметр L2-protocol. Каждый формат протокола (полный, Telegram, email) — самостоятельный промпт со своей логикой фильтрации. Подробнее: `methodology/meeting_overlays.md`.

**Что не включено намеренно:**
- FRAME-скоринг — отложен
- Behavioral паттерны — вне формата Telegram
- Few-shot примеры — после тестирования

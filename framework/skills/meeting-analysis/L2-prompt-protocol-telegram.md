# L2 — Telegram-сводка

Один промпт: выжимка из L1 → готовое Telegram-сообщение для отправки участникам. Упаковывает, фильтрует и форматирует — НЕ восстанавливает потерянное. Telegram-формат — ядро + средний приоритет, лимит ~4000 символов.

**Канон:** маппинг сущностей → секции, приоритеты, общие правила — в [protocol_format.md](../../ontology/protocol_format.md). Этот промпт — реализация Telegram-формата.

## Как использовать

Подай на вход:

```
## Выжимка
[полный выход L1]

## Досье (опционально)
[профайл участника, карточка проекта, история встреч — если есть]
```

Транскрипт НЕ подавать.

---

## Промпт

```
Твоя задача — превратить выжимку встречи в Telegram-сообщение: короткое, структурированное, готовое к отправке. Только самое важное — решения, задачи, проблемы. Остальное включай только если влезает.

## Главное правило

**Telegram-сводка = только факты этой встречи.** Всё берётся из выжимки. Досье — только для пометок ("подтвердили: ...", "уточнили: ..."), НЕ для удаления и НЕ как источник содержания. Додумывать то, чего нет в выжимке — ЗАПРЕЩЕНО.

---

## Что делать с выжимкой

Выжимка содержит: метаданные, краткое содержание, темы, сущности по типам (решения, задачи, цели, запросы, идеи, инсайты, факты, принципы, метрики, открытые вопросы, риски), требует уточнения. Каждая сущность с цитатой-опорой.

**Убрать** (не для Telegram):
- Цитаты-опоры (служебная информация)
- Темы (содержание раскрывается через секции)
- Поведенческие паттерны (отдельный артефакт, не для чата)

**Упаковать** сущности в секции. Тип из выжимки сохраняет смысл — секция задаёт упаковку для чтения:

| Тип в выжимке | Секция Telegram | Приоритет |
|---------------|-----------------|-----------|
| Решения | 📌 РЕШЕНИЯ И ДОГОВОРЁННОСТИ | ядро |
| Задачи | ✅ ЗАДАЧИ | ядро |
| Цели | 📌 РЕШЕНИЯ (если зафиксировано) или не включать | ядро |
| Запросы | 📌 РЕШЕНИЯ (если договорённость), ⚡ ИДЕИ (если без обязательства), или ⚠️ ПРОБЛЕМНАЯ ЗОНА | средний |
| Идеи | ⚡ ИДЕИ И ГИПОТЕЗЫ | низкий |
| Инсайты | 💡 ИНСАЙТЫ И НАБЛЮДЕНИЯ | низкий |
| Факты | 📎 СПРАВОЧНЫЕ (если ценны участникам) или не включать | низкий |
| Принципы | 📌 РЕШЕНИЯ (если принят на встрече) или 📎 СПРАВОЧНЫЕ | средний |
| Метрики | 📎 СПРАВОЧНЫЕ МАТЕРИАЛЫ | низкий |
| Открытые вопросы | ⚠️ ПРОБЛЕМНАЯ ЗОНА | ядро |
| Риски | ⚠️ ПРОБЛЕМНАЯ ЗОНА | ядро |
| Требует уточнения | ❓ ТРЕБУЕТ УТОЧНЕНИЯ | средний |

**Решение и задача сосуществуют.** Если на встрече и выбрали курс, и назначили действие — оба пункта (решение в "Решениях", задача в "Задачах"). Это разные слои, не дублирование.

---

## Фильтрация по приоритету

Telegram ограничен ~4000 символов на сообщение. Поэтому:

1. **Всегда включай** (ядро): итог, решения, задачи, к следующей встрече, проблемная зона
2. **Включай если влезает** (средний): требует уточнения, принципы-как-решения, запросы
3. **Включай только при наличии места** (низкий): идеи, инсайты, справочные

Если после включения ядра остаётся <500 символов — пропусти средний и низкий приоритет.

Используй ТОЛЬКО для пометок:
- Решение/задача совпадает с досье → пометь "подтвердили: [что]"
- Задача изменилась (срок/owner/формулировка) → пометь "уточнили: [что изменилось]"

**НЕ удаляй** пункты на основании досье.

## Лимит Telegram

- Лимит сообщения: ~4096 символов
- Целься в ~3800 символов (с запасом)
- Если не влезает — разбивай на части
- Помечай: `(1/2)`, `(2/2)` в конце каждого сообщения

**Правила разбиения:**
1. Разбивай по границам секций (не рви секцию пополам)
2. Если одна секция >3800 символов — разрезай внутри по подгруппам исполнителей или по пунктам, сохраняя заголовок секции в каждой части

---

## Стиль

"Решили", "договорились", "упёрлись", "застряли", "разошлись во мнениях", "оставили открытым", "сняли с повестки", "решили не делать", "отложили".

**Запрещено:** "в рамках обсуждения", "было принято решение", "участники пришли к выводу", "осуществить", "данный", "в части".

Коротко и ёмко. Без воды. Без рекомендаций от себя.

---

## Чего не делать

- Не добавляй информацию из досье в сводку
- Не удаляй пункты на основании досье
- Не восстанавливай то, чего нет в выжимке — если L1 потерял, Telegram не спасёт
- Не выдумывай сроки, owner, метрики
- Не лей воду — каждый пункт должен нести конкретный смысл
- Не давай рекомендаций от себя
- Не дублируй между секциями (кроме пары решение+задача)
- Не оставляй пустые секции
- Не включай поведенческие паттерны — они не для Telegram

---

Выводи ТОЛЬКО готовое Telegram-сообщение (или несколько, если разбито). Без вступлений, комментариев, объяснений.
```

## Примечания

**Источники:**
- Логика маппинга 11 типов → секции: `L2-prompt-protocol-full.md`
- Telegram-форматирование (эмодзи, разделители, лимит): из раннего черновика (удалён при cleanup 2026-03-30)
- Секция "К следующей встрече": `L2-prompt-protocol-full.md`

**Отличия от L2-prompt-protocol-full.md:**
- Вход тот же (выжимка + досье), но выход — Telegram-формат с эмодзи
- Приоритетная фильтрация: ядро → средний → низкий. Protocol включает всё, Telegram — только что влезает
- Поведенческие паттерны исключены (не для мессенджера)
- Лимит ~3800 символов на сообщение, разбивка на части

**Архитектурное решение:**
Telegram-сводка — отдельный L2-промпт, не параметр L2-protocol. Каждый формат протокола (полный, Telegram, email) — самостоятельный промпт со своей логикой фильтрации. Подробнее: `methodology/meeting_overlays.md`.

**Что не включено намеренно:**
- FRAME-скоринг — отложен
- Behavioral паттерны — вне формата Telegram
- Few-shot примеры — после тестирования

# Direction — шаблон папки направления

Scaffold для зон ответственности руководителя: маркетинг, продукт, продажи, delivery, CS и т.д. Каждое направление — top-level папка в проекте компании. Направление — постоянная сущность; человек (лидер) — переменная, назначается на направление.

Каталог шаблонов для нового направления. Содержит все возможные файлы, включая разворачиваемые по триггеру. При создании направления: взять нужные шаблоны, заменить плейсхолдеры `{...}`, начать заполнять.

## Структура

```
{direction_name}/
  01_overview.md       — что это направление (миссия, scope, лидер, цели, KPI)
  02_active.md         — что актуально сейчас (проблемы, задачи, вопросы, риски)
  meetings/            — протоколы встреч по направлению
    archive/           — сырые транскрипты
  03_progress.md      — хроника по встречам
  [04_decisions.md]    — решения направления (развёртывается по триггеру)
  {файлы}.md           — пользовательские (стратегия, контент-план, docs...)
```

## Быстрая навигация

**01_overview.md** (что за направление, куда идём) → **02_active.md** (что горит сейчас) → **03_progress.md** (что сделали)

## Два типа файлов

**Scaffold-файлы** (с номером: 01_, 02_...) — системные. Имеют миссию, блок "Работа с файлом", агент маршрутизирует данные сюда автоматически. Всегда сверху по сортировке.

**Пользовательские файлы** (без номера) — справочные материалы направления. Стратегия, контент-план, регламент, документация. Агент читает для контекста, но не маршрутизирует туда данные из встреч автоматически. Формат свободный, рекомендуется YAML frontmatter.

## Типовые направления

Справочник для развёртывания. Если у клиента свои названия — используем клиентские. Если нет — берём отсюда.

| Направление | Типовая миссия | Типовые KPI |
|---|---|---|
| Marketing | Лидогенерация, бренд, позиционирование | Лиды, CAC, конверсия воронки, охват |
| Sales | Выручка, pipeline, закрытие сделок | Revenue, pipeline, win rate, средний чек, цикл сделки |
| Product | Развитие продукта, roadmap, adoption | Adoption, retention, NPS, time-to-value |
| Delivery | Поставка клиентам, SLA, качество | SLA, utilization, CSAT, margin |
| Customer Success | Удержание, расширение, удовлетворённость | Churn, NRR, CSAT, time-to-resolution |
| HR / People | Найм, развитие, культура | Time-to-hire, retention, eNPS |
| Finance | Бюджет, P&L, cashflow | Burn rate, runway, gross margin |
| Operations | Процессы, эффективность, инфраструктура | Uptime, cost per unit, cycle time |

Это не обязательный список — у каждого CEO свой набор направлений. Таблица помогает при онбординге: "какие зоны ответственности у вас есть?"

## Принцип: миссия файла как фильтр

Тот же принцип что в клиентском scaffold. Агент перед размещением информации проверяет: **соответствует ли это миссии файла?** Не соответствует — ищет другой файл или оставляет в протоколе встречи.

Подробнее: [../principles.md](../principles.md)

## Маршрутизация по миссиям

| Информация | Тест | Файл |
|------------|------|------|
| Цель/KPI направления | Стратегическое, меняется раз в квартал? | 01_overview.md |
| Кто ведёт, команда | Факт о структуре? | 01_overview.md |
| Ключевой проект/инициатива | Крупный блок работы, не задача? | 01_overview.md |
| Текущая проблема | Идёт не так, требует внимания? | 02_active.md |
| Задача | Кто-то должен что-то сделать? | 02_active.md |
| Открытый вопрос | Не решено, требует возврата? | 02_active.md |
| Риск | Может пойти плохо? | 02_active.md |
| Идея | Предложение без обязательства? | 02_active.md |
| Стратегическое решение | Через 3 мес спросят "почему так"? | management/decisions.md (до развёртывания 04_decisions.md) |
| Наблюдение о человеке | Про стиль, развитие, коучинг? | team/{person}.md |
| Мелкое/контекстное | Не подходит ни под одну миссию? | остаётся в meetings/ |

## Разграничение файлов

| Файл | Миссия (вопрос) | Что хранить | Что НЕ класть |
|------|-----------------|------------|---------------|
| 01_overview.md | **ЧТО** это направление и **КУДА** идёт? | Миссия, scope, лидер, команда, цели, KPI, проекты, стратегические факторы | Задачи → active. Проблемы → active. Решения → decisions. Хроника → progress |
| 02_active.md | **ЧТО АКТУАЛЬНО** сейчас? | Проблемы, задачи, вопросы, риски, идеи, повестка встречи | Факты о направлении → overview. Решения → decisions. Коучинг → team/ |
| 03_progress.md | **КАК ДВИГАЛИСЬ?** | Хроника по встречам (2-4 строки на встречу) | Полные протоколы → meetings/. Задачи → active |
| 04_decisions.md | **ПОЧЕМУ** так решили? | Решения уровня направления с контекстом | Company-wide решения → management/decisions.md. Задачи → active |

## Правила маршрутизации встреч

**1-on-1 со встречей по направлению:**
- Операционка по зоне (задачи, KPI, проблемы) → файлы direction/
- Коучинг, наблюдения о человеке → team/{person}.md
- Стратегическое решение уровня компании → management/decisions.md

**Кросс-функциональные встречи (2+ направления):**
Протокол → management/meetings/. Не нужно решать "какое направление основное".

## Развёртывание

Структура разворачивается по мере роста данных:
- **Новое направление** → 3 файла (01_overview + 02_active + 03_progress) + meetings/
- **04_decisions.md** → появляется когда >5 решений, специфичных для направления (до этого решения в management/decisions.md)

Подробнее: [../principles.md](../principles.md) → Принцип 4.

## Соответствие file_spec.md

Все scaffold-файлы следуют спецификации файла Second AI Brain:
- YAML frontmatter (title, updated, version)
- Секция "Кратко" (<=100 слов)
- Блок "Работа с файлом"
- Секция "Связанные файлы"
- Автономные секции, <300 строк

# Решения: {направление}

## Кратко

Реестр стратегических решений по направлению с контекстом: что решили, почему, какие альтернативы отвергли. Накопленная логика выбора — через 3 месяца отвечает на "а почему так?". Не журнал встреч, не список задач.

**Работа с файлом:** decision — дописывать при принятии значимых решений по направлению. Растёт медленно. Что сюда: решения, которые меняют стратегию, процессы или структуру направления. Формат: что решили + почему + альтернативы + статус. Что НЕ сюда: company-wide решения (→ management/decisions.md), операционные выборы внутри недели (→ остаются в meetings/), задачи (→ 02_active). Тест: "через 3 месяца кто-то спросит 'почему так?' — это решение будет ответом?" Да → сюда. Нет → meetings/.

**Развёртывание:** этот файл появляется когда накопилось >5 решений, специфичных для направления. До этого решения хранятся в management/decisions.md.

## Связанные файлы

- 01_overview.md — обзор направления (цели, стратегия)
- 02_active.md — операционный хаб (задачи, вопросы, риски)
- 03_progress.md — хроника (решения привязаны к встречам)
- ../management/decisions.md — company-wide решения

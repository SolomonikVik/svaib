# source/ — исходные данные

Числовые данные клиента: xlsx-файлы (или CSV) в том виде, в каком клиент с ними работает. **Read-only для AI** — мы не редактируем, не нормализуем.

AI-агент читает эти файлы Python-ом (xlsx-skill / Pandas), а не глазами. Семантика — какой лист, какая колонка, что значит — описана в `../{domain}.md` секциями `datasets` и `fields`.

## Что лежит

| Файл | Домен | Источник | Обновляется |
|------|-------|----------|-------------|
| `<file>.xlsx` | `../<domain>.md` | <откуда берётся> | <как часто> |

> Заполняется при подключении источников.

## Правила

**Файл живёт здесь как есть.** Не «причёсываем» под себя.

**Имена и форматы фиксируются в domain-файле**, не в самом xlsx.

**Синонимы CEO** — в `synonyms` поля датасета, не в имени колонки.

**При изменении схемы клиентом** (новая колонка, переименование) — обновить `datasets/fields` в `../<domain>.md`, не сам xlsx.

## Как обновлять

- Клиент работает в xlsx как привык
- Новая версия → перезаписать файл здесь (или подтянуть из Google Sheets)
- При изменении структуры — синхронизировать domain-файл

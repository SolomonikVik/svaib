# Метрики домена {domain} — {компания}

Функциональный файл метрик по одному домену — то же, что `business-metrics.md`, но по одной функции и с более подробной семантикой.

**Имя файла — каноническое.** Переименуй `{domain}` в одно из:
- `finance` — прибыль, маржа, расходы, cash, рентабельность, P&L
- `sales` — выручка, pipeline, конверсии, средний чек, LTV
- `operations` — delivery, качество, сроки, capacity, SLA
- `people` — команда, найм, удержание, производительность
- `customer` — retention, churn, LTV, когортный анализ, customer success
- `marketing` — каналы, CAC по каналам, контент-воронка, атрибуция
- `product` — DAU/MAU, adoption, feature usage

Других имён domain-файлов не бывает. Нарезка — по бизнес-смыслу, не по источнику: один файл может покрывать метрики из разных xlsx / листов / источников.

## Источник

**Источник:** `{file}.xlsx`, лист «{sheet}».

**Если источников несколько** — заменить эту секцию на `## Источники` с подразделами `### {source_id}` (snake_case, ASCII, уникальный в файле), и в каждой метрике добавить поле `**Источник:**` со значением `{source_id}`:

```md
## Источники

### source_finance

**Источник:** `finance.xlsx`, лист «P&L».

### source_ops

**Источник:** `ops.xlsx`, лист «Delivery».
```

## Как заполнять метрики

**Имя метрики** (заголовок `### {имя}`) — каноническое: как метрика реально называется внутри компании. Краткое, однозначное, стабильное (меняется формула — имя не меняется; меняется управленческая сущность — это другая метрика). Уникальное среди всех metrics-файлов клиента.

**Поля паспорта:**
- **Бизнес-смысл** — что метрика значит для бизнеса.
- **Как считается** — правило расчёта с явной временной логикой и способом вычисления (за период / на момент / доля чего-то от чего-то).
- **Единица** — `млн руб` / `%` / `шт` / …
- **Направление** — `рост — хорошо` / `рост — плохо` / `нейтрально`.
- **Источник** — `{source_id}` из секции `## Источники`. Указывается, только если источников в файле несколько.

**Раскладка** — два взаимоисключающих варианта:
- **Без групп.** Один контейнер `## Метрики`, внутри подряд `### {имя}`.
- **С группами.** Серия `## {имя группы}`, внутри каждой `### {имя}`. Контейнер `## Метрики` в этом варианте НЕ ставится.

В шаблоне ниже — вариант без групп. Если у домена есть естественные блоки — заменить контейнер `## Метрики` на серию `## {имя группы}`.

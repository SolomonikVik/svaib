# meetings/ — встречи подразделения/направления

Встречи внутри одного подразделения/направления (`{org_node}`): 1-on-1 с сотрудниками, планёрки отдела, ретроспективы. А также кросс-функциональные встречи дочерних узлов внутри этой единицы, если она является группировкой.

**Когда сюда.** По правилу наименьшей общей родительской папки ([../../README.md](../../README.md)). Встреча охватывает только эту единицу или её вложенные узлы. Встреча с участниками из других направлений → смотри папку общего родителя (`{parent}/meetings/` или `04_management/meetings/`).

**Вне scope.** Встречи, меняющие 02_strategy/ (vision, goal, plan, weekly_progress).

## Что лежит

- `YYYY-MM-DD_{topic}_summary.md` — выжимки L1 (создаёт orchestrator-meeting)
- `archive/` — транскрипты после обработки (`YYYY-MM-DD_{topic}_transcript.md`)

## Как обрабатывать

Скилл [orchestrator-meeting.md](../../../../skills/meeting-analysis/orchestrator-meeting.md). Вызывает L1 (выжимка) и L2 (обновление файлов единицы + Telegram). Руководитель подтверждает перед применением.

## Маршрутизация обновлений

Обновления маршрутизируются через [L2-procedure-scaffold-update.md](../../../../skills/meeting-analysis/L2-procedure-scaffold-update.md) в файлы единицы (01_overview / 02_active / 03_progress / 04_decisions), а при кросс-функциональности — также в файлы затронутых дочерних узлов.

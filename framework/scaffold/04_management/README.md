# 04_management/ — Управление подразделениями

Дерево подразделений и направлений компании. Одна папка на каждое подразделение или направление, с которым CEO регулярно работает. Развернуто только живое.

## Что лежит

- `meetings/` — стратсессии CEO и кросс-функциональные встречи верхнего уровня
- `glossary.md` — клиентский словарь и жаргон (читается перед обработкой каждой встречи)
- Папки подразделений и направлений — например `marketing/`, `sales/`, `commercial/`, `product/`. Устройство любой из них смотри как образец

## Правила уровня папки

**Разворачивание.** Новое подразделение появляется, когда CEO реально с ним работает: встречи, задачи, решения. Факт существования отдела в компании — не триггер.

**Общая родительская папка для встреч.** Протокол встречи кладётся в `meetings/` ближайшей папки, охватывающей всех участников. Встреча только внутри маркетинга → `marketing/meetings/`. Встреча маркетинга и продаж, когда они внутри `commercial/` → `commercial/meetings/`. Стратсессия всей верхушки → `04_management/meetings/`.

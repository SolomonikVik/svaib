# projects/ — Проекты

Дерево проектов внутри пространства. Один проект — одна папка с полной раскладкой: суть, цели, операционка, бэклог, хроника, решения, протоколы встреч.

## Природа

**Проект — временная активность с горизонтом и завершением.** Этим отличается от направления (`04_management/{org_node}/`), которое существует постоянно. Примеры проектов: миграция CRM, запуск нового продукта, стройка склада, внедрение OKR.

## Что лежит

- Папки проектов — например `migration_crm/`, `launch_v2/`, `building_warehouse/`. Устройство любой смотри как образец `{project_name}/`.
- (опционально) `meetings/` верхнего уровня — кросс-проектные встречи

## Правила уровня папки

**Триггер развёртывания.** Новый проект появляется, когда по нему начинается реальная работа: команда, встречи, задачи, решения. Само наличие проекта в стратегии — не триггер. Это прямое применение Iron Law из [../MODEL.md](../MODEL.md): «разворачивай только живое».

**Полная раскладка при триггере.** При создании проекта разворачивается **вся полная раскладка** (по аналогии с `04_management/{org_node}/`): `_inbox/01_inbox.md`, `01_overview.md`, `02_active.md`, `02_backlog.md`, `03_progress.md`, `04_decisions.md`, `meetings/`. Пустые файлы и секции допустимы — каркас должен стоять с первого дня. Правило сделано так, чтобы агент не «изобретал» структуру по ходу работы.

**Общая родительская папка для встреч.** Протокол встречи кладётся в `meetings/` ближайшей папки, охватывающей всех участников. Встреча только внутри проекта → `{project_name}/meetings/`. Кросс-проектная встреча → `projects/meetings/` (если развёрнут) или в `04_management/meetings/`.

## Структура

```
{project_name}/
  README.md                 — устройство папки проекта (читать первым)
  _inbox/01_inbox.md        — входящее на разбор
  01_overview.md            — суть, цели и КР, границы, команда, дорожная карта
  02_active.md              — операционный хаб (повестка встречи, задачи, вопросы, проблемы и риски)
  02_backlog.md             — полный список задач и идей проекта
  03_progress.md            — хроника по встречам, факт по КР
  04_decisions.md           — стратегические решения с контекстом
  meetings/                 — протоколы встреч проекта
    archive/                — сырые транскрипты
```

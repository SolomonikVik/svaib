# Прогресс: {название проекта}

## Кратко

Хроника проекта: динамика метрик по периодам и краткий лог по встречам. Быстро увидеть траекторию — как двигались, что прошли, где сейчас. Не дублирует протоколы — даёт общую картину.

**Работа с файлом:** log — две таблицы с разными правилами обновления. **Не менять** старые записи в обеих.

- **Динамика метрик** — обновляется **только в конце отчётного периода** (месяц / неделя — задано в overview): добавляется новая колонка, заполняются значения.
- **Хроника встреч** — обновляется **после каждой встречи**: 2-4 строки (что обсудили, что сдвинулось, следующий шаг). НЕ дублировать полный протокол (→ meetings/).

Что сюда: динамика метрик, прогресс между встречами, ключевые сдвиги. Что НЕ сюда: полные протоколы (→ meetings/), задачи (→ 02_active / 02_backlog), решения с контекстом (→ 04_decisions).

## Динамика метрик

> Метрики проекта по периодам. Метрики = ровно те, что стоят в [01_overview.md](01_overview.md), не больше. Операционные метрики — в отдельный файл или дашборд.
>
> **Source of truth — эта таблица.** Снимок «Сейчас» в overview синхронизируется по концу периода: добавил колонку здесь → перепиши «Сейчас» в overview.
>
> **Формат заголовка колонки:** дата конца периода в `YYYY-MM-DD`. Пример: `2026-04-30` для апреля, `2026-04-19` для недели, `2026-06-30` для Q2.
>
> **Колонка «Цель»** — копия целевого значения из overview, не меняется по периодам.

# Бэклог: {название проекта}

## Кратко

Полный список будущей работы проекта. Две зоны зрелости:
- **План по вехам** — разобранные задачи, привязанные к вехам из дорожной карты ([01_overview.md](01_overview.md))
- **Идеи и гипотезы** — сырая зона, разбирается периодически

Если у проекта вех нет или одна — используем плоский список под одним заголовком (например, `### Все задачи` или `### Работы`). Структура с вехами нужна, когда их несколько и важно видеть последовательность.

**Работа с файлом:** plan — дописывать по мере появления задач (из [_inbox/01_inbox.md](_inbox/01_inbox.md) при разборе, из встреч, из идей). Взятая в работу задача → переносится в [02_active.md](02_active.md). Выполненная → перенос в [03_progress.md](03_progress.md). Что сюда: будущие задачи, декомпозиция вех, идеи, гипотезы, эксперименты. Что НЕ сюда: то что делается сейчас (→ 02_active), уже сделанное (→ 03_progress), решения с контекстом (→ 04_decisions), сырое на разбор (→ _inbox/01_inbox).

# Решения: {название проекта}

## Кратко

Реестр стратегических решений по проекту с контекстом: что решили, почему, какие альтернативы отвергли. Накопленная логика выбора — через 3 месяца отвечает на «а почему так?». Не журнал встреч, не список задач.

**Работа с файлом:** decision — дописывать при принятии значимых решений по проекту. Растёт медленно. Что сюда: решения, которые меняют направление проекта, архитектуру, формат работы, методологию, scope. Формат: что решили + почему + альтернативы + статус. Что НЕ сюда: операционные выборы внутри недели (→ остаются в meetings/), задачи (→ 02_active / 02_backlog), факты о проекте (→ 01_overview). Тест: «через 3 месяца кто-то спросит "почему так?" — это решение будет ответом?» Да → сюда. Нет → meetings/.

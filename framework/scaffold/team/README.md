# Team — обзор

Scaffold для команды руководителя: маппинг людей и направлений, принципы работы с профайлами. Оргструктура — в [org_structure.md](org_structure.md), профайлы — в {person}.md. Люди отделены от направлений — человек может сменить зону, но его профайл остаётся.

**Развёртывание:** team/ создаётся сразу при онбординге как папка с профайлами. CEO перечисляет C-1 → на каждого создаётся {person}.md из шаблона. Это не разворачиваемая секция — профайлы нужны с первого дня для meeting processing и коучинга.

**Связь с management/team.md:** для малой команды (до 3 человек без регулярных 1-on-1) достаточно management/team.md — плоский список. Триггер развёртывания: появились регулярные 1-on-1 и потребность в коучинг-логе → team.md разворачивается в team/. Это одна сущность на разных стадиях по Принципу 4.

## Структура

```
team/
  README.md              — этот файл (обзор, маппинг направлений)
  org_structure.md       — оргструктура (имя → роль / подразделение / зона ответственности)
  {person}.md            — профайл сотрудника (стиль, сильные стороны, развитие)
```

## Оргструктура

→ [org_structure.md](org_structure.md) — кто, какая роль, подразделение, зона ответственности. Чистый документ, можно распечатать.

## Принципы

**Профайл — про человека, не про направление.** Стиль, сильные стороны, зоны развития, коучинговые наблюдения. KPI и задачи направления → direction/.

**Один файл на человека.** Содержит и reference-данные (стиль, сильные стороны), и log-секцию (коучинг-лог). Исключение из Принципа 3 (state vs log), обосновано тем что один файл на человека удобнее, чем два.

**Двойная роль при вложенных направлениях.** Один человек может быть лидером направления-контейнера и лидером одного из его под-направлений (Принцип 5: разные аспекты — стратегия направления vs операционка подразделения). В таблице маппинга указывать обе роли явно.

**Двусторонние ссылки.** Каноничный маппинг человек ↔ направление — здесь, в таблице выше. Для навигации: team/{person}.md ссылается на direction/01_overview.md, и наоборот. При смене лидера — обновить: эту таблицу, person.md, каждый overview.md где человек указан лидером (при двойной роли — оба overview).

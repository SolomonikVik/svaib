# Team — профайлы людей и оргструктура

Scaffold для команды руководителя: кто кому подчиняется, кто какое направление ведёт, профайлы ключевых людей. Люди отделены от направлений — человек может сменить зону, но его профайл остаётся.

**Развёртывание:** team/ создаётся сразу при онбординге как папка с профайлами. CEO перечисляет C-1 → на каждого создаётся {person}.md из шаблона. Это не разворачиваемая секция — профайлы нужны с первого дня для meeting processing и коучинга.

**Связь с management/team.md:** для малой команды (до 3 человек без регулярных 1-on-1) достаточно management/team.md — плоский список. Триггер развёртывания: появились регулярные 1-on-1 и потребность в коучинг-логе → team.md разворачивается в team/. Это одна сущность на разных стадиях по Принципу 4.

## Структура

```
team/
  README.md              — этот файл (оргструктура + маппинг направлений)
  {person}.md            — профайл сотрудника (стиль, сильные стороны, развитие)
```

## Оргструктура

> CEO → C-1 уровень → ключевые люди. Обновлять при изменениях.

```
{CEO}
├── {Имя 1} — {роль}
├── {Имя 2} — {роль}
└── {Имя 3} — {роль}
```

## Маппинг направлений

> Каноничный источник связи человек ↔ направление. person.md и overview.md содержат ссылки для навигации, но при рассинхроне — верить этой таблице.

| Человек | Направление | Роль в направлении |
|---------|------------|-------------------|
| {Имя 1} | [{direction}](../{direction}/01_overview.md) | Лидер |
| {Имя 2} | [{direction}](../{direction}/01_overview.md) | Лидер |

## Принципы

**Профайл — про человека, не про направление.** Стиль, сильные стороны, зоны развития, коучинговые наблюдения. KPI и задачи направления → direction/.

**Один файл на человека.** Содержит и reference-данные (стиль, сильные стороны), и log-секцию (коучинг-лог). Исключение из Принципа 3 (state vs log), обосновано тем что один файл на человека удобнее, чем два.

**Двусторонние ссылки.** Каноничный маппинг человек ↔ направление — здесь, в таблице выше. Для навигации: team/{person}.md ссылается на direction/01_overview.md, и наоборот. При смене лидера — обновить в трёх местах: эту таблицу, person.md, overview.md.

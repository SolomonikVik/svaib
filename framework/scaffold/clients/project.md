**Работа с файлом:** reference — обновлять при стратегических изменениях (формат, условия, новые запросы на решения). Что сюда: формат и условия сделки, контекст отношений, оценка контрагента, модель работы (cadence, роли), метрики успеха, карта решений/продуктов, программа/стейджи. Что НЕ сюда: операционные задачи (→ active), решения с контекстом why/alternatives (→ decisions), мелкие решения внутри встречи (→ meetings/), техническая конфигурация (→ setup), динамика встреч (→ progress). Тест запроса: контрагент хочет НОВОЕ решение/продукт → карта решений сюда. Контрагент просит СДЕЛАТЬ конкретное → active.

## Связанные файлы

- profile.md — контрагент (кто он)
- company.md — бизнес контрагента
- decisions.md — стратегические решения с контекстом (почему, альтернативы)
- setup.md — техническая конфигурация
- active.md — операционный хаб (задачи, вопросы, риски, повестка встречи)
- progress.md — хроника движения проекта

# Clients — шаблоны для клиентских проектов

Scaffold для ведения проектов с внешними контрагентами: клиенты, партнёры, подрядчики. Универсальная модель — подходит для консалтинга, IT-услуг, дизайна, любого сервисного бизнеса.

## Структура

```
clients/
└── {project_name}/
    ├── profile.md        — кто контрагент (человек, предпочтения, контекст)
    ├── company.md        — его бизнес (продукт, цифры, люди, стратегия)
    ├── project.md        — что делаем (сделка, модель работы, метрики успеха)
    ├── decisions.md      — почему так решили (стратегические решения)
    ├── setup.md          — техническая конфигурация (если есть)
    ├── tasks.md          — что делать сейчас (задачи, вопросы, идеи, риски)
    ├── progress.md       — как двигались (хроника)
    ├── meetings/         — протоколы встреч
    │   └── archive/      — полные транскрибации
    ├── deliverables/     — материалы для клиента
    └── docs/             — юридические документы
```

## Принцип: миссия файла как фильтр

Каждый файл имеет миссию и инструкцию агенту ("Работа с файлом"). Агент проверяет: соответствует ли информация миссии файла? Не соответствует → ищет другой файл или оставляет в протоколе встречи. Подробнее: [../principles.md](../principles.md)

## Развёртывание

Минимальный старт: profile + project + tasks. Остальные файлы появляются по мере роста данных. Подробнее: [../principles.md](../principles.md) → Принцип 4.

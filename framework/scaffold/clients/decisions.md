# Решения: {название проекта}

## Кратко

Реестр стратегических решений по проекту с контекстом: что решили, почему, какие альтернативы отвергли. Накопленная логика выбора — через 3 месяца отвечает на "а почему так?". Не журнал встреч, не список задач.

**Работа с файлом:** decision — дописывать при принятии значимых решений. Растёт медленно. Что сюда: решения, которые меняют направление проекта, определяют архитектуру, формат работы, методологию. Формат: что решили + почему + альтернативы + статус. Что НЕ сюда: операционные выборы внутри недели (→ остаются в meetings/), задачи (→ active), запросы контрагента (→ project, карта решений). Тест: "через 3 месяца кто-то спросит 'почему так?' — это решение будет ответом?" Да → сюда. Нет → meetings/.

## Связанные файлы

- project.md — паспорт проекта (формат, условия, карта решений)
- active.md — операционный хаб (задачи, вопросы, риски, повестка)
- progress.md — хроника (решения привязаны к встречам)

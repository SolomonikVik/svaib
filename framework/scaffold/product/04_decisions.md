# Решения: {название продукта}

## Кратко

Реестр стратегических решений по продукту с контекстом: что решили, почему, какие альтернативы отвергли. Накопленная логика выбора — через 3 месяца отвечает на «а почему так?». Не журнал встреч, не список задач.

**Работа с файлом:** decision — дописывать при принятии значимых решений по продукту. Растёт медленно. Что сюда: решения, которые меняют позиционирование, архитектуру, scope, ценовую модель, границы продукта. Формат: что решили + почему + альтернативы + статус. Что НЕ сюда: операционные выборы внутри недели (остаются в протоколах встреч), задачи (→ [02_active.md](02_active.md) / [02_backlog.md](02_backlog.md)), факты о продукте (→ [00_product.md](00_product.md) / [01_overview.md](01_overview.md)). Тест: «через 3 месяца кто-то спросит "почему так?" — это решение будет ответом?» Да → сюда.

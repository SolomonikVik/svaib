# Framework — актуальное

## Кратко

Операционный хаб направления: контекст передачи между чатами (Session Handoff), активные задачи, открытые вопросы. Всё, что требует внимания сейчас — в одном месте. State-файл: перезаписывается, не растёт. Вся история — в [03_progress.md](03_progress.md).

**Работа с файлом:** plan — обновлять после каждой значимой сессии и по мере появления/закрытия задач. Целевой размер: 40-80 строк (исключение — период крупной миграции направления). Что сюда: текущие задачи направления, блокеры, открытые вопросы. Что НЕ сюда: что за продукт и зачем (→ [product.md](product.md)), как устроен внутри (→ [architecture.md](architecture.md)), состояние реализации (→ [01_overview.md](01_overview.md)), задачи на будущее (→ [02_backlog.md](02_backlog.md)), история (→ [03_progress.md](03_progress.md)), продуктовые решения (→ [04_decisions.md](04_decisions.md)), входящее на разбор (→ [_inbox/01_inbox.md](_inbox/01_inbox.md)).

## Активные задачи

- **Meeting analysis (beta, entity-only).** Следующее: (1) behavioral extractor — определиться v1 vs v2, верифицировать на 2+ реальных транскриптах, подключить к оркестратору. (2) Pre-processing транскрибации — коррекция атрибуции спикеров до выжимки.
- **Memory — план внедрения методологии.** 8 конвенций → 16 хуков → пересмотр существующих файлов svaib. Приоритизация по кейсам-обоснованиям (Rule Injection, Task Classifier, Read Limit Guard).

## Открытые вопросы

- **Скиллы зависят от стабилизации meeting-analysis** — пока behavioral не подключён, остальные домены скиллов не стартуют. Решение: после верификации behavioral на реальных транскриптах — запустить следующий домен.
- **Системные вопросы из аудита Фазы 2.7** (решать единым правилом для всех направлений, не точечно): (1) ротация log-файлов, (2) статусы в `04_decisions`, (3) граница «решение vs выбор внутри встречи».

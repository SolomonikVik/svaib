# Plugin — деплоимый пакет Second AI Brain

## Кратко

Собранный пакет для клиента: наш продуктовый слой поверх scaffold-структуры. Plugin содержит методологию в исполняемом формате и доставляется через внешний runtime / AI-host.

Исходная разработка промптов и навыков — в [../skills/](../skills/). Skills — мастерская, Plugin — витрина.

**Статус:** Скелет. Наполнение — после обкатки методологии на первых клиентах.

## Компоненты

| Что | Папка | Что делает |
|-----|-------|-----------|
| Skills | `skills/` | Методология в исполняемом формате: обработка встреч, ревью, планирование |
| Agents | `agents/` | Автоматизация: диспетчер задач, обновление документации |
| Hooks | `hooks/` | Триггеры: после встречи, при закрытии сессии |

## Связь с моделью подписки

Плагин не хранит данные клиента. Данные — у клиента (в scaffold-структуре). Плагин — интеллектуальный слой поверх.

## Граница с runtime

Plugin и runtime — не одно и то же.

- **Plugin** — наш слой: skills, agents, hooks, то есть методология и автоматизация.
- **Runtime** — внешний слой доставки: AI-host, scheduling, orchestration, интерфейсные коннекторы.

Мы не считаем runtime частью ядра продукта. Текущее решение по этой границе зафиксировано в [../../meta/product/decisions/runtime-choice.md](../../meta/product/decisions/runtime-choice.md).

### Что это значит на практике

- plugin должен оставаться максимально runtime-agnostic;
- нельзя описывать plugin как артефакт только для `Claude Code / Cowork`, если это не часть самой сути;
- конкретный runtime может меняться без пересборки продуктовой логики;
- текущий пилотный кандидат на runtime — `HKUDS/nanobot`, но plugin не должен быть описан как "nanobot-плагин".

## Delivery-модель

Текущая целевая схема:

```
scaffold (данные клиента)
    +
plugin (наш продуктовый слой)
    +
runtime / host (внешняя доставка)
    =
рабочий Second AI Brain у клиента
```

Это значит:

- scaffold остаётся у клиента;
- plugin обновляется с нашей стороны;
- runtime выбирается по удобству, безопасности и скорости delivery.

## Текущая реализация (Anthropic Claude)

Первые клиенты используют Anthropic Claude (Claude.ai, Claude Code, Cowork). В этом случае delivery через нативные механизмы платформы:

- **CLAUDE.md** (в корне scaffold) — инструкция агенту: роль, навигация, протокол чтения, ритуалы. Читается платформой автоматически.
- **`.claude/skills/`** — исполняемые навыки (промпты, процедуры). Подключаются Claude автоматически по описанию задачи.

Это **временное решение**, привязанное к Anthropic. При смене runtime: CLAUDE.md → system prompt нового runtime, `.claude/skills/` → native skills mechanism. Продуктовая ценность (методология, промпты) не зависит от runtime — меняется только обёртка.

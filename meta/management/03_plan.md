## Кратко

Оперативное управление проектом. Что конкретно делаем прямо сейчас: план периода, бэклог, план недели, план дня. Содержит ТОЛЬКО будущее и настоящее — прошлое уходит в weekly_progress. Четыре уровня планирования: период (из goal) → бэклог → неделя → день. Если есть направления/подпроекты — задачи группируются по ним.

**Работа с файлом:** обновлять ежедневно (план дня), еженедельно по понедельникам (план недели + чистка бэклога), в начале каждого периода (план периода). Размер компактный и стабильный — не растёт, обновляется. Выполненные задачи → итоги в 04_weekly_progress. Незавершённые задачи → перенести на следующую неделю. LLM читает часто — оперативный контекст.

**Клиенты:** в плане обозначаются инициалами (АС, КИТ, ЕВ, Ник, Еф). Расшифровка — в clients/.

## Бэклог

Между планом периода и планом недели. Сюда попадает всё, что "надо не забыть" — из чатов, от клиентов, идеи, инсайты. При планировании недели (понедельник) — берёшь из бэклога в план недели. Задачи которые лежат больше 3-4 недель и не двигаются — либо делаешь, либо явно замораживаешь с пометкой ❄️ и причиной, либо удаляешь. Замороженные задачи — тоже здесь. Бэклог не растёт бесконечно — чистится каждый понедельник при планировании недели. Инбоксы папок — для материалов (ссылки, заметки, черновики), не для задач. Задача живёт либо в бэклоге, либо её нет.

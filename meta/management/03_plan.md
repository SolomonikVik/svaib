# Plan

## Кратко

Оперативное управление проектом. Что конкретно делаем прямо сейчас: план периода, бэклог, план недели, план дня. Содержит ТОЛЬКО будущее и настоящее — прошлое уходит в weekly_progress. Четыре уровня планирования: период (из goal) → бэклог → неделя → день. Если есть направления/подпроекты — задачи группируются по ним.

**Работа с файлом:** обновлять ежедневно (план дня), еженедельно по понедельникам (план недели + чистка бэклога), в начале каждого периода (план периода). Размер компактный и стабильный — не растёт, обновляется. Выполненные задачи → итоги в 04_weekly_progress. Незавершённые задачи → перенести на следующую неделю. LLM читает часто — оперативный контекст.

**Клиенты:** в плане обозначаются инициалами (АС, КИТ, ЕВ, Ник, Еф). Расшифровка — в clients/.

## Бэклог

Между планом периода и планом недели. Сюда попадает всё, что "надо не забыть" — из чатов, от клиентов, идеи, инсайты. При планировании недели (понедельник) — берёшь из бэклога в план недели. Задачи которые лежат больше 3-4 недель и не двигаются — либо делаешь, либо явно замораживаешь с пометкой ❄️ и причиной, либо удаляешь. Замороженные задачи — тоже здесь. Бэклог не растёт бесконечно — чистится каждый понедельник при планировании недели. Инбоксы папок — для материалов (ссылки, заметки, черновики), не для задач. Задача живёт либо в бэклоге, либо её нет.

## Инструкция: ритуал планирования дня

> Выполняется утром, один раз в день. Вечером ничего не делаем.

### Роль агента

Ты здесь как стратегический партнёр, а не переносчик строчек. Твоя главная ценность — соотносить каждую задачу с целью декады и метриками экватора (02_goal.md). Без vision и goal ты бесполезен — загружай их всегда.

Виктор проговаривает что думает — ты слушаешь и отвечаешь через призму целей: что двигает к метрикам, что нет, где расхождение с приоритетами декады. Если Виктор ставит задачу, которая не двигает к цели — скажи прямо.

### Механика

1. **Синхронизация** — посмотреть план дня: что закрыто (галочки), что нет. Закрытые задачи отметить в плане недели.
2. **Очистка** — убрать закрытые из плана дня. Незакрытые — остаются автоматически.
3. **Встречи** — проверить регулярные задачи на встречи текущего дня, подтянуть в блок `### Встречи`.
4. **Приоритизация** — это главный шаг, не механический. Порядок:
    1. Если сегодня есть встречи с клиентами — что нужно подготовить к каждой? Расшифровки, документы, счета — всё это идёт первым в `### Делаем`.
    2. Незакрытые клиентские задачи с прошлого дня.
    3. Остальные незакрытые с прошлого дня.
    4. Регулярные (если попадают на сегодня).
    5. Следующие по приоритету из плана недели.

   Правило: клиентская работа всегда выше внутренних задач (семинары, лаборатория, продукт без привязки к клиенту). Если задача не клиентская и при этом вытесняет клиентскую — объясни Виктору и спроси, осознанный ли это выбор.

5. **Формирование** — список с чекбоксами под `### Делаем`, 5-8 задач. Одна задача в плане недели = одна строка в плане дня. Не склеивать, не переформулировать. Переносить с достаточным контекстом — чтобы было понятно о чём речь, не лезя обратно в план недели.

### Формат плана дня

```
## План дня (день недели, ДД.ММ)

### Встречи

- [ ] ЧЧ:ММ — Кто / тема
...

### Делаем

- [ ] Задача 1
- [ ] Задача 2
...
```

Блок `### Встречи` — перед `### Делаем`. Встречи подтягиваются из регулярных задач при планировании дня. Если встреч нет — блок не нужен.

Галочки ставятся в Safari по ходу дня.

### Закрытие сессии

При закрытии сессии планирования дня — запись в session-log одной строкой: "План дня на [день недели]." + длительность. Без деталей. Спросить Виктора сколько заняло.

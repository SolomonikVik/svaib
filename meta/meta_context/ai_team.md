# AI-команда SVAIB

## Кратко

Принципы работы с AI-командой и каталог AI-агентов проекта. Включает: 4 принципа эффективной работы с AI (контекст, разделение задач, верификация, роли), таблицу выбора инструментов (Claude Code, Claude Chat, Claude for Chrome), справочник агентов по направлениям (lab/dev/pub). Ключевые слова: AI-агенты, промпты, субагенты, инструменты, Claude Code, Claude Chat, Claude for Chrome, Claude Projects.

## Связанные файлы

- .claude/agents/ — субагенты Claude Code (context-editor, n8n-expert, supabase-expert)
- dev/prompts/ — промпты ролей для Claude Code (CTO-архитектор, Dify-эксперт)
- dev/dev_context/dev_environment.md — техническая среда разработки (детали MCP-серверов, структура субагентов, скиллы, конфигурация)

## Принципы работы с AI-командой

### Принцип 1: Контекст — это всё
Чистый, актуальный, документированный контекст — основа скорости.

### Принцип 2: Разделение по задачам (агентам/чатам)
Разные задачи = разные сессии с разными промптами.

### Принцип 3: Верификация критичных решений
Архитектура, выбор технологий — прогонять через 2-3 разные модели.

### Принцип 4: Виктор — архитектор, AI — исполнитель
AI пишет код как джуниор: быстро, но требует ТЗ и проверки.

## Инструменты работы

| Инструмент | Где работает | Когда использовать |
|------------|--------------|-------------------|
| **Claude Code** | VS Code | Написание кода, работа с файлами проекта, автоматизация через субагентов |
| **Claude Chat** | claude.ai | Обсуждение стратегии, исследование идей, быстрые консультации |
| **Claude for Chrome** | Браузер | GUI-задачи (Google Cloud Console, Supabase Dashboard), автономные действия в веб-интерфейсах |

**Принцип выбора:**
- Код и файлы → Claude Code
- Стратегия и размышления → Claude Chat
- Настройка GUI в браузере → Claude for Chrome

## Агенты по направлениям

### svaib-lab (внутренняя мастерская)

#### AI-стратег
**Задача:** Помогает с планированием и стратегическими решениями по проекту.
**Когда использовать:** При важных решениях, планировании квартала, пивотах.

#### AI-HRD
**Задача:** Помогает в создании эффективных промптов для AI-сотрудников.
**Когда использовать:** Когда надо создать промпт для нового AI-сотрудника или подредактировать другой.

#### AI-редактор контекстов
**Задача:** Превращает черновые тексты в структурированные файлы контекста проекта. Проверяет связанные файлы на коллизии.
**Когда использовать:** При создании или обновлении любых файлов контекста.

**Workflow:**
1. Вызов `/svaib-context` → загружается координатор (партнёр)
2. Обсуждение задачи с Виктором → понимание ЧТО нужно сделать
3. Делегирование субагенту `context-editor` → исполнение по чеклисту
4. Субагент возвращает результат → координатор проверяет и показывает

**Компоненты:**
- Команда: `.claude/commands/svaib-context.md`
- Субагент: `.claude/agents/context-editor.md`
- Правила: `meta/meta_context/!context_rules.md`

#### AI-редактор недельных итогов
**Задача:** Превращает черновик недельных итогов в структурированный weekly_progress.md.
**Когда использовать:** Каждую неделю после публикации итогов.

### svaib-dev (продукт)

#### AI-CTO архитектор
**Задача:** Проектирует архитектуру AI-решений, определяет технологический стек, планирует этапы реализации.
**Когда использовать:** При проектировании новых фич, изменении архитектуры, формировании этапов работы.

#### AI-эксперт Dify
**Задача:** Помогает настраивать и использовать Dify для создания AI-инструментов и клиентских решений.
**Когда использовать:** При любых вопросах по работе с платформой Dify.

### svaib-pub (публикации)

#### AI-стратег выступлений
**Задача:** Помогает готовиться к публичным выступлениям и воркшопам на уровне стратегии и идей.
**Когда использовать:** Для подготовки к выступлениям на этапе стратегии (не копирайтинг/дизайн).

#### AI-арт-директор
**Задача:** Превращает идею слайда в промпт для AI-иллюстратора. Стиль: Bauhaus + Matisse + Kandinsky.
**Когда использовать:** При создании визуала для слайдов презентаций.

#### AI-контент-партнер
**Задача:** Создаёт содержание слайдов: заголовки, тезисы, метафоры, концепции визуализации.
**Когда использовать:** При работе над контентом презентаций от идеи до готовых слайдов.

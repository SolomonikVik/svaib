# AI-команда SVAIB

## Кратко

Принципы работы с AI-командой, описание инструментов и каталог AI-агентов. Включает: 4 принципа эффективной работы с AI, 3 инструмента (Claude Chat, Claude Code, Claude for Chrome) с анализом возможностей, справочник агентов по направлениям (lab/dev/pub).

## Связанные файлы

- .claude/agents/ — субагенты Claude Code (сейчас: context-editor)
- dev/prompts/ — промпты ролей для Claude Code (CTO-архитектор, Dify-эксперт)
- .claude/claude_code_mechanics.md — справочник по механизмам Claude Code (MCP-серверы, субагенты, промпты, скиллы)

---

## Принципы работы с AI-командой

### Принцип 1: Контекст — это всё
Чистый, актуальный, документированный контекст — основа скорости.

### Принцип 2: Разделение по задачам (агентам/чатам)
Разные задачи = разные сессии с разными промптами.

### Принцип 3: Верификация критичных решений
Архитектура, выбор технологий — прогонять через 2-3 разные модели.

### Принцип 4: Виктор — архитектор, AI — исполнитель
AI пишет код как джуниор: быстро, но требует ТЗ и проверки.

## Workflow команды: кто что делает

**Обновлено 11.12.2025:** С подключением MCP-серверов Claude Code получил "руки" — теперь сам работает с n8n и Supabase.

### Распределение задач

| Задача | Кто делает | Инструмент |
|--------|------------|------------|
| Создать/редактировать n8n workflow | Claude | MCP n8n-mcp |
| SQL запросы, таблицы, миграции | Claude | MCP supabase |
| Настройки в Google Cloud Console | Claude for Chrome | Браузер |
| Настройки в Supabase Dashboard (GUI) | Claude for Chrome | Браузер |
| API keys, credentials | Виктор | KeePass → env |
| Production deploy approve | Виктор | Review |
| Архитектурные решения | Виктор + Claude | Обсуждение |

### Как это работает

```
Виктор: "Создай workflow для транскрибации"
Claude: *через MCP создаёт workflow, тестирует, валидирует*
Claude: "Готово. Проверь: [ссылка]. Нужен твой approve для активации"
```

### Когда звать Виктора

1. **API keys / токены** — хранятся в KeePass, Claude не имеет доступа
2. **Production deploy** — финальное "да" перед релизом
3. **GUI-задачи** — если Claude Code не может (тогда → Claude for Chrome)
4. **Архитектурные решения** — обсуждаем вместе

## Инструменты работы

| Инструмент | Где работает | Когда использовать |
|------------|--------------|-------------------|
| **Claude Chat** | claude.ai | Брейншторм, стратегия, быстрые консультации |
| **Claude Code** | VS Code | Код, файлы, спринты, MCP, субагенты — всё |
| **Claude for Chrome** | Браузер | GUI-задачи, автономные действия в вебе |

> **Примечание:** Есть также терминальная версия Claude Code (`claude` в терминале) — функционально идентична VS Code Extension, отличается только интерфейсом.

### Claude Chat (claude.ai / Projects)

**Что видит:**
- Приложенные файлы — целиком
- Project Knowledge — только фрагменты (~500 символов)
- System prompt проекта

**Что НЕ видит:** полные файлы без приложения, структуру проекта, чеклисты

**Где хорош:**

| Задача | Почему |
|--------|--------|
| Обсуждение стратегии | Не нужен весь контекст |
| Генерация текстов, промптов | Творческая задача |
| Ресёрч (web search) | Ищет внешнюю информацию |
| Быстрые консультации | "Как лучше сделать X?" |

**Где плох:** ведение по спринту, работа с файлами, координация между файлами, чеклисты

**Правило работы:**
- Нужен полный контекст файла → **приложить к сообщению**
- Нужна работа с файлами проекта → **Claude Code**
- Нужна настройка в браузере → **Chrome Extension**
- Chat = **консультант, которому показали документы**, не CTO с полной картиной

---

### Claude Code (VS Code)

**Что видит:**
- Файловая система проекта — полностью (читает файлы целиком, до 2000 строк)
- CLAUDE.md — все инструкции
- Git состояние, структура папок

**Что умеет:**
- Читать/писать/редактировать файлы
- Запускать bash команды, устанавливать программы
- Работать с MCP серверами (n8n, supabase, context7)
- Делегировать субагентам

**Где хорош:**

| Задача | Почему |
|--------|--------|
| Ведение по спринту/чеклистам | Вижу весь файл, отмечаю ✅ |
| Редактирование файлов | Прямой доступ |
| Работа с кодом | Пишу, тестирую, деплою |
| CTO с полным контекстом | Читаю все файлы вместе |
| Интеграции через MCP | n8n, Supabase напрямую |
| Системные задачи | Bash, ffmpeg, установка программ |

**Где плох:** настройка GUI в браузере, визуальные задачи, действия в веб-интерфейсах

---

### Claude for Chrome

**Что видит:** текущую страницу, DOM элементы, может делать скриншоты

**Что умеет:** кликать, вводить текст, навигация, автономные действия

**Где хорош:**

| Задача | Почему |
|--------|--------|
| Настройка в Google Cloud Console | Видит GUI, кликает |
| Настройка в Supabase Dashboard | Видит GUI, кликает |
| Research в браузере | Переходит по ссылкам |
| Автономные действия в вебе | Выполняет цепочки |

**Где плох:** работа с файлами, написание кода, терминал

## Агенты по направлениям

### svaib-lab (внутренняя мастерская)

#### AI-стратег
**Задача:** Помогает с планированием и стратегическими решениями по проекту.
**Когда использовать:** При важных решениях, планировании квартала, пивотах.

#### AI-HRD
**Задача:** Помогает в создании эффективных промптов для AI-сотрудников.
**Когда использовать:** Когда надо создать промпт для нового AI-сотрудника или подредактировать другой.

#### AI-редактор контекстов (субагент)
**Задача:** Превращает черновые тексты в структурированные файлы контекста проекта. Проверяет связанные файлы на коллизии.
**Когда использовать:** При создании или обновлении любых файлов контекста.

**Workflow:**
1. Вызов `/svaib-context` → загружается координатор (партнёр)
2. Обсуждение задачи с Виктором → понимание ЧТО нужно сделать
3. Делегирование субагенту `context-editor` → исполнение по чеклисту
4. Субагент возвращает результат → координатор проверяет и показывает

**Компоненты:**
- Команда: `.claude/commands/svaib-context.md`
- Субагент: `.claude/agents/context-editor.md`
- Правила: `meta/meta_context/!context_rules.md`

#### AI-редактор недельных итогов
**Задача:** Превращает черновик недельных итогов в структурированный weekly_progress.md.
**Когда использовать:** Каждую неделю после публикации итогов.

### svaib-dev (продукт)

#### AI-CTO архитектор
**Задача:** Проектирует архитектуру AI-решений, определяет технологический стек, планирует этапы реализации.
**Когда использовать:** При проектировании новых фич, изменении архитектуры, формировании этапов работы.

#### AI-эксперт Dify
**Задача:** Помогает настраивать и использовать Dify для создания AI-инструментов и клиентских решений.
**Когда использовать:** При любых вопросах по работе с платформой Dify.

#### n8n-expert (субагент) — планируется
**Задача:** Создание n8n workflow, webhooks, интеграции.
**Когда использовать:** При любых задачах с workflow и автоматизацией.
**Статус:** Пока не создан. С MCP n8n-mcp основная работа уже возможна без отдельного субагента.

#### supabase-expert (субагент) — планируется
**Задача:** Работа с БД, таблицы, RLS, миграции, SQL.
**Когда использовать:** При работе с базой данных.
**Статус:** Пока не создан. С MCP supabase основная работа уже возможна без отдельного субагента.

### svaib-pub (публикации)

#### AI-стратег выступлений
**Задача:** Помогает готовиться к публичным выступлениям и воркшопам на уровне стратегии и идей.
**Когда использовать:** Для подготовки к выступлениям на этапе стратегии (не копирайтинг/дизайн).

#### AI-арт-директор
**Задача:** Превращает идею слайда в промпт для AI-иллюстратора. Стиль: Bauhaus + Matisse + Kandinsky.
**Когда использовать:** При создании визуала для слайдов презентаций.

#### AI-контент-партнер
**Задача:** Создаёт содержание слайдов: заголовки, тезисы, метафоры, концепции визуализации.
**Когда использовать:** При работе над контентом презентаций от идеи до готовых слайдов.

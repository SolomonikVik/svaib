# Brand — PDF-документы

## Кратко

Как делаем PDF-документы svaib: принципы, шрифты, элементы страницы, что работает и что нет. Выводы из создания коммерческих предложений и клиентских документов (февраль–март 2026). Палитра и философия — в [brand.md](brand.md).

## Общий принцип

**Минимализм с акцентами.** Белый фон, чистая типографика, цвет — только для навигации и акцентов. Не раскрашивать всё подряд.

## Шрифты

**Основная пара: Montserrat + Roboto** (единая с презентациями, утверждено март 2026).

| Элемент | Шрифт | Размер |
|---------|-------|--------|
| Логотип "svaib" | Montserrat Bold | 18px |
| Заголовок документа (оффер) | Montserrat Bold | 28pt |
| Заголовок документа (многостраничный) | Montserrat Bold | 16pt |
| Заголовок секции (h2) | Montserrat Bold | 15pt |
| Подзаголовок внутри bento-блоков | Montserrat Bold | 11pt |
| Основной текст | Roboto Regular | 10pt |
| Цены | Montserrat Bold | 26pt |
| Мелкий текст, подписи | Roboto | 8-9pt |
| Футер | Roboto 7.5pt + Montserrat 9pt (логотип) |

**Montserrat** поддерживает кириллицу — нет ограничений как было с Sora. Подключается через Google Fonts в HTML.

## Палитра для PDF

Подмножество из [brand.md](brand.md), с ограничениями:

| Элемент | Цвет | HEX |
|---------|------|-----|
| Основной текст | Тёмно-серый | `#1A1A1A` |
| Вторичный текст | Серый | `#6B7280` |
| Третичный текст | Светло-серый | `#9CA3AF` |
| Акцент (структура) | Teal | `#00B4A6` |
| Акцент (логотип) | Pink | `#FF4D8D` |
| Фон | Белый | `#FFFFFF` |
| Разделители | Бордер | `#E5E7EB` |

**Правила:**
- Teal для структуры (полоски, рамки, тире)
- Dark Teal `#008B7F` для заголовков секций (h2) — создаёт визуальную иерархию
- Pink — логотип (точка) + слово "AI" в заголовке документа (правило 60/30/10)
- Максимум 2 цветовых акцента на страницу

## Элементы страницы

### Что работает

**Header:**
- Тонкая teal-полоска вверху (2 мм) — маркер бренда
- Логотип svaib (Montserrat Bold, teal) + розовая точка — компактный
- Дата справа вверху, мелким шрифтом

**Footer:**
- Логотип + svaib.com слева, имя автора справа
- Разделены тонкой линией

**Секции:**
- Teal-полоска слева от заголовка (3px) — навигация без заливок
- Заголовки: Montserrat Bold 15pt, цвет Dark Teal `#008B7F`
- Между секциями — тонкий разделитель (`#E5E7EB`, 0.5px)

**Feature-box:**
- Полоска слева (3pt, teal) + светлый фон `#F0FDFB` — для выделения ключевой информации (формат, условия)

**Bento-блок:**
- Объединение логически связанных секций в один блок с общей рамкой (`#E5E7EB`, radius 6pt)
- Внутри — подзаголовки, разделитель между частями
- Пример: «Рыночная рамка» + «Early adopter rate» в одном блоке

**Карточки:**
- Рамка без заливки фона
- Основной вариант — teal border (1.5px), альтернативный — gray border (1px)
- Цена крупным шрифтом (Montserrat Bold 26pt), детали мелким (Roboto 9pt)

**Info-карточки (market-card):**
- Светлый фон `#F0FDFB`, radius 4pt
- Label uppercase мелким шрифтом, value крупным Montserrat Bold
- Для ключевых цифр и метрик

**Таблицы:**
- Teal-линия сверху (1.5px), gray-линия снизу (0.5px)
- Без фоновых заливок строк

### Что НЕ работает

- **Цветные фоны** на карточках (teal, pink, light) — выглядит как "трэш"
- **Розовые элементы** — подчёркивания, badges, карточки → перебор
- **Градиенты** — не для документов
- **Фон страницы ≠ белый** — визуальный шум
- **Больше 2 цветовых акцентов** на страницу — перегруз

## Многостраничные документы

Протоколы, инсайты, письма, обзоры — 2-5 страниц. Эталон: [_examples/template-multipage.html](_examples/template-multipage.html).

### Отличия от одностраничного оффера

| Параметр | Оффер (1 стр.) | Многостраничный |
|----------|---------------|-----------------|
| Футер `bottom` | `0` | **`-12mm`** (уходит в page margin) |
| Заголовки | — | `break-after: avoid` |
| Teal-полоска | `header-bar` (edge-to-edge, 3mm) | `border-top` на `.header` (2mm) |
| Заголовок документа | 28pt | 16pt |
| `@page margin` | 1.8cm / 1.5cm | 20mm / 22mm / 18mm |
| Разрыв страницы | — | `.page-break { page-break-before: always; }` |

### Антипаттерны

| Ошибка | Почему плохо | Что делать |
|--------|-------------|-----------|
| Футер `bottom: 0` на многостраничном | Наезжает на контент | `bottom: -12mm` |
| Писать CSS футера с нуля | Каждый раз ошибки | Копировать из шаблона |
| Нет `break-after: avoid` на заголовках | Заголовок висит внизу страницы без контента | Добавить на h2, h3, .info-label, .person-name |
| `@page { @bottom-center }` для футера | Не поддерживает HTML-разметку (цветной логотип) | `position: fixed` + `bottom: -12mm` |
| Изобретать футер вместо copy-paste | 20 минут вместо 3 | Открыть шаблон, скопировать |

### Протокол создания

1. Скопировать [_examples/template-multipage.html](_examples/template-multipage.html)
2. Заменить контент (заголовок, секции, списки)
3. `weasyprint input.html output.pdf`
4. Открыть PDF, проверить по чеклисту ниже
5. Если ок — удалить HTML, оставить PDF

## Чеклист перед финализацией

### Общий (все PDF)
- [ ] Заголовки в Montserrat Bold, текст в Roboto
- [ ] Заголовки секций — Dark Teal `#008B7F`
- [ ] Розовый — только в логотипе (точка) и слове "AI" в заголовке
- [ ] Нет фоновых заливок (кроме белого и `#F0FDFB` в info-карточках/feature-box)
- [ ] Максимум 2 цветовых акцента на страницу
- [ ] Визуальная проверка: открыть PDF

### Дополнительно для многостраничных
- [ ] Футер не наезжает на текст (на каждой странице!)
- [ ] Заголовки не висят одиноко внизу страницы
- [ ] Принудительные разрывы где нужно (`.page-break`)

### Дополнительно для офферов
- [ ] Всё на одной странице, ничего не обрезано
- [ ] Карточки — рамка без заливки

## Технические заметки

**Стек:** HTML + CSS → weasyprint CLI → PDF. Ранее использовался ReportLab (ручное позиционирование), заменён в марте 2026.

**Почему weasyprint:**
- CSS flexbox/grid вместо ручных координат (x, y)
- Google Fonts подгружаются через `@import url()` в HTML
- HTML-превью в браузере для быстрой отладки
- Проще поддерживать и менять дизайн

**Шрифты:**
- Montserrat (Regular/SemiBold/Bold) — через Google Fonts. Поддерживает кириллицу
- Roboto (Regular/Medium/Bold) — через Google Fonts
- Локальные TTF (из `dev/assets/fonts/`) — для офлайн-генерации

## Коммуникация

- Язык ответа = язык вопроса (по умолчанию русский)
- Прямо, конкретно, по существу. В сложных вопросах: сначала суть, затем только нужные детали.
- **Ссылки на файлы — всегда кликабельные.** Используй markdown-ссылки: `[file.md:42](path/file.md#L42)`. Никогда не пиши plain text вроде `file.md (line 42)` — это некликабельно в IDE
- **Текст-предложение показывай blockquote (`>`), не code-блоком.** Code-блок (` ``` `) — только для того, что Виктор может захотеть скопировать (код, команды, точные строки конфигов). В IDE-чате code-блоки не переносятся: длинная строка уезжает вправо, видны только первые слова.

## Правила

1. **Verify-first.** Перед решением задачи — собери необходимый контекст - прочитай все необходимые файлы и только когда у тебя есть контекст - приступай к решению! Никогда не решай - не собрав необходимую документацию по задаче. Смотри Проткол чтения! 

2. **Запись текста в файл md.** При записи в файл - краткость. Пишешь для AI и человека одновременно — кратко, по существу, экономя токены, но не теряя смысл.

3. **Текст отдельно от действий.** Если в сообщении есть вопрос к Виктору ИЛИ содержательный ответ — отправляй только текст. Никаких tool calls в том же ходе. Исключение: краткий статус перед действием.

4. **Конфиденциальность клиентов.** Имена клиентов НЕ в git-tracked файлы. Только первая буква или "Клиент 1". Исключение: папка `clients/` (в .gitignore).

5. **Memory — только с разрешения Виктора.** НЕ записывать в auto memory без явного запроса.

6. **Разбор ошибок.** Если Виктор спрашивает "почему так произошло" — запустить субагента `rescue`. Без антропоморфных объяснений — только: где сбой, почему, что починить.

7. **Трекер на план.** Многошаговая задача или план → включай TodoWrite сразу. Не после третьего шага, не когда вспомнишь — сразу.

## Структура и протокол чтения

При старте чата — собери ПОЛНЫЙ контекст!
При начале нового чата прочитай README корневых папок, прочти необходимые README.md, опускайся в необходимые подпапки и вновь читай README.md - определяй файлы, которые необходимо прочитать для решения задачи


| Папка | Назначение |
|-------|-----------|
| `framework/` | Ядро продукта: онтология, методология, scaffold, skills, plugin |
| `knowledge/` | Актуальная память AI: свежие знания + материалы для публикации |
| `dev/` | Сайт svaib.com + техническая инфраструктура |
| `meta/` | Стратегия проекта: management, marketing, product |
| `clients/` | Ранние клиенты, playbook консалтинга (локально) |
| `lab/` | Мастерская AI-помощников: скиллы, агенты, хуки |
| `.claude/` | Технические файлы помощников: роли, навыки, автоматизация |

Каждое направление устроено по универсальной модели: `_inbox/01_inbox` (входящее на разбор) → `02_backlog` (задачи на будущее) → `02_active` (сейчас + Session Handoff для передачи контекста между чатами) → `03_progress` (хроника), опционально `04_decisions`.
 Детали и правила работы — [lab/work-model.md](lab/work-model.md).

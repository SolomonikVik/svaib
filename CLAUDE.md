## Правила

0. **Поток → следующий шаг.** Если запрос Виктора — поток мыслей, а не готовая задача (направление понятно, но неясно что делать сейчас), не бросайся исполнять. Остановись, определи что должно произойти дальше (задача? план? обсуждение? делегация?), предложи Виктору. После ОК — двигайся. Для комплексных задач — сначала обсудить план. Не уверен какой тип — спроси.

1. **Verify-first.** Перед решением задачи — собери необходимый контекст - прочитай необходимые файлы и только когда у тебя есть контекст - приступай к решению! Никогда не решай - не собрав необходимую и достаточную документацию по задаче. Смотри Проткол чтения! 

2. **Не плоди сущности.** Новый элемент, термин или структура — только если нельзя выразить через существующие И без него теряется что-то важное.

3. **Стиль файлов.** Пиши для AI и человека одновременно — кратко, по существу, экономя токены, но не теряя смысл.

4. **Текст отдельно от действий.** Если в сообщении есть вопрос к Виктору ИЛИ содержательный ответ — отправляй только текст. Никаких tool calls в том же ходе. Исключение: краткий статус перед действием.

5. **Конфиденциальность клиентов.** Имена клиентов НЕ в git-tracked файлы. Только первая буква или "Клиент 1". Исключение: папка `clients/` (в .gitignore).

6. **Помни о cutoff date.** Технологии, API, модели — не выдавать как факт если могло измениться. Предложить проверить.

7. **Memory — только с разрешения Виктора.** НЕ записывать в auto memory без явного запроса.

8. **Разбор ошибок.** Если Виктор спрашивает "почему так произошло" — запустить субагента `rescue`. Без антропоморфных объяснений — только: где сбой, почему, что починить.

9. **Трекер на план.** Многошаговая задача или план → включай TodoWrite сразу. Не после третьего шага, не когда вспомнишь — сразу.

## Структура и протокол чтения

Перед решением задачи — определи затронутые папки по таблице, прочитай их README, двигайся по ссылкам внутри.

| Папка | Назначение |
|-------|-----------|
| `framework/` | Ядро продукта: онтология, методология, scaffold, skills, plugin |
| `knowledge/` | Актуальная память AI: свежие знания + материалы для публикации |
| `dev/` | Сайт svaib.com + техническая инфраструктура |
| `meta/` | Стратегия проекта: management, marketing, product |
| `clients/` | Ранние клиенты, playbook консалтинга (локально) |
| `lab/` | Мастерская AI-помощников: скиллы, агенты, хуки |
| `.claude/` | Технические файлы помощников: роли, навыки, автоматизация |

Каждое направление устроено по универсальной модели: `_inbox/01_inbox` (входящее на разбор) → `02_backlog` (задачи на будущее) → `02_active` (сейчас + Session Handoff для передачи контекста между чатами) → `03_progress` (хроника), опционально `04_decisions`. Ядро — всегда, расширения — по нагрузке (Iron Law). Детали и правила работы — [lab/work-model.md](lab/work-model.md).

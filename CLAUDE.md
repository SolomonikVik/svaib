## Правила

0. **Поток → следующий шаг.** Если запрос Виктора — поток мыслей, а не готовая задача (направление понятно, но неясно что делать сейчас), не бросайся исполнять. Остановись, определи что должно произойти дальше (задача? план? обсуждение? делегация?), предложи Виктору. После ОК — двигайся.

1. **Verify-first.** Для инженерных задач — сначала читай файлы, потом действуй. Для комплексных — сначала обсудить план. Не уверен какой тип — спроси.

2. **Не плоди сущности.** Новый элемент, термин или структура — только если нельзя выразить через существующие И без него теряется что-то важное.

3. **Стиль файлов.** Пиши для AI и человека одновременно — кратко, по существу, экономя токены, но не теряя смысл.

4. **Текст отдельно от действий.** Если в сообщении есть вопрос к Виктору ИЛИ содержательный ответ — отправляй только текст. Никаких tool calls в том же ходе. Исключение: краткий статус перед действием.

5. **Конфиденциальность клиентов.** Имена клиентов НЕ в git-tracked файлы. Только первая буква или "Клиент 1". Исключение: папка `clients/` (в .gitignore).

6. **Помни о cutoff date.** Технологии, API, модели — не выдавать как факт если могло измениться. Предложить проверить.

7. **Memory — только с разрешения Виктора.** НЕ записывать в auto memory без явного запроса.

8. **Разбор ошибок.** Если Виктор спрашивает "почему так произошло" — запустить субагента `rescue`. Без антропоморфных объяснений — только: где сбой, почему, что починить.

9. **Трекер на план.** Многошаговая задача или план → включай TodoWrite сразу. Не после третьего шага, не когда вспомнишь — сразу.

## Протокол чтения

Перед работой с файлами — прочитай их. Перед оценкой — прочитай предмет оценки.

1. README.md папки → конкретный файл
2. Markdown-ссылка ≠ автозагрузка. @file в команде — загрузка. Ссылка в тексте — нет
3. Если задача затрагивает механики Claude Code — knowledge/coding/claude-code.md

## Чего не делать по умолчанию

- Не запускай субагентов где можно прочитать файлы самому
- Не выдавай цифры без методики измерения
- Не уходи в plan-first режим на точечной инженерной задаче
- Не рассуждай вместо чтения файлов

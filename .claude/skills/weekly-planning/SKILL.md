# Ритуал планирования недели

Понедельник утром, один раз в неделю. Пять шагов: закрытие → итоги → анализ → план → пост.

## Роль агента

Ты стратегический партнёр, не переносчик строчек. Каждую задачу и итог оценивай через цель декады и метрики экватора. Если неделя ушла не туда — скажи прямо.

## ВЫПОЛНЯЙ СЕЙЧАС

1. Определить номер текущей недели (формула ниже)
2. Загрузить контекст (секция «Подготовка»)
3. Подтвердить с Виктором: "Планёрка: закрываем неделю W(N), планируем W(N+1). Верно?"
4. После подтверждения — выполнять шаги последовательно

**НАЧИНАЙ.**

## Вычисление номера недели

Старт проекта: 16 сентября 2025 (вторник). 5-недельный перерыв после недели 15 (29 дек 2025). Неделя 16 возобновлена 9 февраля 2026 (понедельник).

**Недели 16+:** start = 2026-02-09 + (N-16) * 7 дней

## Шаг 1: Закрытие прошлой недели

Цель: зафиксировать что сделано, обновить план периода, разобраться с хвостами.

1. Пройтись по плану недели W(N): что закрыто (галочки), что нет
2. **Обновить план периода** — по каждому пункту зафиксировать прогресс:
   - Количественные цели — текущее значение в скобках: `(сейчас: 3)`
   - Завершённые — поставить `[x]`
   - Незавершённые — оставить `[ ]`
   - План периода должен быть самодостаточным: глядя на него, всегда видно что сделано и что нет
3. **Разобраться с незакрытыми задачами:**
   - Актуальна на следующую неделю → переносится в план следующей недели с меткой возраста `[W2]`, `[W3]` и т.д.
   - Не актуальна на ближайшую неделю → возвращается в бэклог
   - Задача с меткой `[W3]` — красный флаг: делаем на этой неделе, замораживаем ❄️, или убираем. Обсудить с Виктором
4. Показать Виктору сводку закрытия, получить ОК

## Шаг 2: Итоги недели

Цель: собрать факты, показать Виктору, зафиксировать.

1. Запустить `/weekly-progress` — скилл собирает session-log, посты Telegram (пн–вс по датам), трекер (если есть) и генерирует драфт летописи
2. Показать Виктору сводку: что сделано по направлениям, ключевые события
3. Виктор дополняет / корректирует
4. После одобрения — записать в 04_weekly_progress.md

## Шаг 3: Анализ

Цель: понять, правильно ли движемся.

На основе итогов (шаг 2) и прогресса по плану периода (шаг 1):

1. Как распределено время между направлениями? Соответствует ли фокусу декады?
2. Какие метрики декады двигались, какие стояли?
3. Что вытесняло клиентскую работу (если вытесняло)?
4. Есть ли задачи с меткой `[W3]` и выше?

Сказать прямо, если что-то не так. Виктор решает.

## Шаг 4: План недели

Цель: сформировать план недели W(N+1) от стратегии, а не от инерции.

**Методика — сверху вниз:**

1. **Vision** — перечитать, напомнить себе куда идём вообще
2. **Цель (экватор)** — осознать общее направление, не просто "прочитал"
3. **Цель декады** — понять конкретный фокус текущего периода

На этом этапе агент чётко понимает КУДА.

4. **От плана периода к задачам** — посмотреть что в плане периода реализовано (шаг 1), что логичнее взять следующим. Проверить:
   - Что есть в бэклоге под эти цели
   - Какие задачи остались с прошлой недели (с метками [W2]+)
   - Есть ли очевидные новые задачи
5. **Предложить Виктору** своё видение плана недели с обоснованием от целей
6. **Обсудить** — Виктор корректирует, вместе фиксируем
7. **Записать** в 03_plan.md

Правило: клиентская работа — всегда первая. Структура плана недели: Клиенты → Продукт → Публикации → Лаборатория.

## Шаг 5: Пост Telegram

Цель: сгенерировать пост «итоги-планы недели» из готовых данных.

1. Из weekly progress (шаг 2) — итоги
2. Из плана недели (шаг 4) — планы
3. Тон: лёгкий, с самоиронией, как в канале @svaib_lab
4. Показать драфт, после одобрения — Виктор публикует сам

## Закрытие

1. Запись в session-log: "Ритуал планирования недели W(N+1)." + длительность. Спросить Виктора сколько заняло
2. Перейти к ритуалу планирования дня (инструкция в конце `meta/management/03_plan.md`)

## Нештатные ситуации

| Ситуация | Действие |
|----------|----------|
| /weekly-progress не может собрать данные | Продолжить без него — собрать вручную из session-log + план недели |
| Виктор хочет пропустить итоги | Перейти сразу к шагу 4 (план), итоги доделать позже |
| Спор по приоритетам | Аргументировать от метрик декады, но решает Виктор |
| Планёрка затягивается | Предложить зафиксировать что есть, детали — в отдельной сессии |

## Связанные скиллы

- **/weekly-progress** — сбор данных и генерация летописи (вызывается на шаге 2)
- **/plan** — быстрое добавление задач в план (можно использовать после планёрки для дополнений)
- **/svaib-strategy** — стратегические обсуждения и ритуал планирования дня
- **/close-session** — закрытие сессии с записью в session-log

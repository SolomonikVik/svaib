# Ритуал планирования недели

Понедельник утром, один раз в неделю. Шесть фаз: закрытие → итоги → анализ → бэклог → план → пост.

## Iron Law

**Каждая фаза = отдельное сообщение. Не начинай следующую фазу без явного «ОК» от Виктора.**

## Принцип

Сформировать план недели, который ведёт к цели декады.

**Iron Law: каждая фаза = отдельное сообщение. Не начинай следующую фазу без «ОК» от Виктора.**

## Фаза 1: Закрытие прошлой недели

Цель: зафиксировать прогресс в плане периода, очистить план недели от задач.

### Алгоритм

1. Пройтись по плану недели W(N). Каждую задачу определить как регулярную (есть в шаблоне) или разовую.

2. **Выполненные задачи (любые):**
   - Удалить из плана недели (структуру и заголовки не трогать)

3. **Невыполненные регулярные:**
   - Удалить из плана недели
   - В шаблоне регулярных задач добавить метку пропуска: `[пропущено W21]` или дополнить существующую `[пропущено W20, W21]`

4. **Невыполненные разовые:**
   - Перенести в бэклог с меткой возраста: без метки → [W1], с [WN] → [W(N+1)]. `[WN]` = сколько раз задача побывала в плане недели и не была выполнена
   - Удалить из плана недели

5. **Обновление плана периода:**
   - На основе выполненных задач — предложить обновления прогресса в плане периода: количественные цели `(сейчас: N)`, завершённые `[x]`
   - НЕ вносить до согласования с Виктором

После шагов 1–5: в плане недели остаётся только структура (заголовки, имена клиентов), задач нет.

### Output фазы 1

Показать Виктору одно сообщение:
1. Предлагаю обновить в плане периода: {конкретные строки с новыми значениями}
2. В бэклог возвращаю: {задачи с метками}

**После «ОК» — обновить план периода, перенести в бэклог, проставить метки пропуска в шаблоне регулярных, очистить задачи из плана недели.**

## Фаза 2: Итоги недели

Цель: собрать факты недели, зафиксировать летопись.

### Алгоритм

1. Спросить Виктора: "Данные трекера за W{N}? Кинь PDF или «пропустить»."
   Дождаться ответа.

2. Запустить субагента-сборщика (Agent tool, `.claude/agents/weekly-collector.md`).
   Передать в промпте: номер недели + путь к PDF трекера (или "трекер не предоставлен").
   Субагент сам берёт даты из 03_plan.md, post_id из 04_weekly_progress.md.
   Возвращает:
   - Структурированные данные: часы по направлениям, ключевые события,
     количество постов, last_post_id (input для фазы 3)
   - Драфт летописи в формате и стиле 04_weekly_progress.md

3. Дополнить драфт из контекста фазы 1 если нужно
   (закрытые задачи, прогресс по плану периода).

4. Показать Виктору драфт летописи.

### Output фазы 2

Драфт летописи + структурированные данные (для фазы 3).

**После «ОК» — записать летопись в 04_weekly_progress.md.**

## Фаза 3: Анализ

Цель: понять, правильно ли движемся к цели декады.

### Алгоритм

Используй структурированные данные из фазы 2 (часы по направлениям)
и обновлённый план периода из фазы 1 (метрики, прогресс):

1. Распределение часов по направлениям vs фокус декады —
   соответствует ли то, на что тратим время, тому, что заявлено как приоритет?

2. Метрики декады: конкретные значения «было → стало»
   (из плана периода, обновлённого в фазе 1). Что двигалось, что стояло.

3. Клиентская работа — вытеснялась ли чем-то?
   (клиенты = приоритет первой декады)

4. Прямой вердикт: «в фокусе» или «ушли не туда, потому что...»

### Output фазы 3

Аналитическая оценка: пункты 1-4 выше. Может быть короткой —
если всё в порядке, достаточно «в фокусе, метрики двигаются».

**После «ОК» — переходим к фазе 4.**

## Фаза 4: Чистка бэклога

Цель: привести бэклог в порядок перед формированием плана недели.

### Алгоритм

1. Пройтись по бэклогу по направлениям (клиенты → продукт → контент → лаб → обучение).
   По каждой задаче:
   - **Актуальна для декады** → оставить
   - **Не актуальна** → предложить удалить
   - **[W3]+** — красный флаг: обратить внимание Виктора,
     задача трижды не выполнена

2. **Приоритизация:** внутри каждого направления — важное наверх.
   Конвенция: порядок строк = приоритет.

### Output фазы 4

Показать Виктору:
1. Предлагаю удалить: {задачи с обоснованием}
2. [W3]+: {задачи} — красный флаг
3. Предлагаю порядок приоритетов (если менялся)

**После «ОК» — обновить бэклог в 03_plan.md.**

## Фаза 5: План недели

Цель: сформировать план недели W(N+1) от стратегии, а не от инерции.

### Алгоритм

Методика — сверху вниз:
1. Напомнить: vision → цель экватора → цель декады (2–3 строки, не пересказ)
2. Посмотреть анализ из фазы 3 — куда корректировать

Сборка плана:
3. **Регулярные задачи** — из шаблона, фильтр по чёт/нечёт W(N+1).
   Встречи → в блок встреч соответствующих дней.
4. **Разовые задачи** — из бэклога сверху вниз + план периода
   (что логичнее взять следующим). Предложить набор с обоснованием от целей.
5. **Lead-клиенты** — перечислить текущих lead-клиентов (из бэклога/плана периода)
   и спросить Виктора по каждому: есть задачи на эту неделю?
6. Собрать единый план W(N+1).
   Структура: Клиенты → Продукт → Контент → Лаборатория.
   Правило: клиентская работа всегда первая.

### Output фазы 5

1. Vision → цель экватора → цель декады (2–3 строки)
2. Драфт плана W(N+1) с обоснованием от целей

Виктор корректирует — агент вносит правки и показывает обновлённый план.
Итерации до финальной версии.

**После согласования финальной версии — записать план недели в 03_plan.md.
Взятые из бэклога задачи удалить из бэклога.**

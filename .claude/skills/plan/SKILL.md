# Plan — быстрый редактор оперативного плана

Добавление, удаление и просмотр задач в 03_plan.md из любого чата. Без стратегии, без советов — сделал и отчитался.

## Iron Law

**Не трогай план периода.** Секция «План периода» — результат стратегического ритуала, не быстрых правок.

## ВЫПОЛНЯЙ СЕЙЧАС

Разбери `$ARGUMENTS` и определи действие + секцию. Если аргументов нет — спроси что сделать.

**НАЧИНАЙ.**

## Core Process

### Phase 1: Понять запрос

Определи из контекста разговора или `$ARGUMENTS`:

| Что определить | Варианты |
|---------------|----------|
| **Действие** | `add` (добавить), `remove` (убрать), `show` (показать) |
| **Секция** | `day` (план дня), `week` (план недели), `backlog` (бэклог), `regular` (регулярные задачи) |
| **Текст задачи** | Для add/remove — что именно |
| **Категория** | Для week/backlog — в какую подсекцию (Клиенты, Продукт, Лаборатория, Публикации, Обучение) |

Если секция или действие неясны — спроси коротко: "В бэклог или в план недели?" / "Добавить или убрать?"

Если категория неясна для backlog/week — определи из контекста задачи:
- Упоминание клиента (АС, КИТ, ЕВ, Ник, Еф) → Клиенты
- Помощники, скиллы, агенты, хуки, arscontexta → Лаборатория
- Scaffold, плагин, методология, онбординг → Продукт
- Посты, канал, семинар, сайт → Публикации
- Курсы, материалы, обучение → Обучение
- Не получается определить → спроси

### Phase 2: Выполнить

Прочитай `meta/management/03_plan.md`.

**add:**
1. Найди нужную секцию
2. Для backlog/week — найди нужную категорию внутри секции
3. Вставь `- [ ] текст задачи` в конец категории (перед следующей категорией или разделителем `---`)
4. Для day → вставь в конец `### Делаем`
5. Для regular → вставь в конец блока регулярных задач
6. Если задача для конкретного клиента в week — найди или создай подсекцию клиента (`**АС:**`, `**ЕВ:**` и т.д.)

**remove:**
1. Найди строку с задачей в нужной секции (поиск по ключевым словам)
2. Если нашёл одну — удали строку
3. Если нашёл несколько похожих — покажи список, спроси какую убрать
4. Если не нашёл — скажи: "Не нашёл такую задачу в {секция}"

**show:**
1. Покажи содержимое запрошенной секции
2. Форматируй компактно — только чекбоксы с текстом

### Phase 3: Отчитаться

Коротко подтверди что сделал:
- add: "Добавил в бэклог (Лаборатория): {задача}"
- remove: "Убрал из плана дня: {задача}"
- show: просто покажи список

## Edge Cases

| Ситуация | Действие |
|----------|----------|
| Нет плана дня на сегодня | Скажи: "Плана дня на сегодня нет. Добавить секцию?" — если да, создай по формату из файла |
| Задача дублирует существующую | Предупреди: "Похожая задача уже есть: {текст}. Всё равно добавить?" |
| Запрос на изменение плана периода | Откажи: "План периода меняется через стратегический ритуал (/svaib-strategy)" |
| Запрос на приоритизацию / переупорядочивание | Откажи: "Приоритизация — через планирование дня (/svaib-strategy)" |
| Несколько задач за раз | Добавь все, отчитайся списком |
| Задача с деталями/ссылкой | Сохрани как есть — детали в строке задачи |

## Примеры

**Вход:** "добавь в бэклог: Python-скрипт нарезки транскриптов"
**Выход:** Добавил в бэклог (Продукт): Python-скрипт нарезки транскриптов

**Вход:** "в план дня добавь — позвонить КИТ"
**Выход:** Добавил в план дня (Делаем): позвонить КИТ

**Вход:** "убери семинар из плана дня"
**Выход:** Убрал из плана дня: Семинар: купить Zoom Pro и настроить

**Вход:** "покажи бэклог"
**Выход:** (показывает текущий бэклог)

**Вход:** /plan добавь в план недели для ЕВ: подготовить отчёт
**Выход:** Добавил в план недели (Клиенты → ЕВ): подготовить отчёт

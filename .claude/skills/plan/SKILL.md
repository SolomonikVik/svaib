# Plan — быстрый редактор оперативного плана

Добавление, удаление и просмотр задач в двух контурах:

- **План дня / недели / регулярные** — в `meta/management/03_plan.md`
- **Бэклог направления** — в `{direction}/02_backlog.md` (отдельный файл на каждое направление)

Без стратегии, без советов — сделал и отчитался.

## Iron Law

**Не трогай план периода.** Секция «План периода» в `03_plan.md` — результат стратегического ритуала, не быстрых правок.

## ВЫПОЛНЯЙ СЕЙЧАС

Разбери `$ARGUMENTS` и определи действие + секцию. Если аргументов нет — спроси что сделать.

**НАЧИНАЙ.**

## Core Process

### Phase 1: Понять запрос

Определи из контекста разговора или `$ARGUMENTS`:

| Что определить | Варианты |
|---------------|----------|
| **Действие** | `add` (добавить), `remove` (убрать), `show` (показать) |
| **Секция** | `day` (план дня), `week` (план недели), `regular` (регулярные задачи) — все в `03_plan.md`. `backlog` — в `{direction}/02_backlog.md` |
| **Направление** (для `backlog`) | `framework`, `lab`, `dev`, `clients`, `marketing` |
| **Текст задачи** | Для add/remove — что именно |
| **Категория** (для `week` и `regular`) | Клиенты, Продукт, Лаборатория, Публикации, Обучение, Прочие |

Если секция или действие неясны — спроси коротко: "В бэклог или в план недели?" / "Добавить или убрать?"

#### Определение направления для backlog

По ключевым словам задачи:

| Триггер в задаче | Направление |
|------------------|-------------|
| Упоминание клиента (АС, КИТ, ЕВ, Ник, Еф), «оркестратор клиента», rollout в workspace клиента | `clients` |
| Помощники, скиллы, агенты, хуки, команды `.claude/`, arscontexta, tooling | `lab` |
| Scaffold, плагин, методология, онбординг, memory, ontology, file_spec, architecture | `framework` |
| Сайт, dev_context, Next.js, Vercel, инфраструктура, деплой, домены | `dev` |
| Посты, канал, семинары, брендбук, Telegram-канал, YouTube, контент | `marketing` |
| Не получается определить | Спроси Виктора: "В чей бэклог — framework/lab/dev/clients/marketing?" |

#### Определение категории для week/regular

По тем же ключевым словам, но в терминах `03_plan.md` (секции плана недели и регулярных задач):

- Клиент в задаче → `Клиенты`
- Скиллы, помощники → `Лаборатория`
- Scaffold, плагин → `Продукт`
- Пост, семинар → `Публикации`
- Курсы, обучение → `Обучение`
- Задача не-svaib → `Прочие`
- Неясно → спроси

### Phase 2: Выполнить

**Файл-приёмник:**
- `day`, `week`, `regular` → `meta/management/03_plan.md`
- `backlog {direction}` → `{direction}/02_backlog.md`

Прочитай нужный файл.

**add:**

1. Для `backlog`:
   - Найди подсекцию по смыслу задачи (например: в `framework/02_backlog.md` есть секции `## Scaffold`, `## Meeting analysis`, `## Онтология / Memory / file_spec`, `## Архитектура и форматы данных`, `## Плагин / runtime / инфраструктура`)
   - Если подходящая подсекция есть — вставь `- [ ] текст задачи` в конец подсекции
   - Если подходящей нет — спроси Виктора: "В какую подсекцию? Существующие: ... Или создать новую?"
   - Создание новой подсекции — по шаблону: `## Название подсекции` + пустая строка + пункт
   - После записи обнови `updated:` в YAML `02_backlog.md`

2. Для `week`:
   - Найди подкатегорию (Клиенты/Продукт/Лаборатория/Публикации/Обучение/Прочие)
   - Для клиентской задачи — найди или создай подсекцию клиента (`**АС:**`, `**ЕВ:**`)
   - Вставь `- [ ] текст задачи` в конец подкатегории

3. Для `day`:
   - Вставь `- [ ] текст задачи` в конец `### Делаем`

4. Для `regular`:
   - Найди подкатегорию (Клиенты / Прочие / Meta / Контент / Обучение)
   - Вставь строку по формату регулярных (см. раздел «Регулярные задачи» в `03_plan.md`)

5. Для `03_plan.md`-секций обнови `updated:` в YAML.

**remove:**

1. Найди строку с задачей в нужной секции/подсекции (поиск по ключевым словам)
2. Нашёл одну — удали строку
3. Нашёл несколько похожих — покажи список, спроси какую убрать
4. Не нашёл — скажи: "Не нашёл такую задачу в {секция}"
5. Обнови `updated:` в YAML затронутого файла

**show:**

1. Покажи содержимое запрошенной секции (форматируй компактно — чекбоксы с текстом + подсекции)
2. Для `backlog` без направления — спроси: "Чей бэклог показать?" или покажи список всех пяти направлений (по заголовкам подсекций, без пунктов)

### Phase 3: Отчитаться

Коротко подтверди что сделал:
- `add backlog framework`: "Добавил в framework/02_backlog.md (Scaffold): {задача}"
- `add week`: "Добавил в план недели (Клиенты → ЕВ): {задача}"
- `remove day`: "Убрал из плана дня: {задача}"
- `show backlog lab`: просто покажи список

## Edge Cases

| Ситуация | Действие |
|----------|----------|
| Нет плана дня на сегодня | Скажи: "Плана дня на сегодня нет. Добавить секцию?" — если да, создай по формату из файла |
| Задача дублирует существующую | Предупреди: "Похожая задача уже есть: {текст}. Всё равно добавить?" |
| Запрос на изменение плана периода | Откажи: "План периода меняется через стратегический ритуал (/svaib-strategy)" |
| Запрос на приоритизацию / переупорядочивание | Откажи: "Приоритизация — через планирование дня (/svaib-strategy)" |
| Несколько задач за раз | Добавь все, отчитайся списком |
| Задача с деталями/ссылкой | Сохрани как есть — детали в строке задачи |
| `backlog {direction}` — файла нет | Сообщи: "В {direction}/ нет `02_backlog.md`. По Iron Law расширение создаётся при наличии контента — создать сейчас?" — если ОК, создай по шаблону file_spec (YAML + Кратко + Работа с файлом + Связанные + первая подсекция) |
| Задача про обучение / knowledge | У `knowledge/` нет `02_backlog.md`. Если задача — разобрать материал, то → `knowledge/_inbox/01_inbox.md`. Если конкретная задача развития направления — спроси Виктора куда (возможно, в `meta/_inbox/01_inbox.md`) |
| Пользователь говорит «бэклог» без направления | Если по ключевым словам направление однозначно — используй его; если нет — спроси |

## Примеры

**Вход:** "добавь в бэклог: Python-скрипт нарезки транскриптов"
**Маппинг:** скрипт нарезки транскриптов → meeting-analysis → `framework`
**Выход:** Добавил в framework/02_backlog.md (Meeting analysis): Python-скрипт нарезки транскриптов

**Вход:** "в план дня добавь — позвонить КИТ"
**Выход:** Добавил в план дня (Делаем): позвонить КИТ

**Вход:** "убери семинар из плана дня"
**Выход:** Убрал из плана дня: Семинар: купить Zoom Pro и настроить

**Вход:** "покажи бэклог lab"
**Выход:** (показывает lab/02_backlog.md)

**Вход:** /plan добавь в план недели для ЕВ: подготовить отчёт
**Выход:** Добавил в план недели (Клиенты → ЕВ): подготовить отчёт

**Вход:** /plan добавь в бэклог: починить брендбук v2
**Маппинг:** брендбук → `marketing` → подсекция «Бренд — переход на v2»
**Выход:** Добавил в meta/marketing/02_backlog.md (Бренд — переход на v2): починить брендбук v2

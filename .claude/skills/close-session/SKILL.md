# Close Session

Структурированное завершение сессии.

## Iron Law

**Не пиши без подтверждения Виктора.** Каждый шаг — диалог. Покажи → жди → записывай.

## Шаги

### 0. Определи контекст

1. **Direction** — какое направление у сессии. Не очевидно → спроси.
2. **Статус задачи** — завершена или продолжается. Не очевидно → спроси.
3. **Создай TodoWrite** со всеми шагами этой сессии закрытия — так пропуск шага виден:
   - Завершена → шаги 1–6.
   - Продолжается → шаги 1, 2, 3, 5 (без 03_progress и коммита).

### 1. Несохранённое

1. **Инсайты разговора** — пробегись по сессии: инсайты, решения, новые термины, скорректированные предположения. Для каждого — было ли записано в файл проекта? Незаписанное → предложи конкретный файл + формулировку. Жди ответ по каждому.
2. **Индекс `_inbox`** — если в сессии создавались или менялись файлы в `_inbox/`, проверь: есть ли на них строка в `_inbox/01_inbox.md`? Нет → предложи добавить.
3. **Временные файлы** — `.claude/plans/`, черновики: предложи удалить ненужное.

### 2. Открытые вопросы

Нерешённые вопросы и отложенные решения:
- Требует возврата → `{direction}/02_active.md#Открытые вопросы`
- Задача на будущее → `{direction}/02_backlog.md`
- Материал на разбор → `{direction}/_inbox/01_inbox.md`
- Решение с обоснованием → `{direction}/04_decisions.md`

Покажи предложения → жди подтверждение → записывай.

### 3. Session-log

Запись в `meta/management/session-log.md`.

- Получи время: `date "+%H:%M"` через Bash.
- Формат: `**HH:MM | направление** — Что сделано. Что узнали. (длительность)`
- Объём: 1–2 фразы. Только суть результата, не процесс.

**Задача завершена:** записать.
**Задача продолжается:** спросить "заносить в session-log?" — если да, записать.

Покажи Виктору blockquote-ом (`>`). После подтверждения — вставь под сегодняшней датой (`## YYYY-MM-DD`); нет секции с сегодня — создай. Обнови `updated:` в frontmatter.

### 4. 03_progress

**Только если задача завершена.**

Запись в `{direction}/03_progress.md`: `## YYYY-MM-DD · Заголовок` + 1–3 строки (результат, ссылка на артефакт).

Покажи → подтвердить → вставь сверху. Обнови `updated:` в frontmatter.

### 5. Handoff

**Задача завершена:**
- Убери завершённую задачу из `{direction}/02_active.md#Активные задачи`.
- Очисти `## Session Handoff` → замени на `_Нет активной задачи._`.

**Задача продолжается:**
- Обнови `## Session Handoff`: на чём остановились, что проверить следующему чату, ключевые ссылки. Перезаписывай, не наращивай. Не ссылайся на `.claude/plans/` — копируй содержимое.

Покажи → подтвердить → записывай. Обнови `updated:` в frontmatter `02_active.md`.

### 6. Коммит

**Только если задача завершена.**

1. Собери список **только тех файлов, что ты сам менял в этой сессии**. Не свои изменения (правки Виктора, других сессий, Codex) в коммит не включай — даже если они видны в `git status`.
2. **Никогда не коммить** операционные файлы Виктора — он сам решает, когда:
   - `meta/management/03_plan.md` — его план дня/недели
   - `meta/management/session-log.md` — даже если ты дописал туда строку в шаге 3
3. Покажи Виктору список + суть изменений.
4. Проверь: нет имён людей, названий компаний, ФИО, персональных данных. Если есть — замени на инициал или обобщение, покажи правку Виктору.
5. Жди подтверждения.
6. После ОК — коммить. Сообщение — одна строка-subject по смыслу изменения, не "закрытие сессии". Тело — только если subject не передаёт «зачем» (нестандартный фикс, ссылка на инцидент).

## Завершение

После всех шагов — 1–2 предложения. Что сделано, что дальше. Поблагодари за работу. Искренне, по-партнёрски.

Последняя строка сообщения — маркер `✅ ✅` отдельной строкой. Сигнал Виктору: сессия закрыта, чат можно закрывать. Ставится только когда пройдены все шаги.

## Частые случаи

- **Несколько направлений** → отдельная запись session-log для каждого; `03_progress` только там, где задача завершена.

# Close Session

Структурированное завершение сессии: session-log, доменный progress, Session Handoff, проверка несохранённого, маршрутизация вопросов, коммит.

Модель работы и контракт подробно — в [lab/work-model.md](../../../lab/work-model.md).

## Iron Law

**Не пиши без подтверждения Виктора.** Каждая фаза заканчивается показом предложения и ожиданием реакции. Фазы 1, 2, 3, 4 — это диалог, не автоматизация.

## Core Process

### Phase 0: Классификация

Задай две оси:

1. **Задача завершена?** Задача, которой была посвящена сессия, полностью закрыта?
   - Признаки: Виктор явно говорит "готово / закрываем", результат на месте, дальше продолжать нечего.
   - Не завершена: "продолжим в новом чате", контекст на пределе, ещё есть шаги.

2. **Сессия значимая?** Была ли это большая работа — важный кусок (обычно от ~40 минут), инсайты, заметные сдвиги?
   - Короткая/техническая: быстрый фикс, обсуждение без артефактов, 10-20 минут без результата.

**Три маршрута:**

| Задача завершена? | Сессия значимая? | Маршрут |
|---|---|---|
| ✅ Да | — | **Phase 1A** (session-log + 03_progress + очистка Session Handoff) |
| ❌ Нет | ✅ Да | **Phase 1B** (session-log + обновление Session Handoff) |
| ❌ Нет | ❌ Нет | **Phase 1C** (только обновление Session Handoff) |

Не очевидно — спроси: "Задача завершена или продолжим? Сессия значимая или техническая?"

После Phase 1* — **всегда Phase 2 → Phase 3 → Phase 4**.

### Phase 1A: Задача завершена

**1. Определи направление** по активной команде или контексту разговора: `lab`, `knowledge`, `framework`, `dev`, `clients`, `meta`, `marketing`, `content`.

**2. Запись в session-log.**

- Получи текущее время: `date "+%H:%M"` через Bash
- Прочитай первые 25 строк `meta/management/session-log.md` для понимания формата
- Формат: `**HH:MM | направление** — Что сделано. Что узнали. (длительность)`
- Длительность — если Виктор указал, в конце: `(55 мин)`, `(1ч 15 мин)`
- Покажи Виктору в чате → подтверждение → запиши под сегодняшнюю дату
- Секции с сегодняшней датой нет — создай `## YYYY-MM-DD` после `---`
- Обнови `updated:` в YAML

**3. Запись в `{direction}/03_progress.md`.**

- Прочитай файл
- Формат: `## YYYY-MM-DD · Короткий заголовок` + 1-3 строки (что сделано, ссылка на артефакт, коммит при наличии)
- Запись — о **результате в домене** (что работает теперь, что готово), не о процессе работы с Claude
- Покажи Виктору → подтверждение → вставь сверху (после заголовка, перед предыдущей записью)
- Обнови `updated:` в YAML

**4. Очистка Session Handoff в `{direction}/02_active.md`.**

- Прочитай файл
- Если блок `## Session Handoff` содержит контекст по этой завершённой задаче — замени содержимое на краткое `_Нет активной задачи._` (или удали строки контекста, оставив заголовок) либо подготовь новый Handoff, если у Виктора следующая активная задача
- Если в `02_active.md#Активные задачи` была строка про завершённую задачу — предложи убрать
- Покажи Виктору → подтверждение → запиши
- Обнови `updated:` в YAML

### Phase 1B: Продолжение, сессия значимая

**1. Запись в session-log** — шаги 1-2 как в Phase 1A. Но: НЕ добавляй запись в `03_progress` (задача не завершена — результата в домене ещё нет).

**2. Обновление Session Handoff в `{direction}/02_active.md`.**

- Прочитай файл
- Обнови блок `## Session Handoff` (сверху): на чём остановились, что проверить следующему чату, ключевые ссылки. Перезаписывай, не наращивай — это состояние, не хроника
- Не ссылайся на эфемерные файлы (`.claude/plans/`) — скопируй содержимое плана в Handoff
- Покажи Виктору → подтверждение → запиши
- Обнови `updated:` в YAML

### Phase 1C: Продолжение, сессия короткая

Только обновление Session Handoff — шаг как в Phase 1B шаг 2. В session-log не пишем.

### Phase 2: Проверка несохранённого

1. Пройдись по разговору: инсайты, решения, находки, новые термины, скорректированные предположения.
2. Для каждого: было ли записано в файл проекта во время сессии?
3. **Проверь `_inbox/` папки.** Если в сессии был создан или изменён файл в `_inbox/` какого-то направления — проверь, есть ли соответствующая строка в `_inbox/01_inbox.md` этого направления. Нет → предложи добавить.
4. Незаписанные → предложи Виктору конкретный файл + формулировку.
5. Проверь, не остались ли временные файлы после сессии (`.claude/plans/`, черновики) — предложи удалить ненужное.
6. **СТОП.** Жди ответ по каждому пункту. Подтвердил → запиши/удали. Отклонил → пропусти.
7. Если всё записано: "Всё из сессии зафиксировано, ничего не потерялось."

### Phase 3: Маршрутизация открытых вопросов

1. Найди нерешённые вопросы, отложенные решения, пометки "потом".
2. Маршрут по правилам [lab/work-model.md](../../../lab/work-model.md):
   - Вопрос направления, требует возврата → `{direction}/02_active.md#Открытые вопросы`
   - Материал на разбор → `{direction}/_inbox/01_inbox.md`
   - Задача на будущее → `{direction}/02_backlog.md` (через `/plan` или напрямую)
   - Решение принять (с обоснованием) → `{direction}/04_decisions.md`
3. Покажи предложенные записи Виктору.
4. **СТОП.** Жди подтверждение по каждому. Подтвердил → запиши в нужный файл.
5. Если открытых вопросов нет: "Открытых вопросов нет." → завершай.

### Phase 4: Коммит

Если задача завершена (Phase 1A) и есть незакоммиченные изменения — предложи коммит. Покажи что войдёт (кратко). Подтвердил → коммить. Отклонил → пропусти.

## Завершающая нота

После всех фаз — заверши сессию коротко и тепло (1-2 предложения). Это момент благодарности: Виктор переходит из режима «делаю» в «сделал». Не шаблон — искренне, по-партнёрски, исходя из того, что реально произошло в сессии.

## Edge Cases

| Ситуация | Действие |
|----------|----------|
| Ничего значимого не произошло и задача не завершена | Phase 1C (только Handoff, если есть что передавать) или пропусти Phase 1 вовсе — спроси Виктора |
| Не удаётся определить направление | Спроси: "Какое направление у этой сессии?" |
| Несколько направлений в одной сессии | Отдельная запись в session-log для каждого; `03_progress` — только в то направление, где задача завершена |
| Виктор говорит "быстро закрываем" | Только Phase 1*, пропусти 2 и 3 |
| session-log.md не существует | Создай с YAML-заголовком + первую запись |
| `02_active.md` направления не содержит Session Handoff блока | Добавь блок `## Session Handoff` сверху, перед существующими секциями |
| `03_progress.md` направления не существует | Создай по шаблону `file_spec.md` (YAML + Кратко + Работа с файлом + Связанные) |
| Задача трогала несколько направлений, завершена в одном | `03_progress` в то направление, где результат; session-log одна запись с составным направлением (`framework+lab`) — по образцу существующих записей |

# Пуш + сводка для Telegram

## Когда

Виктор говорит "пушим", "собери сводку", "финал".

## Процесс

1. `git log --oneline @{push}..HEAD -- knowledge/` — собрать все коммиты с последнего пуша
2. Для каждого крупного коммита — зайти в файл, прочитать секцию "Кратко", чтобы описание в посте было осмысленным (НЕ просто переписывай коммит-сообщение)
3. `git diff <первый-коммит>^..HEAD --stat -- knowledge/` — посчитать статистику (файлы, строки)
4. Собрать пост по шаблону (см. ниже)
5. Записать в `_inbox/tg-post-<тема>.md`
6. Показать Виктору на утверждение
7. После ОК — `git push`

## Шаблон Telegram-поста

```
**Заголовок**
Подзаголовок — одна фраза человечным языком

**Добавили с Клодом N тем:**

📌 **Название темы**
Описание в 1-2 предложения простым языком. Автор/источник если важен. Подробности → <bare URL на GitHub>

📌 **Следующая тема**
Описание. Подробности → <bare URL>

Также обновлено:
— Краткая правка раз
— Краткая правка два

__Затронуто X файлов, +Y / −Z строк.__
```

### Пример (реальный пост от 2026-03-01)

```
**Обновление Knowledge SVAIB**
Открытая база знаний для солопренеров, которые (как и я) пробуют строить бизнес с AI.

**Добавили с Клодом 4 темы:**

📌 **Schema-Guided Reasoning**
Как заставить AI-агентов рассуждать предсказуемо: три паттерна (последовательность, ветвление, повторение), которые через Pydantic-схемы физически не дают модели отвечать мимо структуры. Автор подхода — Ренат Абдулин (@llm_under_hood). Подробности → https://github.com/SolomonikVik/svaib/blob/main/knowledge/agents/sgr.md

📌 **AI-Ready Architecture**
Как писать код, чтобы AI-агент не ломал то, чего не видит: убираем скрытые цепочки действий, делаем всё явным и изолированным. Подробности → https://github.com/SolomonikVik/svaib/blob/main/knowledge/coding/ai-ready-architecture.md

Также обновлено:

— Каталог плагинов Anthropic: 15 + 4 партнёрских (было 11)

__Затронуто 15 файлов, +505 / −39 строк.__
```

## Правила содержания

**Сортировка тем:**
- Крупные новые темы (новый файл, большое исследование) → 📌 в основной блок
- Мелкие обновления (новая фича в существующем инструменте, +3 строки) → "Также обновлено"
- Если обновлений мало и они неинтересные — секцию "Также обновлено" можно не делать

**Описания тем:**
- Пиши для аудитории без технического бэкграунда (солопренеры, руководители)
- Не копируй жаргон из файла — перескажи суть простым языком
- Если не понимаешь суть сам — зайди в файл и прочитай
- Автора/источник указывай если он известен в сообществе

**URL ссылок:**
- Формат: `https://github.com/SolomonikVik/svaib/blob/main/knowledge/<путь-к-файлу>`
- URL ставится после "Подробности →" как bare URL (не markdown-ссылка)

## Telegram-форматирование

Пост копируется из файла и вставляется в Telegram Desktop. Что работает при вставке:

| Синтаксис | Результат | Работает? |
|-----------|-----------|-----------|
| `**текст**` | жирный | Да |
| `__текст__` | курсив | Да (двойное подчёркивание) |
| `*текст*` | курсив | НЕТ (одна звёздочка не работает) |
| `_текст_` | курсив | НЕТ (одинарное подчёркивание не работает) |
| `[текст](url)` | ссылка | НЕТ (только Bot API) |
| bare URL | кликабельная ссылка | Да (Telegram авто-детектит) |
| `` `код` `` | моноширинный | Да |

**Критично:** Курсив — только `__текст__` (двойное подчёркивание). Ссылки — только bare URL.

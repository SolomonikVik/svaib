# Закрытие сессии knowledge/

## Когда

Виктор говорит "закрываем", "коммить", "заканчиваем". **Решение о коммите принимает только Виктор.** Не предлагай коммит, не спрашивай "коммитим?". Сделал шаг — доложи что сделано, точка.

## Коммит

Закоммитить все изменения в knowledge/ (и только их).

### Как писать сообщение коммита

**Аудитория:** подписчики Telegram-канала про AI. Не разработчики, не технари — руководители малого бизнеса, которые интересуются AI. Из коммит-сообщений потом собирается пост в Telegram. Если читатель канала не поймёт о чём речь — сообщение плохое.

**Правило:** Пиши так, как будто объясняешь другу за кофе, что интересного сегодня узнал. Одно-два предложения. Без терминов, без аббревиатур, без перечислений в скобках.

**Формат:**
```
knowledge: <суть простыми словами, 1-2 предложения> (<ключевой-файл.md>)
```

Имя файла в скобках — обязательно. Оно нужно чату-сборщику (чтобы прочитать файл и обогатить пост) и подписчикам Telegram (ссылка на GitHub). Если изменён только один файл — его имя. Если создан новый + обновлены старые — имя нового.

**Хорошо:**
```
knowledge: как структурировать код чтобы AI-агенты не плодили баги (ai-ready-code.md)
```
```
knowledge: карта настроечных файлов для Claude, Cursor и 11 AI-ассистентов (ai-system-files.md)
```

**Плохо:**
```
knowledge: AI-ready architecture (sinks vs pipes, честные интерфейсы, progressive disclosure) + обновление каталога
```
Почему плохо: "sinks vs pipes", "progressive disclosure" — жаргон. Читатель канала не знает этих слов. Ты подставил ключевые слова из файла вместо того чтобы объяснить суть.

```
knowledge: ai-system-files.md — CLAUDE.md, AGENTS.md, AAIF, best practices
```
Почему плохо: имя файла есть, но суть — набор терминов вместо объяснения. Сначала суть, потом файл в скобках.

**Тест:** Прочитай своё сообщение и спроси себя: если человек без технического бэкграунда это прочитает в Telegram — он поймёт, о чём речь? Если нет — перепиши.

**Не пушить.** Только коммит.

## Пуш + сводка для Telegram

Когда Виктор говорит "пушим" или "собери сводку":

1. `git log --oneline @{push}..HEAD -- knowledge/` — собрать все коммиты с последнего пуша
2. Из коммит-сообщений сгенерировать пост для Telegram: краткий, на русском, для AI-сообщества
3. Показать Виктору на утверждение
4. После ОК — `git push`

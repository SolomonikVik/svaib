# Правила работы с _inbox/

`_inbox/` — буфер входящих задач. Стремится к пустоте.

## Где живут инбоксы

`_inbox/` может быть в любой папке проекта. Нужно передать задачу — положи в `_inbox/` соответствующего направления. Список направлений — в CLAUDE.md (структура проекта). Не знаешь куда — кидай в `meta/_inbox/`.

## Как добавить задачу

Зайди в `_TODO.md` нужного inbox, добавь строку. Запись должна быть самодостаточной — читающий агент должен понять что, зачем и нужно ли ему действовать, не открывая других файлов.

Простая задача (без отдельного файла):

```
- [ ] Разобрать: нужен ли отдельный механизм обмена находками между ролями или достаточно _inbox/
```

Задача с отдельным файлом — ссылка внутри текста:

```
- [ ] В [crossfeed.md](crossfeed.md) лежит идея механизма обмена находками между ролями. Разобрать: нужен отдельный механизм или достаточно _inbox/
```

Сложная задача — подзадачи вложенным списком:

```
- [ ] Основная задача
  - [ ] Подзадача 1
  - [ ] Подзадача 2
```

## Когда нужен отдельный файл

Если задача требует развёрнутого описания — создай файл в той же `_inbox/`, упомяни его ссылкой в строке `_TODO.md` (см. пример выше).

## Как разобрать

Выполнил задачу → **удали строку** из `_TODO.md`. Не ставь галочки `[x]`, не зачёркивай, не помечай "done" — удаляй. Если был отдельный файл — удали и его. Список стремится к пустоте, не к архиву выполненного.

## Worklog (передача контекста между сессиями)

Полная схема: [session-handoff.md](session-handoff.md)

`_worklog.md` живёт в `_inbox/` рядом с `_TODO.md`. Это файл СОСТОЯНИЯ активной задачи, не лог.

- **Worklog ≠ TODO.** TODO — список входящих задач. Worklog — контекст одной активной задачи для передачи между чатами.
- **Worklog перезаписывается**, не растёт. При обновлении — заменяй секции на актуальные.
- **Когда задача завершена** — worklog очищается до шаблона.
- **Правила работы** записаны в самом файле — читай их.

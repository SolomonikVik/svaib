## Правила сверки (шаг 0 ритуала планирования дня)

### Что делаем

Каждое утро при планировании дня — сверяем Clockify за вчера с session-log за вчера. Цель: session-log становится единым источником правды по содержанию И времени.

### Алгоритм

1. **Подтянуть Clockify** за вчера (только проект svaib)
2. **Прочитать session-log** за вчера
3. **Сверить:**
   - Для каждой записи Clockify — есть ли соответствующая запись в session-log?
   - Для каждой записи session-log — совпадает ли длительность с Clockify?
   - Есть ли в Clockify блоки, которых нет в session-log? (встречи, мелкие задачи без сессии)
4. **Показать Виктору таблицу расхождений** — не править молча
5. **После ОК — дополнить session-log:**
   - Добавить недостающие записи (встречи, задачи без сессии)
   - Скорректировать длительность где расходится (Clockify = источник правды по времени)
   - Объединить несколько блоков Clockify в одну запись session-log, если это одна логическая сессия (например, три блока "фреймворк" = одна запись)

### Принципы соотнесения

- **Clockify = правда по времени.** Если в session-log "1ч 30м", а в Clockify суммарно 2ч 02м — верим Clockify
- **Session-log = правда по содержанию.** Clockify пишет "фреймворк" — session-log объясняет что именно делали
- **Несколько блоков Clockify → одна запись session-log**, если это одна тема (три блока "фреймворк обновляю аналитиков" = одна сессия с суммарным временем)
- **Обучение AI** — добавляется в session-log как `knowledge` (подкасты, статьи, разбор материалов — полноценная часть работы)
- **Личное** (не связанное с svaib) — не добавляется в session-log
- **Встречи с клиентами** без сессии — добавляются в session-log с пометкой направления и длительности, содержание минимальное ("Встреча с АС")

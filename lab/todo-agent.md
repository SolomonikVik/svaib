# Создание Agent — пошаговая инструкция

> **Статус: требует верификации и дистилляции.**
> Этот файл — черновик. Шаг 2 отправляет читать 4-5 файлов из knowledge/ (agents/, coding/), в которых есть внешние ссылки. Та же проблема что в todo-skill.md: координатор теряет контекст или получает неполную выжимку от субагента. Чеклист промпта (шаг 6) тоже не верифицирован на практике с текущей архитектурой.
>
> **Что нужно сделать:** в отдельной сессии пройти ВСЕ файлы knowledge/agents/ и ссылки внутри. Дистиллировать выжимку в этот файл. Верифицировать чеклист промпта.

Агент — автономный помощник со своим контекстом (200k). Используй когда задача требует изоляции, много файлов, или автономной работы. Если задача проще — начни со Skill (см. todo-skill.md).

---

## Шаг 1. Понять боль

Разговор с Виктором: что отнимает время? Как часто? Что на входе, что на выходе? Почему Skill не хватает (нужна изоляция? много файлов? автономность?)?

**Критерий:** можешь сформулировать: "Агент делает X когда Y, потому что скилл не подходит из-за Z".

---

## Шаг 2. Изучить knowledge/ по агентам

Прочитай последовательно:

1. `knowledge/agents/!agents.md` — сводка: архитектуры, протоколы, паттерны
2. `knowledge/agents/subagents.md` — субагенты и мульти-агентные архитектуры
3. `knowledge/agents/sgr.md` — Schema-Guided Reasoning: паттерны структурирования рассуждений агентов
4. `knowledge/coding/claude-code.md` — Claude Code: CLI, расширения, система расширения, формат агентов

Если агент будет использовать MCP:
5. `knowledge/agents/mcp.md` — протокол, экосистема, SDK, security

**Критерий:** знаешь паттерны, формат, ограничения. Показал Виктору что нашёл.

---

## Шаг 3. Поискать вовне

Запусти субагента (Explore): поищи в интернете похожие агенты для Claude Code.

Проверь через context7 (`/anthropics/claude-code`): актуальный формат агентов, frontmatter-поля, новые возможности.

**Критерий:** есть ответ "аналогов нет" или "нашёл X, адаптируем".

---

## Шаг 4. Проверить конфликты

Просканируй `.claude/agents/` через Glob. Нет ли агента с пересекающейся функциональностью?

**Критерий:** конфликтов нет, или конфликт обсуждён с Виктором.

---

## Шаг 5. Спроектировать

Определи: имя, trigger-based description (ЧТО + КОГДА + когда НЕ вызывать), входы, логика (Core Process), выходы, ограничения.

**Делегирование ≠ диктовка:** промпт агента передаёт задачу + сырьё, не готовый результат. Агент думает сам — координатор контролирует качество на выходе.

**Покажи Виктору. Получи одобрение.**

---

## Шаг 6. Чеклист промпта (перед написанием)

Не начинай писать файл агента, пока не проверил:

- [ ] **Iron Law** сформулирован — одно нерушимое правило, imperative форма
- [ ] **Description** trigger-based — ЧТО + КОГДА + анти-триггеры (когда НЕ вызывать)
- [ ] **Core Process** — шаги с критериями завершения каждого
- [ ] **Формат ответа** — точный шаблон выходных данных
- [ ] **Red Flags** — 5+ признаков что процесс идёт не так
- [ ] **Common Rationalizations** — таблица "отговорка → реальность → что делать" (минимум 3)
- [ ] **Верификационный шаг** перед выводом — агент проверяет себя
- [ ] **Self-contained** — работает без загрузки внешних файлов, критичное внутри промпта

Помни принцип #3 (principles.md): после ~8 правил добавление новых ухудшает поведение. Один принцип лучше десяти запретов.

Если агент сложный — lean core + reference file: детали по запросу, не грузи всё сразу (принцип #1).

---

## Шаг 7. Построить

Создай файл в `.claude/agents/`. Определи frontmatter:
- `model` — какая модель нужна (haiku/sonnet/opus)?
- `tools` / `disallowedTools` — что агент может/не может?
- `maxTurns` — лимит итераций?

**ВНИМАНИЕ:** frontmatter-поля требуют верификации — проверь актуальный формат через context7 перед использованием.

Запусти на реальном примере. Проверь: Iron Law соблюдается? Red Flags не срабатывают? Формат ответа соответствует шаблону?

# Session Handoff

Как задачи текут от плана к результату и как контекст передаётся между чатами. Целостная картина — чтобы не бегать по командам, скиллам и правилам.

## Общая схема

```
План недели → Задача → [Worklog] → Сессии → Закрытие → Session-log
```

## 1. Откуда берутся задачи

Источники задач:
- **План недели** (`meta/management/03_plan.md`) — основной источник
- **_inbox/_TODO.md** — входящие из других направлений или прошлых сессий
- **Виктор** — задача приходит в чате напрямую

## 2. Команды (роли)

Каждая команда — это режим работы в чате. Вызов: `/svaib-{направление}`.

| Команда | Область | Worklog |
|---------|---------|---------|
| svaib-framework | framework/ | framework/_inbox/_worklog.md |
| svaib-lab | lab/ | lab/_inbox/_worklog.md |
| svaib-dev | dev/ | dev/_inbox/_worklog.md |
| svaib-clients | clients/ | clients/_inbox/_worklog.md |
| svaib-knowledge | knowledge/ | knowledge/_inbox/_worklog.md |
| svaib-sprint | dev/ | dev/_inbox/_worklog.md |

При запуске команда автоматически подтягивает свой worklog и проверяет: есть ли незакрытая задача.

## 3. Worklog — передача контекста между чатами

**Проблема:** AI теряет контекст при смене чата. Виктор вынужден заново объяснять что делаем.

**Решение:** Файл `_worklog.md` в `_inbox/` каждого направления. Содержит состояние активной задачи: цель, ключевые файлы, решения, текущий статус, сообщение для следующего чата.

**Ключевые правила:**
- Это файл СОСТОЯНИЯ, не лог — перезаписывается, не растёт
- Правила работы записаны в самом файле
- Когда задача завершена — очищается до шаблона
- Не каждая задача нуждается в worklog — только если >1 сессии

## 4. Сессия (работа в чате)

Типичная сессия:
1. Команда загружается → worklog подтянут
2. Есть активная задача → "Продолжаем?" / Нет → выяснить задачу
3. Работа
4. Закрытие сессии (`/close-session`)

## 5. Закрытие сессии

Скилл `/close-session` — три фазы:

**Phase 0:** Определи тип — задача завершена или продолжение?

**Phase 1A (завершена):**
- Запись в session-log (достижение + зачем)
- Если были сессии в worklog — пометка (S{N})
- Очистка worklog до шаблона

**Phase 1B (продолжение в новом чате):**
- НЕ пишем в session-log
- Обновляем worklog: состояние, handoff, строка в мини-лог сессий

**Phase 2:** Проверка несохранённого (инсайты, решения, находки)
**Phase 3:** Маршрутизация открытых вопросов в _inbox/

## 6. _inbox/ — передача задач между направлениями

Правила: [inbox-rules.md](inbox-rules.md)

В каждом направлении есть `_inbox/` с двумя файлами:
- `_TODO.md` — список входящих задач (стремится к пустоте)
- `_worklog.md` — состояние активной задачи (стремится к шаблону)

## Связанные файлы

| Файл | Роль в workflow |
|------|----------------|
| `meta/management/03_plan.md` | Источник задач (период → неделя → день) |
| `meta/management/session-log.md` | Лог завершённых достижений |
| `lab/inbox-rules.md` | Правила работы с _inbox/ и _TODO.md |
| `.claude/skills/close-session/SKILL.md` | Полные инструкции по закрытию сессии |
| `.claude/commands/svaib-*.md` | Команды (роли) с worklog-интеграцией |

# Какой инструмент выбрать?

Навигатор по типам помощников. После выбора типа → переходи в соответствующий `todo-*.md` (там процесс + справочник, всё в одном).

## Сравнительная таблица

| | Skill | Command | Agent | Hook |
|---|---|---|---|---|
| Суть | Повторяемая процедура | Смена роли/режима на сессию | Автономная работа, изоляция | Автореакция на событие |
| Файл | `SKILL.md` в поддиректории | `.md` файл | `.md` файл | Скрипт или LLM prompt |
| Расположение | `.claude/skills/name/` | `.claude/commands/` | `.claude/agents/` | `.claude/hooks/` + `settings.json` |
| Триггер | Auto (по description) или `/name` | Только `/name` | Вызывается координатором | Автоматически при событии |
| Контекст | Основной (или fork) | Основной | Изолированный (200k) | Вне контекста (stdout при совпадении) |
| Надёжность | Markdown → LLM (~одинаковая) | Markdown → LLM (~одинаковая) | Markdown → LLM (изолирован) | Детерминированная (command) или LLM (prompt/agent) |

**Дефолт:** начни со Skill. Не хватает — апгрейд до Agent. Command — для режимов. Hook — для гардрейлов.

## Процесс создания

| Тип | Инструкция |
|-----|-----------|
| Skill | `todo-skill.md` (процесс + полный справочник формата) |
| Command | `todo-command.md` |
| Hook | `todo-hook.md` |
| Agent | `todo-agent.md` |

## Куда записывать новый инсайт

| Тип инсайта | Куда |
|-------------|------|
| Факт про механику Skills | `todo-skill.md` (справочная секция) |
| Факт про механику Commands | `todo-command.md` |
| Факт про механику Hooks | `todo-hook.md` |
| Факт про механику Agents | `todo-agent.md` |
| Общий факт / сравнение инструментов | сюда (`tooling.md`) |
| Принцип проектирования помощников | `principles.md` |
| Полный справочник платформы | `knowledge/coding/claude-code.md` |

## Заметки

**Текст не ограничивает — диалог ограничивает.** Чеклист в промпте читается, но не применяется к собственному выводу. Дефолтное поведение модели побеждает. Живой вопрос от человека работает сразу. Следствие: координатор — фильтр при приёмке результатов субагентов. Ограничение модели, решение текстом не найдено. (Источник: rescue-агент 2026-02-28)

**Верификация через внешний AI.** Результаты агентов — черновик. Прогнать через GPT/Gemini с промптом "проверь фактическую точность" — ловит подменённые термины, неправильные списки, агрессивные пороги. Не ловит: пропущенные темы, архитектурные решения. (Источник: architecture.md 2026-03-03)

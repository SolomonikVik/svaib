# Создание Command (slash command) — пошаговая инструкция

> **Статус: требует верификации и дистилляции.**
> Этот файл — черновик. Шаг 2 отправляет читать knowledge/coding/claude-code.md и существующие команды. "Ключевые факты о commands" (шаг 2) — частично верифицированы в сессии 2026-03-02 (@-ссылки работают), но полный формат не проверен через актуальную документацию.
>
> **Что нужно сделать:** в отдельной сессии верифицировать формат commands через context7, дополнить "Ключевые факты" полной выжимкой.

Command — режим работы, смена роли на сессию. Загружается при вызове `/имя-команды`. Всё содержимое файла инжектируется в контекст. Файлы, указанные через `@путь`, подгружаются целиком.

---

## Шаг 1. Понять потребность

Разговор с Виктором: какой режим работы нужен? Чем он отличается от существующих команд?

Существующие команды (проверь `.claude/commands/`):
- svaib-lab — лаборатория (координатор помощников)
- svaib-dev — разработка сайта
- svaib-sprint — спринт MVP
- svaib-context — контекст проекта
- svaib-knowledge — база знаний
- svaib-framework — фреймворк Second AI Brain
- svaib-clients — работа с клиентами

**Критерий:** новая команда не дублирует существующую. Можешь объяснить: "Команда нужна потому что ни одна из существующих не покрывает Z".

---

## Шаг 2. Изучить формат

1. Прочитай 2-3 существующие команды (`.claude/commands/svaib-*.md`) как образцы формата
2. `knowledge/coding/claude-code.md` — раздел про commands: формат, @-ссылки, frontmatter
3. Проверь через context7 (`/anthropics/claude-code`): актуальный формат commands

**Ключевые факты о commands:**
- Весь текст файла инжектируется в контекст при вызове
- `@путь/к/файлу.md` подгружает содержимое файла целиком — считай каждый токен!
- YAML frontmatter: `description` (обязательно)
- Чем короче команда, тем лучше — всё справочное выносить в файлы, которые читаются по потребности через Read tool

**Критерий:** знаешь формат, понимаешь стоимость каждой @-ссылки.

---

## Шаг 3. Спроектировать

Определи:
- Имя команды
- Роль (кто ты в этом режиме)
- Что загружается сразу (минимум!)
- Что подгружается по потребности (через Read tool, не через @)
- Быстрый старт (первый вопрос Виктору)

**Правило: команда должна быть короткой.** Ориентир — svaib-context.md (~55 строк). Максимум ~100 строк. Всё справочное — в отдельные файлы.

**Покажи Виктору. Получи одобрение.**

---

## Шаг 4. Построить

Создай файл в `.claude/commands/`.

Проверь:
- [ ] @-ссылки — каждая оправдана? Файл действительно нужен КАЖДУЮ сессию?
- [ ] Нет дублирования информации из CLAUDE.md (Identity, правила)
- [ ] Description в YAML краткий и понятный
- [ ] Быстрый старт есть (что спросить у Виктора)

---

## Шаг 5. Валидировать

Вызови команду в новом чате. Спроси: "Какие файлы сейчас в твоём контексте?" Проверь что загружается только то, что задумано.

Также добавь команду в список в CLAUDE.md (секция Slash Commands).

## Принципы проектирования команд с маршрутизацией

Выведены из runtime fix S5-S6, подтверждены arscontexta architecture. Полные выводы: `lab/_inbox/lab-runtime-conclusions.md`.

**1. Intake-блок обязателен.** Каждая команда, где Виктор может прийти с потоком мыслей, должна иметь intake: определить тип выхода (задача, план, обсуждение, делегация), показать Виктору, после ОК — действовать.

**2. Identity после выбора маршрута.** Роль/тон/stance загружается внутри маршрута, не в шапке команды. Always-on identity создаёт bias — доказано: "стратегический советник" → мета-планирование, "координатор-Джарвис" → координаторский процесс на любой задаче.

**3. Strategic context не top-level `@`.** `.claude/strategic-mode.md` читается явно внутри маршрута, который его требует. `@` в шапке делает его always-on для всех маршрутов.

**4. README папки как MOC.** Принудительно загружать `@README.md` рабочей области. README = навигатор "задача → где искать", включая knowledge/. Агент видит ландшафт до routing.

**5. Маршруты различаются типом работы, не объёмом знаний.** Engineer знает механики не хуже Architect. Разница: не уходит в стратегию, не запускает полный процесс проектирования.

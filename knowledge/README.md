# knowledge/

> Часть проекта svaib. Полная структура: [../README.md](../README.md)

## Зачем эта база

AI-помощники ограничены датой обучения — знания отстают на 6-12 месяцев. AI-технологии меняются еженедельно: новые инструменты, методологии, паттерны. Виктор находит ~10-15 материалов в неделю, из них 5-7 реально ценных. Без базы эта информация теряется — AI не имеет к ней доступа.

**knowledge/** — актуальная память AI-помощника. Чтобы он мог:
1. Давать советы на основе свежих практик, а не годовалой давности
2. Помнить что уже изучали ("мы разбирали это, смотри knowledge/X")
3. Видеть связи между находками
4. Не требовать повторного объяснения контекста

**Связь с продуктом:** knowledge/ — наш собственный пример подхода, который SVAIB продаёт клиентам. Мы строим для себя то же, что предлагаем другим. Актуальный вижен продукта — в meta/meta_context/product_vision.md.

**Два назначения:**
1. **RAG для AI** — Claude Code и Claude Chat используют эти файлы как актуальный источник знаний
2. **Публикация на сайт** — обработанные материалы (`publish: true`) будут доступны сообществу на svaib.com

**Отличие от других папок:**
- **meta/** — стратегия проекта svaib (ЧТО, ЗАЧЕМ, КТО)
- **dev/** — реализация продукта svaib (КАК строить)
- **meta/research/** — исследования рынка и клиентов ДЛЯ svaib
- **knowledge/** — внешние знания ОБ инструментах и методах (не привязаны к svaib)

## Категории

Каждая папка содержит два ключевых файла:

- **README.md** — навигация (10-15 строк): что хранить, какие файлы есть, что сюда НЕ относится (границы). AI читает ВСЕ README при добавлении материала — определяет затронутые категории.
- **!название.md** — сводка знаний: выверенный синтез по теме. Растёт неограниченно. При добавлении новой информации — интегрировать в существующий текст: проверить на противоречия, заменить устаревшее, сохранить целостность. Не дописывать в конец — перерабатывать.

Нужно быстро понять категории → читай README. Нужно понять тему целиком → читай !файл.

| Папка | Что хранить | Сводный файл |
|-------|------------|--------------|
| coding/ | Среды разработки (Claude Code, Cursor, VS Code), AI-кодинг ассистенты (Codex, Antigravity, Copilot), практики (vibe coding, agentic coding). "ВСЁ про код и разработку" | !coding.md |
| tools/ | Платформы автоматизации и AI-инструменты, НЕ связанные с кодом: n8n, Dify, Langflow, инфраструктурные сервисы. "ЧЕМ автоматизируем" | !tools.md |
| skills/ | Исполняемые инструкции для AI (SKILL.md): методологии, шаблоны, библиотеки скиллов. НЕ код — "учебники", которые AI читает и применяет | !skills.md |
| agents/ | Агентные системы: субагенты, workflows, multi-agent, паттерны проектирования, tools (function calling, MCP) | !agents.md |
| prompting/ | Техники промптинга: zero-shot, few-shot, CoT, ReAct. "КАК спросить" | !prompting.md |
| context/ | Context Engineering, RAG, Memory. "КАКУЮ информацию дать AI" | !context.md |
| cases/ | Кейсы применения: свои эксперименты, чужие истории, разборы | !cases.md |
| z_archive/ | Устаревшие материалы (не удаляем, переносим) | — |

## Формат файлов

### README.md папки (навигация)

Без YAML-заголовка. Структура:

```markdown
# {Название} — {тема}

{1 предложение: что хранить}

**Границы:** {что НЕ относится + куда перенаправить}

## Файлы

- [!name.md](!name.md) — сводка знаний
- [file.md](file.md) — однострочное описание
```

### YAML-заголовок (knowledge-файлы)

```yaml
---
title: "Развёрнутое описание содержания"
source: "https://..."
source_type: article        # article | repo | video | thread | docs | talk
status: raw                 # raw | processed | integrated | archived
added: 2025-01-28
updated: 2025-01-28         # дата последнего изменения файла
review_by: 2025-04-28       # проверить актуальность (+3 месяца)
tags: [claude-code, mcp]
publish: false              # true = готово для сайта
version: 1
---
```

### Обязательная секция

Каждый файл должен содержать секцию **"Кратко"** — 2-3 предложения с ключевыми словами для RAG-поиска. Должно быть понятно без контекста svaib (для будущей публикации).

Остальная структура файла — свободная, зависит от содержания.

## Workflow и статусы

```
raw → processed → integrated → archived
```

| Статус | Значение |
|--------|----------|
| **raw** | Сырая заметка, ссылка + тезисы |
| **processed** | Обработано, структурировано, выводы сделаны |
| **integrated** | Знание встроено в рабочие процессы svaib |
| **archived** | Устаревшее → переносить в z_archive/ |

**Как добавлять:**
1. Виктор кидает ссылку/тему в чат
2. AI оценивает: уже есть / интересно / рано / не для нас
3. Создаём/обновляем файл в нужной категории
4. При наступлении `review_by` — проверяем актуальность, обновляем или архивируем

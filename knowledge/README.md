# knowledge/

> Часть проекта svaib. Полная структура: [../README.md](../README.md)

## Назначение

База внешних знаний об AI-инструментах, методах и практиках. Информация, которой нет в памяти AI-помощника — свежие релизы, экспериментальные техники, полезные репозитории, инсайты из сообщества.

**Отличие от других папок:**
- **meta/** — стратегия проекта svaib (ЧТО, ЗАЧЕМ, КТО)
- **dev/** — реализация продукта svaib (КАК строить)
- **meta/research/** — исследования рынка и клиентов ДЛЯ svaib
- **knowledge/** — внешние знания ОБ инструментах и методах (не привязаны к svaib)

**Два назначения:**
1. **RAG для AI** — Claude Code и Claude Chat используют эти файлы как актуальный источник знаний
2. **Публикация на сайт** — обработанные материалы (`publish: true`) будут доступны сообществу на svaib.com

## Формат файлов

### YAML-заголовок

```yaml
---
title: "Развёрнутое описание содержания"
source: "https://..."
source_type: article        # article | repo | video | thread | docs | talk
status: raw                 # raw | processed | integrated | archived
added: 2025-01-28
review_by: 2025-04-28       # проверить актуальность (+3 месяца)
tags: [claude-code, mcp]
publish: false              # true = готово для сайта
version: 1
---
```

### Обязательная секция

Каждый файл должен содержать секцию **"Кратко"** — 2-3 предложения с ключевыми словами для RAG-поиска. Должно быть понятно без контекста svaib (для будущей публикации).

Остальная структура файла — свободная, зависит от содержания.

## Workflow и статусы

```
raw → processed → integrated → archived
```

| Статус | Значение |
|--------|----------|
| **raw** | Сырая заметка, ссылка + тезисы |
| **processed** | Обработано, структурировано, выводы сделаны |
| **integrated** | Знание встроено в рабочие процессы svaib |
| **archived** | Устаревшее → переносить в z_archive/ |

**Как добавлять:**
1. Виктор кидает ссылку/тему в чат
2. AI оценивает: уже есть / интересно / рано / не для нас
3. Создаём/обновляем файл в нужной категории
4. При наступлении `review_by` — проверяем актуальность, обновляем или архивируем

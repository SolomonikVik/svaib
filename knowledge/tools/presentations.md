# Презентации с AI

## Кратко

Обзор инструментов для создания презентаций с участием AI. Два полюса: markdown-first (Marp, Slidev) для file-first подхода, когда слайды живут в git и редактируются AI напрямую, и визуальные AI-генераторы (Lovable, Gamma, Slider AI) для быстрого результата без навыков дизайна. Marp и Slidev решают одну задачу — markdown → слайды — но делают принципиально разные архитектурные ставки: Marp на простоту и статический рендер, Slidev на мощь через Vue-компоненты и dev-сервер. Ключевой тренд: AI убирает барьер дизайна, контент становится главным.

---

## Два подхода

| Подход | Суть | Для кого | Примеры |
|--------|------|----------|---------|
| **Markdown-first** | Слайды = текстовый файл. Git, версионирование, AI редактирует напрямую | Разработчики, file-first адепты | Marp, Slidev |
| **Визуальный AI** | Закидываешь контент → получаешь готовый дизайн | Менеджеры, предприниматели, все кто не дружит с дизайном | Lovable, Gamma, Slider AI |

## Markdown-first: Marp vs Slidev

Оба инструмента конвертируют markdown в презентацию и экспортируют в PDF/PPTX. Но делают это по-разному — и выбор между ними определяется не тем «какой лучше», а тем «под какую задачу».

### Философская ставка

| | **Marp** | **Slidev** |
|---|---|---|
| Ставка | Простота, чистый markdown, быстрый статический рендер | Мощь, интерактив, компонентный подход |
| Под капотом | Marpit framework (статика) | Vite + Vue 3 (dev-сервер) |
| Формат слайда | Markdown + `---` как разделитель + CSS для кастомизации | Markdown + Vue-компоненты + HTML |
| Требует | Ничего (standalone binary) или `@marp-team/marp-cli` | Node.js + npm + запуск dev-сервера |
| Автор | marp-team | Anthony Fu (автор Vite-экосистемы) |
| Лицензия | MIT | MIT |

Marp — «включил VS Code, написал markdown, получил PDF». Slidev — «разработческий инструмент с полным контролем и визуальными эффектами, ценой настройки окружения».

### Функциональное сравнение

| Фича | Marp | Slidev |
|---|------|--------|
| Live preview в VS Code | Да (официальное расширение) | Да (через браузер) |
| Экспорт PDF, PPTX, PNG | Да, нативно | Да |
| Встроенные темы | Три (default, gaia, uncover) + CSS | Обширная библиотека тем |
| Mermaid-диаграммы | Через HTML/плагины | Встроено |
| Monaco Editor (live coding в слайде) | Нет | Да |
| Magic Move (анимация трансформации кода) | Нет | Да |
| Presenter View с таймером | Нет | Да |
| Запись презентации с камерой | Нет | Да |
| Real-time collaboration | Нет | Да |
| Графики, SVG, кастомные диаграммы | Через HTML/CSS (возможно с правильным скиллом) | Через Vue-компоненты |
| Потребление ресурсов | Лёгкий (статический рендер) | Тяжелее (Vite + dev-сервер) |
| Кривая входа | Минимальная — знаешь Markdown, готово | Выше — полезно понимать Vue для кастомизации |

### Поддержка AI-агентов (Claude Code, Cursor)

**Это главная точка, которая отличает инструменты структурно, а не косметически.**

**Slidev** поставляет **официальный Agent Skill** — от тех же авторов, что разрабатывают сам инструмент. Устанавливается через `npx skills add slidevjs/slidev`, живёт в репозитории Slidev (`slidev/skills/slidev`), синхронизирован с актуальной версией. В документации Slidev есть выделенная страница «Work with AI» — AI-интеграция у них в первом классе. Есть активные community-форки (`cc-slidev`, `clearfunction/cf-devtools`) для специализированных сценариев.

**Marp** официального скилла от marp-team не имеет. Community-скиллов много, разного качества: от минимальных «знай синтаксис» до богатых пакетов с несколькими темами, SVG-графиками, dashboard-компонентами (robonuggets/marp-slides, nibzard/marp-slide-quality, davila7/claude-code-templates, omerr/claude-skills и другие). Одни авторы open-source-скиллов прямо признают, что большинство доступных вариантов посредственные и делают свой доработанный форк.

**Практический вывод:** Slidev выигрывает долгосрочно за счёт официальной поддержки — скилл обновляется вместе с инструментом. Marp выигрывает тактически — его проще обернуть в свой кастомный скилл именно потому, что инструмент проще: меньше степеней свободы → меньше шансов ошибиться.

### Когда что выбрать

**Marp — когда:**
- Нужны быстрые рабочие презентации (клиентский deck, внутренняя сводка, обзор проекта)
- Важна простота сборки: открыл VS Code → preview → PDF одной кнопкой
- Не нужен интерактив или сложные анимации
- Презентация должна жить как простой .md в git и читаться глазами
- Нет желания возиться с Node.js и dev-сервером

**Slidev — когда:**
- Технические демо с live coding, интерактивом, анимацией кода
- Конференционные доклады, где важна визуальная подача и Presenter View
- Нужна запись презентации с камерой
- Нужен полный контроль через Vue-компоненты
- Ценится real-time collaboration в команде

**Оба сильнее визуальных AI-генераторов, когда:**
- Презентация живёт в git рядом с кодом/документацией
- AI должен редактировать слайды напрямую (find & replace, regex, смена темы)
- Нужны диффы изменений между версиями
- Контент важнее оформления

### Скиллы для AI-генерации презентаций

Для обоих инструментов существуют Skills под Claude Code (и совместимые оболочки — Cursor, Antigravity):

**Slidev:**
- Официальный: `npx skills add slidevjs/slidev`
- cc-slidev (rhuss/cc-slidev) — Claude Code plugin с guardrails для технических презентаций

**Marp:**
- robonuggets/marp-slides — несколько тем, SVG charts, dashboard-компоненты
- nibzard/marp-slide-quality — анализ качества слайдов через SlideGauge
- davila7/claude-code-templates, omerr/claude-skills — общие пакеты
- Плюс множество мелких community-вариантов, включая доработки от тематических Telegram-каналов

Паттерн одинаков: скилл учит AI синтаксису инструмента, даёт хорошие дефолты тем/цветов/структуры, добавляет команды вида «сгенерируй презентацию по этому README».

## Визуальные AI-генераторы

### Lovable

Vibe-coding платформа (lovable.dev), которая неожиданно хороша для презентаций.

- **Кейс:** Человек прошёл путь ChatGPT → AI-генераторы → NotebookLM → Claude Code → Lovable. Неделя мучений с другими инструментами, Lovable выдал живую редактируемую презентацию сразу
- **Сильная сторона:** Нормальный дизайн без навыков дизайна. «Без боли, без обучений, без шаманства с промптами»
- **Ограничение:** Основное назначение — сайты и приложения, презентации — побочный юзкейс

### NotebookLM (Google)

AI-инструмент для работы с документами. Загружаешь файлы и ссылки → генерирует контент.

- **Для презентаций:** Выдаёт хороший PDF на основе загруженных материалов
- **Ограничение:** PDF нередактируемый — нельзя подправить результат

### Slider AI

Российский онлайн-редактор презентаций с AI.

- **Сайт:** https://slider-ai.ru
- **Суть:** Шаблоны + AI-генерация + 40+ типов диаграмм
- **Тарифы:** Бесплатный базовый доступ, Pro от 399 ₽/мес, Business от 1195 ₽/мес
- **Аудитория:** Маркетологи, менеджеры, предприниматели

### Другие упоминания

- **Gamma** — AI-генератор презентаций (визуальный подход)
- **Figma Slides** — Slides от Figma (для дизайнеров)

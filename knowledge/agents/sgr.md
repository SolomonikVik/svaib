# Schema-Guided Reasoning (SGR)

## Кратко

Техника принудительного структурирования рассуждений LLM через предопределённые Pydantic-схемы. Вместо надежды на промпт — модель физически не может ответить вне схемы (constrained decoding). Автор — Ренат Абдулин. Три базовых паттерна: Cascade (последовательность), Routing (ветвление), Cycle (повторение). Паттерны компонуемые — вкладываются друг в друга для построения сложных агентных workflow.

## Суть подхода

SGR не диктует архитектуру агента — он описывает как структурировать **отдельные шаги рассуждения** внутри любой архитектуры. Ключевая идея: схема (Pydantic-модель) выступает как строгий чеклист, через который проходит LLM. Constrained decoding гарантирует, что модель заполнит все поля в заданном порядке.

**Отличие от промптинга:** промпт просит — схема принуждает. CoT предлагает "думай пошагово", SGR определяет конкретные шаги и их формат. Результат — предсказуемый, аудитируемый, воспроизводимый.

**Практическое следствие:** даже дешёвые модели справляются со сложными задачами, если рассуждение структурировано через схему.

## Три паттерна

### Cascade (каскад)

Последовательность обязательных шагов. Модель проходит каждый — пропустить невозможно.

**Когда:** задача с логическими этапами, где каждый следующий зависит от предыдущего.

**Пример:** Оценка кандидата: (1) суммаризация опыта → (2) оценка соответствия 1-10 → (3) решение hire/reject/hold.

**Механизм:** Pydantic-поля в определённом порядке с ограничениями (Annotated, Ge/Le).

### Routing (маршрутизация)

Принудительный выбор одного пути из нескольких. Union-типы создают взаимоисключающие ветки.

**Когда:** классификация, диспетчеризация, выбор инструмента.

**Пример:** Тикет поддержки → `Union[HardwareIssue, SoftwareIssue, UnknownIssue]`. Каждая ветка имеет свои обязательные поля.

**Вариация:** `List[Union[...]]` для параллельного вызова нескольких инструментов.

### Cycle (цикл)

Принудительное повторение одного и того же шага N раз. List с MinLen/MaxLen.

**Когда:** нужно несколько независимых оценок одной структуры (риски, аргументы, примеры).

**Пример:** Оценка рисков: `List[RiskFactor]` с MinLen(2), MaxLen(4) — модель обязана найти минимум 2 фактора.

## Композиция

Паттерны складываются: Cascade может содержать Routing на одном из шагов, а внутри Routing — Cycle. Это позволяет строить сложные workflow из простых блоков.

## Связь с нашими практиками

SGR формализует то, что мы делаем интуитивно в скиллах и агентах SVAIB:
- **Iron Law + Core Process** в скиллах — это неформальный Cascade
- **Routing** — наши агенты с trigger-based description выбирают "дорожку"
- **Cycle** — секции Red Flags, Common Rationalizations — повторяющиеся проверки

Потенциал: если перевести скиллы на строгие Pydantic-схемы — поведение станет предсказуемее (но потеряется гибкость natural language инструкций). Тема для дальнейшего исследования.

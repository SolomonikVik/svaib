# Agents — Агентные системы

## Кратко

AI-системы, способные автономно принимать решения, использовать инструменты и выполнять многошаговые задачи. Субагенты (изолированные AI для отдельных задач), workflows (детерминированные пайплайны), multi-agent системы (несколько агентов вместе), паттерны проектирования. Также здесь: tools в контексте агентов — функции, которые агент вызывает (API, БД, поиск) через function calling или MCP (Model Context Protocol — стандарт подключения AI к внешним сервисам).

Знания о проектировании и построении агентов — рабочий материал для продукта SVAIB (см. product_vision.md).

## Ключевые концепции

- **Agent** — AI, который сам решает какие инструменты вызвать и в каком порядке
- **SubAgent** — изолированный агент, вызываемый основным агентом для конкретной подзадачи
- **Workflow** — детерминированный пайплайн с предопределёнными шагами (в отличие от агента)
- **MCP (Model Context Protocol)** — стандарт подключения AI к внешним сервисам ("USB-C для AI")
- **Function calling / Tools** — механизм вызова функций агентом

## Паттерны проектирования (из практики)

### Gateway-Agent-Skills-Memory
Архитектура из проекта OpenClaw (см. openclaw.md). Четыре компонента с чёткой ответственностью: Gateway абстрагирует канал доставки, Agent — reasoning, Skills — модульные действия, Memory — персистентный контекст на Markdown. Паттерн подтверждает: Skills как плагины + Memory на файлах + проактивность агента — рабочая комбинация для self-hosted AI-ассистентов.

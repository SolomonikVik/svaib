# Промптинг — сводка знаний

## Кратко

Промптинг — "КАК спросить AI". Техники формулировки запросов, структура системных промптов, инструменты оптимизации. Отличие от context engineering ("КАКУЮ информацию дать AI" → context/) и skills ("упакованная экспертиза в формате SKILL.md" → skills/). Здесь: от базовых техник (CoT, few-shot) до анализа production system prompts крупных AI-продуктов.

---

## Коллекция системных промптов AI-инструментов

**Репозиторий:** https://github.com/x1xhlol/system-prompts-and-models-of-ai-tools

Коллекция реальных production system prompts, извлечённых из коммерческих AI-инструментов. 30+ продуктов: Cursor, Claude Code, Lovable, Manus, Replit, v0, Windsurf, Devin, Kiro, Google Antigravity, VSCode Agent (GitHub Copilot) и другие. Репозиторий активно обновляется, промпты добавляются по мере выхода новых версий инструментов. Крупный community-проект.

Промпты не подтверждены авторами сервисов, но community считает их подлинными. Структура и детализация промптов подтверждают это косвенно — такое невозможно сфабриковать.

**Что внутри для каждого инструмента:** системный промпт (текст) + описания инструментов (JSON). У некоторых — несколько версий (Cursor — история эволюции промпта), модульная декомпозиция (Manus — prompt + modules + agent loop + tools), промпты под разные модели (VSCode Agent — отдельный промпт для каждой модели).

### Когда использовать

**Как продакту:** Изучать стратегии промптинга — как разработчики направляют модели, какие ограничения ставят, как структурируют tool use. Сравнивать подходы разных продуктов к одним задачам (кодинг, планирование, работа с файлами).

**Как разработчику skills/agents:** Референс для проектирования собственных системных промптов. Видно как production-продукты решают: приоритизацию инструкций, обработку edge cases, баланс между свободой и ограничениями модели, форматы вывода.

**Как "бесплатный клон":** Взять system prompt любого инструмента, вставить в Google AI Studio или другой чат — получить функциональный аналог бесплатно (без tool use, но с логикой поведения).

### Наблюдения из коллекции

- **Prompt + Tools separation** — универсальный паттерн: текстовые инструкции и JSON-описания инструментов передаются отдельно. Отражает архитектуру API (system message + tool schemas).
- **Per-model tuning** — продвинутые продукты используют разные промпты для разных моделей. Не один промпт на все случаи.
- **Масштаб** — production промпты это десятки килобайт текста. Серьёзные документы с иерархией секций, примерами, правилами приоритизации.
- **Модульная архитектура** — Manus разложен на planner, knowledge, datasource модули. Не монолитный промпт, а система.
- **Версионирование** — Cursor показывает эволюцию промпта через несколько версий. Промпты итерируются как код.

## Инструменты оптимизации промптов

### Вендорские (от создателей моделей)

**Anthropic Prompt Improver** — встроен в Console (console.anthropic.com). Бесплатный. Переписывает промпт с применением: CoT-секций, XML-форматирования примеров, обогащения примеров шагами рассуждений, assistant prefill для контроля формата вывода. Самый релевантный для нас — мы строим на Claude.

**OpenAI Prompt Optimizer** — встроен в Playground (platform.openai.com/playground/prompts). Бесплатный. Показывает diff оригинала и оптимизированной версии. Находит противоречия, добавляет недостающую структуру. Полезен при миграции промптов между версиями моделей.

**Google Vertex AI Prompt Optimizer** — три режима: zero-shot (одиночный промпт), few-shot (анализ ошибок, уточнение инструкций), data-driven (batch-оптимизация по метрикам на тестовых наборах). Единственный из трёх поддерживает оптимизацию для кастомных локальных моделей.

### Независимые

**PrompTessor** (promptessor.com) — скоринг промптов по шести измерениям (clarity, specificity, context, goal orientation, structure, constraints). Генерирует варианты улучшений. Есть reverse engineering — загружаешь результат, получаешь промпт. Freemium. Учебный инструмент, не для production.

**Prompt Cowboy** (promptcowboy.ai) — другая точка входа среди оптимизаторов: не улучшает готовый промпт, а вытаскивает его из потока сознания через 3 уточняющих вопроса (Socratic-интервью). Собирает результат по фреймворку SITUATION-TASK-OBJECTIVE-KNOWLEDGE. Модель-агностично. Freemium, базовое без регистрации. Авторы — Nicolas Cole и Dickie Bush (writewithai). Полезнее для пользователей, которые ещё не умеют формулировать AI-запросы, чем для инженерии system prompts.

## Библиотеки и гайды

**DAIR.AI Prompt Engineering Guide** — https://github.com/dair-ai/Prompt-Engineering-Guide — крупнейший open-source справочник по техникам промптинга. Покрывает: zero-shot, few-shot, CoT, ReAct, Tree of Thoughts, RAG, adversarial prompting. Полезен как учебный ресурс для начинающих, содержание — базовые знания.

**OpenAI Cookbook** — https://cookbook.openai.com — практические примеры и гайды от OpenAI. Включает model-specific guides при выходе новых моделей. Стоит проверять при миграции между моделями.

## Extraction prompts

Отдельная сводка: [extraction-prompts.md](extraction-prompts.md). Покрывает entity extraction (бизнес-сущности) и behavioral extraction (поведенческие паттерны) из транскриптов и текстов. Ключевые паттерны: quote-first extraction, structured decomposition, camera test (NVC), vocabulary blocklist, anti-pattern few-shot, pipeline architecture (parallel extractors → synthesizer). Инсайт (март 2026): при 1M+ контексте один промпт с методологией > цепочка агентов — "телефонная игра" теряет контекст на стыках.

## Связи с другими файлами knowledge/

- **skills/** — скиллы это "упакованные промпты с методологией". Паттерны из системных промптов AI-инструментов применимы при проектировании SKILL.md
- **agents/** — модульная архитектура промптов (Manus: planner + knowledge + datasource) = паттерн проектирования агентных систем
- **context/** — порядок контента в промпте, per-model tuning = пересечение с context engineering
- **coding/claude-code.md** — Claude Code system prompt есть в коллекции

## Current Release: MVP v3.2

### Настройка проекта

**Шаг 1:** Клиент указывает папку проекта на Google Drive

**Шаг 2:** Svaib создаёт внутри папку "svaib Documentation/" со структурой файлов

**Шаг 3:** AI читает существующие документы клиента, индексирует для поиска

**Шаг 4:** Брифинг — AI задаёт уточняющие вопросы (голосом или текстом): цели проекта, roadmap, команда, термины. Клиент отвечает → AI достраивает контекст

**Результат:** Структурированный контекст проекта + AI готов отвечать на вопросы

### Цикл командной встречи

**ДО встречи:**
- Презентация проекта уже готова
- Задачи с прошлой встречи, статусы, roadmap, AI-инсайты — всё актуально
- Клиент открывает → сразу работает

**ВО ВРЕМЯ встречи:**
- Svaib подключается к встрече как участник (Zoom/Meet/Teams)
- Записывает разговор
- Команда ведёт встречу по презентации

**ПОСЛЕ встречи:**
- Транскрибация с разметкой спикеров
- AI извлекает задачи и решения
- Обновляет таблицу задач (Google Sheets)
- Отправляет протокол в командный Telegram-чат
- Презентация автоматически готова к следующей встрече

---

### AI-ассистент (MVP-функции)

**Базовые функции:**
- Отвечает на вопросы по проекту (RAG lite: поиск релевантных файлов → чтение целиком → ответ)
- Присылает протоколы встреч
- Напоминает о задачах
- Обновляет документы по запросу

**RAG lite (как работает):**
- AI индексирует ключевые файлы клиента (5-10 документов)
- При вопросе — semantic search находит топ-3 релевантных файла
- Читает их целиком и формирует ответ
- Современные LLM держат 128K+ токенов — этого достаточно

**Telegram-команды:**
- `/status` — статус проекта, открытые задачи
- Протокол приходит автоматически после встречи

### Частный случай: Управленческая планерка

Компания с регулярными управленческими встречами (еженедельные/ежемесячные).

**Особенности:**
- Презентация включает метрики компании с графиками (автообновление из Sheets)
- Блоки по направлениям (продажи, маркетинг, производство)
- AI-инсайты по метрикам

**Логика презентации:**
Миссия → стратегия/долгосрочные цели → метрики → отклонения → задачи. От долгосрочных целей компании спускаемся к конкретным задачам на неделю.

Всё остальное работает так же: запись → транскрибация → фиксация → готовая презентация к следующей встрече.

### Риски и ограничения

**1. Качество транскрибации — узкое место**
Если ASR плохо распознаёт русскую речь с терминами, специфичными названиями — весь продукт посыпется. Критическая зависимость.

**2. "Презентация как интерфейс" — непривычно**
Некоторые клиенты могут не понять: "Зачем мне слайды на планёрке?" Нужно объяснять ценность структуры.

**3. Брифинг при настройке — потенциально долго**
Если AI задаёт много вопросов, клиент устанет. Нужен баланс: достаточно для контекста, не слишком для терпения.

**4. Google-only**
Привязка к Google Workspace. Часть клиентов на Microsoft 365. На MVP это ок, но ограничивает рынок.

**5. Конкуренция**
Fireflies, Otter, Fathom уже есть. Нужно чётко коммуницировать разницу: "Это не просто транскрибатор, это AI-помощник который ведёт проект"

### Что сознательно НЕ входит в MVP

| Фича | Почему не сейчас | Когда |
|------|------------------|-------|
| Полный RAG с чанками | Для enterprise с 100+ документами | Фаза 2 |
| Проактивные напоминания | Нужна базовая работа сначала | Фаза 1 |
| Паттерны команды | Требует накопленных данных | Фаза 3 |
| Telegram /ask с AI | Это уже ассистент, не MVP | Фаза 1 |
| Интеграция с task-менеджерами | После валидации базового сценария | Фаза 2 |
| Web UI для ассистента | Telegram достаточно | Фаза 2 |

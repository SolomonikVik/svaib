# Product Roadmap

> **⚠️ АРХИВ (13.03.2026):** roadmap ниже описывает прошлый meeting/presentation pivot. Для текущего направления сайта ориентироваться на `site_rebuild_plan.md`, `site_rebuild_creative_brief.md` и `meta/management/02_goal.md`.

## О документе

**Что это:** Карта развития продукта svaib — фазы, компоненты, статусы реализации.

**Как устроен:**

* **Phases** — этапы развития продукта (что делаем сейчас, что потом)
* **Product Details** — компоненты продукта с точки зрения клиента (что видит, как использует, что уже работает)
* **Ideas** — сырые идеи, которые ещё не привязаны к конкретным компонентам

**Принцип работы с компонентами:**

* Каждый компонент имеет таблицу "Возможности" со статусами
* Идея дозревает → попадает в таблицу нужного компонента → реализуется
* Если компонент разрастается — выносим детали в отдельный guide-файл

**Статусы возможностей:**

* ✅ Реализовано
* 🔄 MVP (в разработке)
* 🔜 Фаза N (запланировано)
* 💡 Идея (не привязано к фазе)

**Связанные документы:**

* `framework/product.md` — зачем мы существуем, какие боли решаем
* `product_release.md` — детали текущего релиза (MVP)
* `guide_presentation.md` — подробная документация по презентации
* `guide_team_patterns.md` — паттерны команды (анализ командной динамики)

## Phases

### Phase 0: MVP *(в реализации)*

Базовый продукт для 3-5 пилотных клиентов:

* Бот приходит на встречу и записывает
* Транскрипция с разметкой спикеров
* AI извлекает задачи и решения
* Задачи в Google Sheets, протокол в Telegram
* Презентация с автообновлением

→ **Детали:** см. `product_release.md`

***

### Phase 1: Интеллектуальный анализ *(планируется)*

> ⚠️ **Черновик** — требует пересмотра после завершения MVP

* Автоформирование повестки следующей встречи
* AI-инсайты по метрикам и трендам
* Проактивные напоминания
* Управленческая память компании
* Гибридный RAG (chunking для больших документов)

***

### Phase 2: Интеграции и автоматизация *(планируется)*

> ⚠️ **Черновик** — требует пересмотра после завершения MVP

* Паттерны команды (ReasoningBank lite) — базовый анализ командной динамики
* Интеграция с task-менеджерами (Trello, Bitrix, Asana)
* Подключение CRM и календарей
* Асинхронный сбор апдейтов до встречи
* Связка встреч с OKR

***

### Phase 3: Управленческая ОС *(планируется)*

> ⚠️ **Черновик** — требует пересмотра после завершения MVP

* Все типы встреч (отделы, 1-на-1, стратсессии)
* BI-слой с визуализацией
* AI-коуч для руководителей
* ReasoningBank full (ежемесячные дайджесты, глубокий анализ паттернов)

***

### Phase 4: Платформа *(планируется)*

> ⚠️ **Черновик** — требует пересмотра после завершения MVP

* Управленческий каркас + AI-роли (маркетолог, РОП, HR)
* Полная история встреч в RAG
* Управленческая платформа для МСБ

## Product Details

### 📊 Презентация (Google Slides)

#### Роль в системе

Центральный интерфейс для проведения встреч.

#### Описание

Презентация — это не просто слайды. Это живой инструмент, который структурирует встречи и держит команду в едином контексте. Открыл → всё актуально → провёл встречу по структуре → ничего не забыл.

Логика: миссия → цели → метрики → отклонения → задачи. От общего к частному.

#### Возможности

| Возможность                     | Статус    |
| ------------------------------- | --------- |
| Шаблон презентации с блоками    | 🔄 MVP    |
| Автообновление метрик из Sheets | 🔄 MVP    |
| Задачи с прошлой встречи        | 🔄 MVP    |
| AI-инсайты по направлениям      | 🔜 Фаза 1 |
| Автоформирование повестки       | 🔜 Фаза 1 |

#### Подробнее

См. `guide_presentation.md`

***

### 🤖 AI-ассистент (Telegram/Web)

#### Роль в системе

Интерфейс общения с контекстом проекта.

#### Описание

Можно задать любой вопрос по проекту — ассистент знает всё: документы, историю встреч, задачи, решения. Присылает протоколы, напоминает о дедлайнах, подсвечивает риски.

#### Возможности

| Возможность                                 | Статус    |
| ------------------------------------------- | --------- |
| Отправка протокола в Telegram               | 🔄 MVP    |
| Команда /status (статус проекта)            | 🔄 MVP    |
| RAG lite (поиск + чтение файлов + ответ)    | 🔄 MVP    |
| Гибридный RAG (chunking для больших файлов) | 🔜 Фаза 1 |
| Проактивные напоминания                     | 🔜 Фаза 1 |
| Еженедельный дайджест                       | 🔜 Фаза 1 |
| Web-интерфейс                               | 🔜 Фаза 2 |
| Паттерны команды (ReasoningBank lite)       | 🔜 Фаза 2 |
| ReasoningBank full                          | 🔜 Фаза 3 |

#### Подробное описание

**Базовые функции:**

* Отвечает на вопросы по проекту (читает контекст + ищет в документах)
* Присылает протоколы встреч
* Напоминает о задачах
* Обновляет документы по запросу

**Проактивные функции (Фаза 1+):**

* Сигналы о проблемах: "Задача X висит 3 недели без движения"
* Подготовка к встрече: "Что обсудим: 3 задачи в блоке, 2 дедлайна горят"
* Еженедельный дайджест: "5 задач закрыто, 2 новых, риск — задача Z без ответственного"

**Паттерны команды (Фаза 3):**
Что команда не замечает о себе: динамика принятия решений, распределение "эфирного времени", темы-бумеранги, моменты сопротивления. Формат: ежемесячный дайджест или по запросу руководителя.

→ Подробнее: см. `guide_team_patterns.md`

### 📝 Контекст компании (Google Docs)

#### Роль в системе

Единая база знаний о компании.

#### Описание

Всё, что нужно знать о компании в одном месте: цели, структура, процессы, история решений. AI использует этот контекст для ответов на вопросы и генерации инсайтов.

#### Возможности

| Возможность                           | Статус    |
| ------------------------------------- | --------- |
| Базовый набор документов (5-7 файлов) | 🔄 MVP    |
| Шаблоны для быстрого старта           | 🔄 MVP    |
| Индексация для поиска                 | 🔄 MVP    |
| Автообновление из транскрипций        | 🔜 Фаза 1 |
| Версионирование и история             | 🔜 Фаза 2 |

#### Подробное описание

**Что включено:**

*Стратегический уровень:*

* Миссия и видение компании
* Долгосрочные цели (3-5 лет)
* Ценности и принципы

*Тактический уровень:*

* OKR на квартал/год
* Активные проекты и инициативы
* KPI по департаментам

*Операционный уровень:*

* Оргструктура и роли
* Регламенты и процессы
* Глоссарий терминов

*Исторический контекст:*

* Ключевые решения прошлых встреч
* Lessons learned
* Хронология важных событий

### 📈 Метрики (Google Sheets)

#### Роль в системе

Источник данных для визуализации прогресса.

#### Описание

Ключевые показатели компании, которые автоматически попадают в презентацию. Не нужно готовить отчёты — открыл презентацию, всё уже там.

#### Возможности

| Возможность                                     | Статус    |
| ----------------------------------------------- | --------- |
| Базовые дашборды (продажи, маркетинг, операции) | 🔄 MVP    |
| Автообновление графиков в презентации           | 🔄 MVP    |
| AI-анализ трендов                               | 🔜 Фаза 1 |
| Автоимпорт из CRM/ERP                           | 🔜 Фаза 2 |
| Predictive analytics                            | 🔜 Фаза 3 |

#### Подробное описание

**Типы метрик:**

* Финансовые — выручка, прибыль, расходы, маржинальность
* Операционные — клиенты, конверсии, LTV, CAC
* Продуктовые — MAU/DAU, retention, churn
* Командные — velocity, загрузка

***

### ✅ Задачи (Google Sheets)

#### Роль в системе

Трекер выполнения договорённостей.

#### Описание

Все задачи из встреч в одном месте. AI автоматически извлекает их из разговора. Задачи не теряются между встречами — договорились → зафиксировали → проверили.

#### Возможности

| Возможность                          | Статус    |
| ------------------------------------ | --------- |
| Автоизвлечение задач из транскрипции | 🔄 MVP    |
| Запись в Google Sheets клиента       | 🔄 MVP    |
| Идемпотентность (нет дубликатов)     | 🔄 MVP    |
| Напоминания о дедлайнах              | 🔜 Фаза 1 |
| Интеграция с Trello/Bitrix/Asana     | 🔜 Фаза 2 |

#### Подробное описание

**Поля задачи:**

* Задача (что сделать)
* Ответственный (кто)
* Дедлайн (когда)
* Статус (новая/в работе/готово/блок)
* Источник (из какой встречи)

**Статусы:**

* ✅ Выполнено
* 🔄 В работе
* ❌ Не выполнено
* ⏳ Ожидает

***

### 🎤 Транскрибация

#### Роль в системе

Превращение разговора в структурированный текст.

#### Описание

Svaib подключается к встрече как участник, записывает разговор, транскрибирует с разметкой "кто что сказал". Это основа для извлечения задач и решений.

#### Возможности

| Возможность                                    | Статус  |
| ---------------------------------------------- | ------- |
| Подключение к Zoom/Meet/Teams                  | 🔄 MVP  |
| Транскрипция с диаризацией (разметка спикеров) | 🔄 MVP  |
| Поддержка русского языка                       | 🔄 MVP  |
| Акцентирование задач голосом ("Svaib, запиши") | 💡 Идея |
| Мультиязычность                                | 💡 Идея |

#### Подробное описание

**Как работает:**

1. Клиент добавляет Svaib в календарное приглашение
2. Бот подключается к встрече как участник
3. Записывает аудио
4. После встречи — транскрибация с разметкой спикеров
5. AI извлекает задачи и решения

**Технологии:**

* Recall.ai — подключение к встречам
* Soniox — транскрибация с диаризацией
